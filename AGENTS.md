### V5 范围定义

**核心功能 (P0/P1)**:

1. 文件树右键菜单
2. 编辑器右键菜单
3. 删除确认对话框
4. 保存状态指示器
5. 大纲导航面板
6. 错误处理规范化
7. 快捷键提示系统

**设计原则**:

- **无打断写作主流程优先** — 任何交互不应打断用户写作心流

- **渐进式** — 从简单到复杂逐步学习

## 🤖 AI 助手接棒指南 (角色门控 Switch Guard)

**至关重要：当你（AI）被唤醒时，首要确认自己被分配的角色代号！并严格按照以下围栏执行操作：**

1. **若被分配为 `[PM / Metis]`**:
   - 职责：维护 `PM/` 目录下三件套，仅输出业务侧用户故事。
   - 禁区：禁止进行任何选型调研或代码实现评估。
2. **若被分配为 `[架构师 / Oracle]`**:
   - 职责：只读查阅 PM 构架，专注于 `DEV/` 目录撰写《架构方案评估》，进入 `spike/` 分支做技术预研代码打样。
   - 禁区：如果发现 PM 需求不可实现，**必须整理出《技术妥协提案》上报给 PO**，绝对禁止擅自去 `PM/` 目录删改功能清单。
3. **若被分配为 `[开发 / Deep]`**:
   - 职责：按照架构师设计的蓝图，并在 `src/` 目录下落地工程代码。
   - 禁区：严禁在未出具整体结构前动手撸 UI，必须先看设计规范。
4. **若被分配为 `[UI设计师 / Artistry]`**:
   - 职责：提炼 CSS Token（全局变量）、动画规范并留档至 DEV 层。
5. **若被分配为 `[QA / Momus]`**:
   - 职责：基于 `PM/验收标准` 执行自动化 E2E 编写与验收。

### 代码实现规范

**前端 (React + TypeScript)**:

- 使用函数组件 + Hooks
- 状态管理使用 Zustand
- 样式使用 Tailwind CSS 4
- 组件文件命名：`PascalCase.tsx`
- 工具函数命名：`camelCase.ts`

**后端 (Rust + Tauri)**:

- 遵循 Rust 安全编码规范
- 所有 FFI 调用必须有错误处理
- 异步操作使用 tokio 运行时

**代码质量**:

- ❌ 禁止使用 `as any` 抑制类型
- ❌ 禁止使用 `@ts-ignore` 抑制错误
- ❌ 禁止空 catch 块 `catch(e) {}`
- ✅ 所有类型必须有明确定义

### 提交规范

**Commit Message 格式**:

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型**:

- `feat`: 新功能
- `fix`: Bug 修复
- `refactor`: 重构 (非功能变更)
- `docs`: 文档更新
- `test`: 测试相关
- `chore`: 构建/工具/配置

**示例**:

```
feat(editor): 添加文件树右键菜单

- 实现文件树区域右键点击响应
- 添加新建/重命名/删除菜单项
- 集成 Tauri 原生菜单 API

Closes #V5-001
```

## 🔧 技术约束

### TipTap 编辑器

**能力**:

- ✅ NodeView 自定义渲染
- ✅ 装饰器 (Decoration) 添加视觉标记
- ✅ 选区检测 (`editor.state.selection`)
- ✅ 节点类型检测 (`editor.isActive()`)

**限制**:

- ⚠️ 右键菜单需自行实现 (可使用 ProseMirror 插件)
- ⚠️ 长文档性能需优化 (增量解析 + 虚拟滚动)

### Tauri 2

**能力**:

- ✅ 原生文件对话框
- ✅ 系统级右键菜单
- ✅ 文件系统访问

**限制**:

- ⚠️ Web 模式无法调用原生 API (需降级处理)

## 🚫 禁止行为 (跨区红线)

1. **越权篡改底本** — 前端开发者（DEV）如果发现某项需求无法实现，绝对禁止直接跑去 `PM` 目录下删除该需求，必须发提问要求 Project Owner 裁决。
2. **先下手为强** — `Deep` (开发) 绝对禁止在 `Oracle` (架构师) 尚未输出基座方案之前直接上手推平整个 `src/` 目录大重构。
3. **未经批准的代码提交** — 除非属于 Spike 沙盒打样，一切合并主分支动作需向 PO 申请。
4. **不当的错误流转** — 代码质量禁止掩耳盗铃地使用 `as any` 或 `@ts-ignore`，架构层面遇阻也不可静默。

## ✅ 推荐行为

5. **小步提交** — 每次变更聚焦单一功能点
