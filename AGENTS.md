## 🤖 AI 助手工作指南

### 启动新任务前

1. **阅读需求文档** — `docs/current/PM/V5 需求文档.md`
2. **查看功能清单** — `docs/current/PM/V5 功能清单.md`
3. **确认验收标准** — `docs/current/PM/V5 验收标准.md`
4. **检查进度板** — `docs/current/V5 项目进度板.md`

### 代码实现规范

**前端 (React + TypeScript)**:

- 使用函数组件 + Hooks
- 状态管理使用 Zustand
- 样式使用 Tailwind CSS 4
- 组件文件命名：`PascalCase.tsx`
- 工具函数命名：`camelCase.ts`

**后端 (Rust + Tauri)**:

- 遵循 Rust 安全编码规范
- 所有 FFI 调用必须有错误处理
- 异步操作使用 tokio 运行时

**代码质量**:

- ❌ 禁止使用 `as any` 抑制类型
- ❌ 禁止使用 `@ts-ignore` 抑制错误
- ❌ 禁止空 catch 块 `catch(e) {}`
- ✅ 所有类型必须有明确定义

### 提交规范

**Commit Message 格式**:

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型**:

- `feat`: 新功能
- `fix`: Bug 修复
- `refactor`: 重构 (非功能变更)
- `docs`: 文档更新
- `test`: 测试相关
- `chore`: 构建/工具/配置

**示例**:

```
feat(editor): 添加文件树右键菜单

- 实现文件树区域右键点击响应
- 添加新建/重命名/删除菜单项
- 集成 Tauri 原生菜单 API

Closes #V5-001
```

## 🔧 技术约束

### TipTap 编辑器

**能力**:

- ✅ NodeView 自定义渲染
- ✅ 装饰器 (Decoration) 添加视觉标记
- ✅ 选区检测 (`editor.state.selection`)
- ✅ 节点类型检测 (`editor.isActive()`)

**限制**:

- ⚠️ 右键菜单需自行实现 (可使用 ProseMirror 插件)
- ⚠️ 长文档性能需优化 (增量解析 + 虚拟滚动)

### Tauri 2

**能力**:

- ✅ 原生文件对话框
- ✅ 系统级右键菜单
- ✅ 文件系统访问

**限制**:

- ⚠️ Web 模式无法调用原生 API (需降级处理)

## 🚫 禁止行为

1. **不要提交代码** — 除非用户明确要求
2. **不要抑制类型错误** — 使用 `as any` 或 `@ts-ignore`
3. **不要删除失败的测试** — 修复根本原因
4. **不要提交敏感文件** — `.env`, `credentials.json` 等
5. **不要进行大规模重构** — 除非明确要求

---

## ✅ 推荐行为

1. **并行执行** — 独立的读取/搜索/代理调用同时发起
2. **使用后台任务** — explore/librarian 代理使用 `run_in_background=true`
3. **遵循现有模式** — 代码风格与现有代码库保持一致
4. **验证后再提交** — 运行 `lsp_diagnostics` 检查错误
5. **小步提交** — 每次变更聚焦单一功能点
