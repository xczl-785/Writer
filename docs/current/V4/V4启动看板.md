## 重构背景

V1-V3 阶段聚焦功能交付与稳定性增强，代码库已具备完整的核心能力。但在快速迭代过程中积累了技术债务：

1. **Editor.tsx 过于庞大**：1319 行代码，承担工具栏、查找替换、表格弹窗、快捷键等多重职责
2. **状态管理分散**：多个 Zustand store 之间的同步逻辑散落在组件中
3. **错误处理不一致**：部分用 console.error，部分用 StatusStore
4. **硬编码常量**：魔法数字散落在各处

V4 的目标是在不改变功能的前提下，通过重构提升代码可维护性和可测试性。

## 风险提示

1. 重构可能引入回归，需强化测试覆盖
2. 大规模拆分可能影响开发效率，需分阶段推进
3. 需确保 V3 功能不受影响
