# Sprint 分期（全阶段路线图）

## 1. 总体原则

1. 终点是 V1 完成（见 `docs/current/V1完成标准.md`）。
2. 每个 Sprint 都有明确“做/不做”。
3. Should/Could 不进入 V1 主线承诺。
4. 执行口径以任务拆分与 QA 门禁为准；路线图负责阶段节奏，不替代 DoD 明细。

## 2. 分期路线

### Sprint 0（0.5 周）— 规划冻结

- 目标：冻结执行口径与评估方式。
- 范围：
  1. 终点定义确认
  2. 全阶段路线确认
  3. 门禁口径确认
- 输出：
  - `docs/current/V1完成标准.md`
  - `docs/current/V1全阶段路线图.md`
- 退出条件：路线图被确认并作为后续唯一节奏基线。

### Sprint 1（1-1.5 周）— 编辑核心闭环

- 目标：完成最小可写闭环。
- 范围（Must 子集，以 `docs/current/Sprint1任务拆分.md` 为准）：
  1. 即时渲染编辑
  2. 快捷键基础子集（标题 1..3、B/I）
  3. 代码块创建与编辑（```）
  4. 撤销/重做（平台等价快捷键）
  5. 基础自动保存链路
- 不做：图片粘贴、新建/重命名/删除、搜索、标签页、导出等。
- 退出条件：用户可稳定完成单文档连续写作。

### Sprint 2（1-1.5 周）— 文件管理闭环

- 目标：完成单工作区文件流。
- 范围（Must 子集）：
  1. 文件夹作为工作区
  2. 文件树展示与切换
  3. 新建/重命名/删除
  4. 切换时 dirty flush
- 不做：搜索、标签页、多工作区。
- 退出条件：工作区内主流程完整可用。

### Sprint 2.5（0.5-1 周）— 交互体验纠偏

- 目标：对齐文件树交互规范与平台习惯，收口体验缺陷。
- 范围：
  1. 新建/重命名/删除交互规范化
  2. Win/macOS 快捷键映射与上下文条件
  3. 文件树显示策略一致性（markdown 过滤、后缀展示）
  4. 错误反馈文案与焦点流收口
- 不做：拖拽、多选、搜索、多工作区。
- 输出：
  - `docs/current/Sprint2.5交互规范.md`
  - `docs/current/Sprint2.5任务拆分.md`
  - `docs/current/QA_CHECKLIST_SPRINT2_5.md`
- 退出条件：S2.5 交互清单全部通过，且无高优先级体验回归。

### Sprint 3（1 周）— 图片与可靠性闭环

- 目标：补齐媒体与数据安全链路。
- 范围：
  1. 图片粘贴与落盘（assets）
  2. 自动保存三策略完整化
  3. D-009 原子写入验证
  4. 保存失败可见反馈闭环
- 退出条件：关键异常不丢数据、反馈可见。

### Sprint 4（0.5-1 周）— 回归与发布就绪

- 目标：完成 V1 验收收口。
- 范围：
  1. Must 12 全量回归
  2. macOS/Windows 关键路径验证
  3. QA 清单通过
  4. 文档与状态收口
- 退出条件：满足 `V1_DONE_CRITERIA` 全部条款。

## 3. 推迟内容处理规则（Should/Could）

1. 统一进入 Post-V1 Backlog，不纳入当前 Sprint 承诺。
2. 允许做接口预留，不允许做功能实现。
3. `表格可视化编辑` 作为 Post-V1 Top Priority。

## 4. 风险控制节奏

1. 每个 Sprint 末必须复盘并固化资产。
2. 若门禁失败，下一 Sprint 不得继续扩范围。
3. 任何新增需求默认进入 Backlog。
