# V3 需求主文档

- 版本：V3
- 状态：Sprint0 梳理中
- 最后更新时间：2026-02-18

## 1. 目标与价值

V3 延续 V2 已验证的核心能力，聚焦“稳定性 + 效率 + 可回归”。

1. 关键编辑链路可恢复、可撤销、可验证。
2. 高频交互在不改变心智前提下持续优化。
3. 继续维持 FR -> Task -> QA -> Evidence 可追溯闭环。

## 2. 范围边界

## 2.1 范围内（Sprint0 待细化）

- 编辑器稳定性与破坏性操作一致化。
- 表格高频操作与边界行为增强。
- 文件树高频流程一致性优化。
- 回归效率和证据标准化。

## 2.2 范围外

- 未进入追溯矩阵且未通过评审的新需求。
- 未评审依赖引入与大规模架构重构。

## 2.3 变更规则

1. 范围变更必须同步更新 `docs/current/V3 需求追溯矩阵.md`。
2. 影响 Sprint 目标的变更必须同步更新 `docs/current/V3启动看板.md`。

## 3. 完成标准（门禁）

1. 需求闭环：PRD -> 任务 -> QA -> 证据路径完整。
2. 自动化门禁：`npm run test`、`npm run build` 持续通过。
3. 缺陷门禁：P0/P1 清零，P2 有留档与修复计划。
4. 文档门禁：`docs/current` 与 `docs/archive` 链接无断链。

## 4. 路线图（Sprint 级）

- Sprint0：需求梳理、范围冻结、任务拆分、追溯骨架建立。
- Sprint1：核心稳定性能力实现与基础回归。
- Sprint2：体验增强与边界修复。
- Sprint3：稳定性治理与全量回归。
- Sprint4：发布审计、证据归档、Go/No-Go 结论。

## 4.1 Sprint1 候选任务池（首批）

1. S1-V3-01：统一破坏性动作提示文案与状态回收策略。
2. S1-V3-02：破坏性动作撤销链路回归（含误删恢复）。
3. S1-V3-03：表格删除与边界行为（空段落、连续表）一致化。

## 4.2 Sprint2 候选任务池（预备）

1. S2-V3-01：表格复杂边界场景修复（跨块选择/多步编辑）。
2. S2-V3-02：文件树高频操作一致性（根目录/选择态/创建目标）。
3. S2-V3-03：查找替换反馈与性能优化。

## 4.3 Sprint3 候选任务池（预备）

1. S3-V3-01：高风险回归最小集执行与问题收敛。
2. S3-V3-02：证据模板标准化与归档一致性检查。

## 5. 技术/设计/工程基线

## 5.1 技术基线

- 前端：React + TypeScript + Vite
- 编辑器：TipTap
- 桌面：Tauri

## 5.2 设计基线

1. 新交互必须可发现、可撤销、可验证。
2. 保持现有视觉与交互一致性，渐进优化。

## 5.3 工程基线

1. 按 Sprint 顺序推进：设计 -> 实现 -> 测试 -> 归档。
2. 关键变更必须提供回归证据与验收口径。
