# V5 组件规范与 CSS Token (组件规范.md)

**版本**: V5.0
**状态**: 草案
**设计**: Artistry (UI/UX)

基于现存 MVP 界面的截图分析（灰白色调、细线分割、蓝色为主行动色点缀），V5 新增的交互组件需严格复用以下定义的原子化 CSS Token，以便开发阶段对接 Tailwind CSS 4。

---

## 1. 原则与跨平台设定

考虑到跨平台要求（macOS, Windows, Linux），我们**不使用高斯模糊 (Backdrop-filter: blur)**，确保低性能设备或某些 Linux 桌面环境下渲染一致。

所有“浮层”（菜单、提示框、Dialog）的核心结构统一为：
```html
<!-- 跨平台浮层基类 (Floating Layer Base) -->
<div class="bg-white dark:bg-zinc-800 border border-zinc-200 dark:border-zinc-700 shadow-xl rounded-md overflow-hidden">
  ...
</div>
```

---

## 2. 核心 CSS Token 定义

### 2.1 色彩系统 (Colors)

| 类别 | 变量用途 | Tailwind 亮色域 | Tailwind 暗色域 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **Surface 背景** | 主背景色 (如编辑器) | `bg-zinc-50` 偏暖灰/白 | `bg-zinc-900` | 减少绝对的 `#FFFFFF/000000` 造成的刺眼 |
| | 浮层背景色 (菜单/弹窗) | `bg-white` | `bg-zinc-800` | 浮层需要比主背景更亮（暗态中同理），以建立层级 |
| | 选中高亮状态 (Hover/Active) | `bg-zinc-100` | `bg-zinc-700` | 用于文件树列表或菜单项的热区高亮 |
| **Text 文本** | 主文本色 | `text-zinc-800` | `text-zinc-100` | |
| | 辅助文本色 (Tooltip/Meta) | `text-zinc-500` | `text-zinc-400` | |
| **Border 边框** | 分界线/描边 | `border-zinc-200` | `border-zinc-700` | 包括分隔线、Input框边界、浮层物理边界 |
| **State 状态指示** | 动作点缀色 (Primary) | `text-blue-500`/`bg-blue-500` | 同左 | 用于选中、聚焦态。对应原图截图的选区蓝与链接蓝 |
| | 已保存 (Success) | `text-emerald-500` | 同左 | 左下角指示器的最后落点 |
| | 未保存 (Warning) | `text-amber-500` | 同左 | 发生 Dirty 时的挂起态 |
| | 报错阻断 (Danger) | `text-red-500` | 同左 | 无权限保存或破坏性操作警醒 |

### 2.2 形状与深度 (Shape & Elevation)

| 属性 | Tailwind 变量 | 适用场景 |
| :--- | :--- | :--- |
| **圆角 (Radius)** | `rounded-md` (约 6px) | 标准组件圆角（如上下文菜单项、按键、Toast）。不宜过大以保持专业感。 |
| | `rounded-lg` (约 8px) | 系统级拦截（Dialog 对话框）的面板大圆角。 |
| **阴影 (Shadow)** | `shadow-sm` | 行内轻量级按键Hover。 |
| | `shadow-md` | Bubble Menu 行内浮动菜单。 |
| | `shadow-xl` | Context Menu 右键菜单、文件删除二次确认 Dialog。用深阴影代替毛玻璃产生视觉隔离。 |

### 2.3 排版与字号 (Typography)

*   **12px (`text-xs`)**: 次要信息、Tooltip 的快捷键提示。
*   **13px/14px (`text-sm`)**: 系统UI的主要控制字号（如菜单项文本、左侧文件树名称、按键标语）。
*   **16px (`text-base`)**: 编辑器默认正文字号、弹窗主标题。

## 3. V5 核心组件规格示例

### 3.1 菜单项 (Menu Item)

*   **高度**: `h-8` (32px) 或内边距 `py-1.5 px-3`
*   **布局**: 左右分布 Flex。左侧图标/名称，右侧快捷键（`text-xs text-zinc-400`）。
*   **交互**: 默认无背景。Hover 时背景转为 `bg-zinc-100/dark:bg-zinc-700`，文字不换色（除非是警告项）。
*   **危险项**: 例如“移至废纸篓”，默认文本为红色 `text-red-500`。Hover 时背景换为柔和的浅红色 `bg-red-50 dark:bg-red-950/30`。

### 3.2 对话框 (Dialog)

*   **遮罩 (Backdrop)**: 完全覆盖界面的半透明黑 `{bg-black/20}`。无模糊。
*   **控制台 (Panel)**: 宽带约束在 `max-w-md`。包含 Header（标题部分）、Body（阐述文字）和 Footer（操作按钮组）。
*   **按键组**: 采用主辅区分。
    *   取消类默认按钮: `bg-white border-zinc-200 text-zinc-700 hover:bg-zinc-50`
    *   主行动或危急按钮: 实心色，如 `bg-red-500 text-white hover:bg-red-600`

### 3.3 左下角保存指示器 (Status Indicator)

*   **尺寸**: 一枚 `10px x 10px` 的圆点，或带有等比例的极简 SVG 图标。
*   **包含文字**: 不建议常态带文字。文字（如“已保存”、“无权限”）可以通过向右滑动展开的动效弹出，或通过 Hover 的 Tooltip 提示，以极致压缩侧边栏底盘空间。

---
_开发对接提示：本规范中枚举的 Tailwind 类名为约束基准，开发落地时建议采用 `@apply` 抽出通用组件，或封装基于这些基本 Token 的 React Component。_
