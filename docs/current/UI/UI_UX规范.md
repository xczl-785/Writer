# Writer V5 UI/UX 核心规范 (Artistry 版)

## 0. 设计哲学：极致克制
Writer V5 的设计核心是 **“只在需要时出现”**。我们坚信：最好的界面是让用户忘记界面的存在，从而全身心投入到“造物心流”中。

---

## 1. 结构与布局 (Layout Architecture)

### 1.1 窗口区域划分
*   **侧边栏 (Sidebar)**：仅存放 **资源管理器 (Explorer)**。采用 `bg-zinc-50` 淡色背景。
*   **主编辑区 (Main Editor)**：全屏留白，无固定工具栏。两侧预留 `10%-15%` 的视口边衬（Padding）。
*   **顶栏 (Header)**：展示路径面包屑 (`文件夹 / 文件名`)。右侧仅保留一个常驻图标：`[ ☰ ] 文档大纲 (Outline)`。
    *   **命名策略**: 资源管理器与面包屑中仅展示文件名，**隐藏 `.md` 后缀**。
*   **全局底栏 (Status Bar)**：高度 28px。承载字数统计、编码格式以及 **V5 呼吸灯状态指示器**。
    *   **编码显示 (Encoding)**：默认显示 `UTF-8`。该功能依赖代码侧的字符编码检测机制，并预留切换入口以解决老旧文件的乱码问题。
    *   **Git 同步状态 (SYNC)**：仅在当前工作空间为 Git 仓库时显示。用于反馈本地硬盘与远程仓库的对齐状态。

### 1.2 大纲处理策略 (Popover Mode)
采用方案 C。点击顶栏 `☰` 图标，在右侧弹出带有精致阴影的 **Popover 菜单**。点选标题即跳转，跳转后弹窗消失。

### 1.3 面包屑导航 (Breadcrumb)
在编辑器顶部展示当前文件路径。支持点击文件夹名在侧边栏中定位。长路径采用中部截断策略 (`Workspace > ... > parent > file.md`)。

## 2. 三层递进式交互模型 (Interaction Hierarchy)

### 层级一：肌肉记忆层 (Natural Input)
针对基础语法（标题、列表、引用等），仅支持实时渲染，不提供界面按钮。

### 层级二：雕刻模式层 (Bubble Menu)
针对行内排版。鼠标划选文本后在选区上方弹出黑底白字的 **Mini-Format-Bar**。
*   **包含项**：加粗、斜体、删除线、高亮(==)、行内代码、超链接。
*   **弹出定位**：菜单水平居中于选区（Selection），垂直方向距离选区上边缘 **8px - 10px**。若选区靠近顶端，则自动翻转至选区下方。
*   **键盘避让**：不夺取焦点，支持快捷键直接操作。

### 层级三：认知导航层 (Ghost Slash & Menu)
解决复杂语法（表格、代码块等）的发现压力。
*   **Ghost Slash (幽灵斜杠)**：空行时光标后方显示灰度提示 `/ 输入以唤出菜单...`。
    *   **样式**：`text-zinc-300` (亮色), `text-zinc-600` (暗色)，斜体 (Italic)，禁止选中。
*   **Slash Menu**：键入 `/` 唤出。支持 **原地搜索 (In-place Search)** 模式。
    *   **交互逻辑**：菜单弹出后，用户直接在正文录入搜索词。录入的文本作为 **Command Fragment (命令碎片)** 渲染。
    *   **搜索反馈**：菜单项随输入实时模糊过滤。按 `Enter` 执行，按 `Esc` 或 `Backspace` 清除搜索词并回退。
    *   **快捷键指引**：明确标注对应快捷键。

## 3. 视觉规范细节 (Visual Standards)

### 3.1 跨平台中性风格 (Neutral Modern)
*   **色盘**：以 `Zinc` 灰阶为基础，主行动色为 `Blue-500`。
*   **选定态**：侧边栏文件选中不再高亮整行，改为文字加粗加深，并在左侧显示 `3px` 宽的蓝色指示线。
*   **块级边界**：光标进入代码块或引用块时，左侧显示高亮装订线（Gutter Line）。

### 3.2 呼吸灯指示器 (Status Dot)
*   **Unsaved**: `amber-400` 常亮。
*   **Saving**: `blue-500` 呼吸闪烁。
*   **Saved**: `green-500` 低透明度（30%）暗示。
*   **Error**: `red-500` 弹跳警示。

## 4. OS 原生顶部菜单层级 (OS Native Menus)

所有功能通过原生菜单兜底，作为用户的“最终词典”。

*   **文件 (File)**：
    *   新建 (`⌘N`) / 打开 (`⌘O`) / 保存 (`⌘S`) / 另存为 (`⇧⌘S`) / 导出 (PDF, HTML, Image)。
*   **编辑 (Edit)**：
    *   撤销 (`⌘Z`) / 重做 (`⇧⌘Z`) / 剪切 (`⌘X`) / 复制 (`⌘C`) / 粘贴 (`⌘V`) / 全选 (`⌘A`) / 查找替换 (`⌘F`)。
*   **段落 (Paragraph)**：
    *   标题 1~6 (`⌘1 ~ ⌘6`) / 引用 (`⇧⌘Q`) / 代码块 (`⌥⌘C`) / 表格 (`⌥⌘T`) / 列表 (有序/无序/任务) / 分割线 (`---`) / 数学公式块 (`⌥⌘M`)。
*   **格式 (Format)**：
    *   加粗 (`⌘B`) / 斜体 (`⌘I`) / 删除线 (`⇧⌘X`) / 下划线 (`⌘U`) / 高亮 (`⇧⌘H`) / 超链接 (`⌘K`) / 图片 (`⇧⌘I`) / 行内代码 (`⌘\``)。
*   **视图 (View)**：
    *   显示大纲 (`⇧⌘O`) / 切换侧边栏 (`⌘\`) / 专注模式 (`F11`) / 源代码模式 (`⌘/`)。
