# V5 UI/UX 规范 (UI_UX规范.md)

**版本**: V5.0
**状态**: 草案
**设计**: Artistry (UI/UX)

基于“沉静、专注、无打断”的产品原则，为了兼容 macOS、Windows 和 Linux，本规范采用 **现代中性风格 (Neutral Modern)** 策略，摒弃平台特异性的材质（如高斯模糊渲染），通过精确的极简边框、实色背景与微观动效构建跨平台一致的高级感。

---

## 一、 整体视觉基调 (Visual Tone)

根据现有的 MVP 界面风格（浅色为主，灰色线条区隔，顶部标题栏融合），V5 的扩展设计应遵循以下美学：

1. **大面留白与结构线**：使用非常轻的辅助线 (`border-zinc-200`) 划分区域，避免色块堆叠。
2. **中性灰度体系**：文字和图标使用不同层级的灰度（如 `text-zinc-800` 为主文本，`text-zinc-500` 为辅助说明）建立信息层级，色彩仅用于表达状态（如保存、错误或危险操作）。
3. **“平面+浮层” 深度模型**：底层视图完全扁平化；所有的浮动层（如右键菜单、提示框、Dialog）必须有明确的阴影 (`shadow-lg` 或 `shadow-xl`) 和圆角边框，以与底层内容物理隔离。

## 二、 交互微动效规范 (Micro-Interactions)

所有动画参数必须遵循“克制且迅捷”的原则。

| 交互类型 | 响应时长范围 | 缓动曲线 (Easing) | Tailwind 类名参考 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **悬停 (Hover)** | 100ms - 150ms | `ease-out` | `transition-colors duration-150 ease-out` | 菜单项高亮、按钮悬停、列表项 Hover |
| **出现 (Enter)** | 150ms - 200ms | `ease-out` | `animate-in fade-in zoom-in-95 duration-200` | 右键菜单弹出、Tooltip 出现、Dialog 弹窗 |
| **消失 (Exit)** | 100ms - 150ms | `ease-in` | `animate-out fade-out zoom-out-95 duration-150` | 右键菜单收起、Dialog 关闭 |
| **状态流转** | 300ms - 500ms | `ease-in-out`| `transition-opacity duration-500` | 状态栏保存圆点的显隐、颜色渐变过渡 |

> **法则**：任何操作响应不得阻断用户输入，动画不能成为操作瓶颈。

## 三、 组件交互行为准则

### 3.1 上下文菜单 (Context Menu & Bubble Menu)

*   **跨平台材质**：背景使用纯色（Day: `bg-white`, Night: `bg-zinc-800`），外加 `border border-zinc-200 dark:border-zinc-700` 与 `shadow-lg`。
*   **触发时机**：
    *   右键击中元素：立刻（0ms延迟）渲染出菜单，辅以 `150ms` 的出现动画。
    *   划选文本松开：100ms 防抖判断后，在选区上方/下方弹出 Bubble Menu。
*   **消失时机**：点击菜单项外任意区域、或执行某菜单项命令后立刻关闭。
*   **列表行为**：支持系统级的键盘上下箭头导航，回车执行。

### 3.2 快捷键视觉提示系统

*   **展示时机**：当用户的鼠标悬停在包含快捷键的独立控件（例如工具栏按钮）上超过 **400ms** 时，显示 Tooltip。
*   **跨平台文案**：
    *   Mac: 使用符号（如 `⌘N`, `⇧⌘P`）。
    *   Win/Linux: 使用文本（如 `Ctrl+N`, `Ctrl+Shift+P`）。
*   **样式表现**：Tooltip 采用深色背景 `bg-zinc-800 text-white` 以提供最高对比度，字号限定为 `text-xs`。

### 3.3 状态反馈 (Status Indicator)

*   **唯一性**：取消零散的全局 Toast 通知（除非系统严重错误），所有的文档状态流转收敛于状态栏左/右下角的指示器。
*   **无扰退场**：当状态从“保存中”流转回“已保存”稳态时，绿色对勾图标应在停留 5 秒后，将透明度降低至 30% (`opacity-30`)，避免视觉干扰。

### 3.4 破坏性操作与容错 (Destructive Actions)

*   **物理文件销毁（如左侧树删除）**：必须阻断并要求二次确认。视觉上，删除按钮文字需标红 (`text-red-500`)，确认弹窗必须将焦点默认放在“取消”按钮上。
*   **数据流错误（如文件无权限、磁盘满）**：不采取模态弹窗阻断。状态栏图标变为红色警示 ⚠️，悬停该图标时，上方浮现包含“错误原因 + 行动建议”的面板（例：“无写入权限。建议另存为新文件”），面板内提供快捷操作按钮。

## 四、 页面结构骨架

根据截图分析，当前 UI 采用经典的三栏式变体布局。

1.  **顶栏 (Header Bar)**：32px - 40px 高度。包含系统窗口控制区（红黄绿灯）、应用标题及居中或右侧的状态/文件信息。
2.  **左侧栏 (Sidebar / File Tree)**：固定或可拖拽宽度（例如 250px），背景色与其右侧的主编辑区需有微秒的明暗区分（或通过 1px 的竖线 `border-r` 分割）。
3.  **主编辑区 (Editor Area)**：核心区域。最大化留白。

### V5 新增层：抽屉式大纲 (Outline Drawer)

*   为保持极简，右侧的大纲面板应设计为**悬浮抽屉**或**可弹性折叠侧边栏**。
*   背景色继承主编辑区（Day: `#FFFFFF`），通过左侧 `border-l` 与主区域划分界线。
