# V1 终点定义（Done Criteria）

## 1. 目标

定义“V1 完成”判定标准，避免开发过程中目标漂移。

## 2. 完成判定（全部满足才算 V1 完成）

### 2.1 范围完成

1. Must 12 全部实现并通过验收（以 `docs/frozen/PRD_V1.md` 为准）。
2. 不得引入 Not in V1 能力。

### 2.2 质量门禁完成

1. D-007 通过：Markdown Roundtrip Spike 通过并有报告。
2. D-009 通过：原子写入策略落地并验证不破坏原文件。
3. 自动保存策略完整生效：800ms 防抖 + 失焦 + 关闭前强制保存。

### 2.3 平台完成

1. macOS 可用。
2. Windows 可用。
3. Linux 为可选，不阻塞 V1 完成。

### 2.4 验收与回归完成

1. Must 12 的功能验收清单全部通过。
2. 核心路径回归通过：

- 打开工作区 -> 新建/编辑 -> 自动保存 -> 重启恢复
- 文件重命名/删除 -> 编辑上下文正确
- 图片粘贴落盘 -> 引用正确

3. 关键异常场景有可见反馈（不可静默失败）。

### 2.5 文档完成

1. 各 Sprint 验收清单完整并通过：
   - `docs/archive/2026-02-15_sprint1-sprint2-closeout/QA_CHECKLIST_V1.md`
   - `docs/archive/2026-02-15_sprint1-sprint2-closeout/QA_CHECKLIST_SPRINT2.md`
   - `docs/current/QA_CHECKLIST_SPRINT3.md`
2. `docs/current/V1 需求追溯矩阵.md` 状态更新为 `DONE`，并完成 FR->Task->QA->Evidence 回填。
3. 复盘结论沉淀到 `docs/archive/`，冻结约束更新到 `docs/frozen/`（若有必要）。

## 3. 非完成状态（任一命中即未完成）

1. Must 12 存在未通过项。
2. 门禁未通过（尤其 D-007/D-009）。
3. 关键流程存在数据丢失或不可见错误。
4. 仅“能跑通”但无回归验收记录。
