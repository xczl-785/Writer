# Sprint 2.5 问题修复留档

用于记录 Sprint 2.5 开发与回归阶段发现的问题、根因和修复结果。

## 使用规则

- 仅记录“已复现”问题。
- 每条问题必须带用例ID（若来自 QA 清单）。
- 修复后必须补充“回归结果”。

## 问题记录模板

| 编号         | 阶段      | 来源用例ID | 严重级别      | 现象 | 根因 | 修复方案 | 影响范围 | 状态                | 回归结果 | 备注 |
| ------------ | --------- | ---------- | ------------- | ---- | ---- | -------- | -------- | ------------------- | -------- | ---- |
| S2.5-BUG-001 | 开发/回归 |            | 阻断/高/中/低 |      |      |          |          | OPEN/FIXED/DEFERRED |          |      |

## 严重级别定义

- 阻断：无法继续主流程验证或造成误删/错写等高风险行为。
- 高：核心流程可继续但结果不可信，需要优先修。
- 中：有明确偏差但可绕过。
- 低：体验问题或文案问题。
