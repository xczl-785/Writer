# Sprint 2.5 交互修正方案（最终确认版）

## 1. 目标

纠正文件树（Explorer）的新建与重命名交互，对齐 VS Code 核心体验，消除视觉抖动与逻辑不一致。

## 2. 交互逻辑修正

### A. 新建（Create）- Ghost Node 模式

- **视觉位置**：
  - 不再使用顶部独立输入框。
  - **Ghost Node**：在目标父目录下（或当前选中文件同级）动态插入一个临时节点。
  - 图标：根据 `New File` / `New Folder` 显示对应图标。
- **提交策略（Enter）**：
  - 校验名称：
    - 合法且不重名 → 创建成功 → 替换为真实节点（File 自动打开）。
    - 失败（空/非法/重名） → **保留输入状态**并 Alert 提示原因（给用户修正机会）。
- **取消策略（Esc）**：
  - 销毁临时节点。
- **失焦策略（Blur）**：
  - **尝试提交（Try Commit）**（修正点）。
  - 若名称为空 → **视同取消**，销毁节点。
  - 若名称不为空：
    - 校验通过 → 创建成功。
    - 校验失败（重名/非法） → **保持输入状态**（不销毁，输入框标红或仅聚焦），或者为了简化：**视同取消**（避免复杂错误态维护）。
    - _决策_：鉴于复杂度，**新建时 Blur 若校验失败 -> 自动取消**（防止残留脏节点）。

### B. 重命名（Rename）- 原地变身模式

- **视觉位置**：
  - 移除 Save / Cancel 按钮。
  - Input 高度、边距与普通文本节点完全一致（去 Border，改背景色），消除高度抖动。
- **提交策略（Enter）**：
  - 校验名称：
    - 成功 → 更新节点。
    - 失败（空/重名/非法） → **Alert 提示原因**，然后 **重置回旧名称**（简化策略）。
- **取消策略（Esc）**：
  - 重置回旧名称，退出编辑态。
- **失焦策略（Blur）**：
  - **尝试提交（Try Commit）**。
  - 若名称未变 → 静默退出。
  - 若名称改变：
    - 校验通过 → 保存。
    - 校验失败 → **静默回滚**到旧名称（不弹窗，不卡死）。

## 3. 实现变更范围

### UI 层 (`Sidebar.tsx`)

1. **状态重构**：
   - 移除 `createMode`（顶部输入框开关）。
   - 引入 `ghostNode` 状态：`{ parentPath: string, type: 'file'|'folder', tempName: string }`。
   - 渲染逻辑：在遍历节点时，匹配 `ghostNode.parentPath`，在子节点列表首位插入 Input 组件。
2. **样式重构**：
   - 封装 `<InlineInput />` 组件：
     - 样式：`w-full bg-transparent outline-none border-none p-0 m-0 leading-none h-auto`。
     - 行为：自处理 Enter/Esc/Blur。
3. **事件处理**：
   - 移除 Save/Cancel 按钮 DOM。
   - 统一 `handleCommit` 逻辑供 Enter 和 Blur 调用。

### 状态层

- 确保 `FileTreeNode` 能区分“真实节点”与“Ghost 节点”。

## 4. 复杂度评估

- **中等**。核心难点在于 Ghost Node 的插入位置计算（特别是多级展开时）。
- 方案可行性：高。

## 5. 验收标准

1. 点击新建 -> 树中出现临时输入框（非顶部）。
2. 重命名 -> 输入框无缝覆盖原文本，无高度跳动。
3. Blur 行为：新建空名消失；新建合法名提交；重命名合法提交；重命名非法回滚。
4. Enter 行为：新建非法弹窗提示；重命名非法弹窗提示并回滚。
