# V2 范围冻结

## 1. 目标

冻结 V2 开发范围，防止执行期需求漂移。

## 2. Must（V2）

1. 表格可视化编辑（Top Priority）
2. 快速文件搜索（工作区内）
3. 查找/替换
4. 顶部快捷菜单（按钮式格式操作入口）

## 2.1 Must-Guard（质量保障项）

1. V1 核心链路回归不破坏
2. 稳定性门禁通过（测试、构建、关键失败反馈）

## 2.2 已实现能力（不纳入 V2 开发承诺）

1. 应用内新建文件夹（V1 已实现，进入回归范围）

## 3. Should（V2）

- 图片拖拽插入
- 面包屑导航
- 最近文件列表

## 4. Could（V2）

- 字数统计
- 主题切换
- 自定义字体/字号

## 5. Not in V2

- 云同步、多人协作、账号系统
- 插件生态、知识图谱
- 多工作区并行编辑

## 6. 冻结规则

1. 未进入 Must 的项不得在 V2 Sprint 承诺。
2. 新需求默认进入 Backlog，评审后纳入下一周期。
