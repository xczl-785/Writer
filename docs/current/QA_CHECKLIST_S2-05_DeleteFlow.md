# S2-05 文件树删除流 - 手工测试用例

> 目标：覆盖「打开目录 -> 新建文件/文件夹 -> 删除节点 -> 状态清理」主流程。

## 测试前置

- 以 Tauri 模式启动应用（非纯浏览器模式）。
- 准备测试目录：`~/Downloads/test`。
- 目录结构建议：

```text
test/
  a.md
  b.md
  folder1/
    c.md
    sub/
      d.md
```

## 用例清单

### TC-01 打开目录成功

- 步骤：点击 `Open Folder`，选择 `~/Downloads/test`。
- 预期：
  - 左侧不再显示 `No folder opened`。
  - 文件树出现目录内容。
  - 右侧 `State Debug -> Path` 显示所选目录绝对路径。

### TC-02 取消打开目录

- 步骤：点击 `Open Folder`，在系统对话框中点击取消。
- 预期：
  - 不弹错误提示。
  - 当前已打开目录状态不被清空。

### TC-03 新建文件

- 步骤：点击顶部 `New File` 图标，输入 `new-note.md`，回车或点击 `Create`。
- 预期：
  - 左侧树中出现 `new-note.md`。
  - 编辑区自动打开该文件。
  - `State Debug -> Active/Open Files` 包含新文件路径。

### TC-04 新建文件夹

- 步骤：点击顶部 `New Folder` 图标，输入 `new-folder`，回车或点击 `Create`。
- 预期：
  - 左侧树中出现 `new-folder`。
  - 不自动打开编辑器文件（Active 保持当前文件或 None）。

### TC-05 新建输入校验

- 步骤：进入新建输入框后输入空字符串（仅空格）并提交。
- 预期：
  - 弹出名称不能为空提示。
  - 不创建任何节点。

### TC-06 删除单文件（非激活）

- 步骤：对未激活文件点击删除图标并确认。
- 预期：
  - 文件从树中消失。
  - `State Debug -> Open Files` 不包含该文件（若原本已打开）。
  - 其他文件状态不受影响。

### TC-07 删除当前激活文件

- 步骤：先打开 `a.md`，再删除 `a.md` 并确认。
- 预期：
  - 文件从树中消失。
  - `Active` 自动切换到其他已打开文件；若无则为 `None`。
  - `Editor` 中不再保留 `a.md` 对应状态。

### TC-08 删除目录（含已打开文件）

- 步骤：打开 `folder1/c.md` 与 `folder1/sub/d.md`，随后删除 `folder1` 并确认。
- 预期：
  - `folder1` 整棵子树消失。
  - `Open Files` 清理所有以 `folder1/` 为前缀的文件。
  - `Editor` 清理同前缀文件状态。
  - 若激活文件位于该目录中，`Active` 切换到安全值（其他打开文件或 None）。

### TC-09 事件隔离

- 步骤：在目录节点上点击删除图标。
- 预期：
  - 仅触发删除确认。
  - 不触发目录展开/折叠。

### TC-10 回归：重命名/打开不受影响

- 步骤：执行一次重命名、一次点击文件打开。
- 预期：
  - 重命名仍可用。
  - 点击文件仍能正常打开。
