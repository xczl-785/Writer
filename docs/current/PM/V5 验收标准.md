## 一、总体验收标准

### 1.1 优先级通过率要求

| 优先级 | 通过率要求 | 说明                               |
| ------ | ---------- | ---------------------------------- |
| **P0** | 100%       | 所有 P0 功能必须完全实现并通过验收 |
| **P1** | ≥ 90%      | 允许少量非关键验收点未满足         |
| **P2** | ≥ 80%      | 核心功能可用即可                   |

### 1.2 质量底线

| 指标                 | 目标值           | 测量方法                      |
| -------------------- | ---------------- | ----------------------------- |
| **P0/P1 Bug 数**     | 0                | 验收测试用例执行结果          |
| **交互一致性**       | ≥ 90 分          | 一致性审查清单评分            |
| **快捷键冲突率**     | 0                | 快捷键映射表审查              |
| **破坏性操作覆盖率** | 100%             | 删除/覆盖类操作确认审查       |
| **错误信息可读性**   | ≥ 80 分          | 错误信息包含原因 + 建议的比例 |
| **性能无感知**       | 操作响应 < 100ms | 关键交互性能测试              |

## 五、验收流程

### 5.1 验收顺序

```
1. P0 功能验收 → 2. P1 功能验收 → 3. P2 功能验收 → 4. 一致性审查 → 5. 性能测试
```

### 5.2 验收不通过处理

| 问题类型             | 处理方式           |
| -------------------- | ------------------ |
| P0 功能未实现        | 阻塞验收，必须完成 |
| P1 功能未通过 ≥ 2 项 | 评估后决定是否延期 |
| 一致性审查 < 90 分   | 返工整改           |
| 性能测试不达标       | 优化后重新测试     |
