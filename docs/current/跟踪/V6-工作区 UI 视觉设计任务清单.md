## 1. 任务概述

### 1.1 设计背景

V6 工作区功能需要新增 6 个 UI 组件的视觉设计。本文档列出所有需要设计的组件、设计要求和交付物清单。

### 1.2 设计原则

| 原则   | 含义                 | 设计指导                           |
| ------ | -------------------- | ---------------------------------- |
| 沉静   | 不炫技、不打扰       | 使用低饱和度颜色，避免高对比度反馈 |
| 专注   | 让内容成为焦点       | UI 元素简洁，减少视觉噪音          |
| 可信赖 | 行为可预测           | 与 macOS/Windows 平台惯例一致      |
| 无打断 | Non-Disruption First | 优先使用非模态通知                 |

### 1.3 设计工具

**推荐工具**: Figma（首选）、Sketch、Adobe XD

**设计系统**: 如项目已有设计系统，请对齐现有令牌；如无，使用本清单定义的临时令牌。

## 2. 设计任务清单

### 2.1 EmptyStateWorkspace（空状态界面）

**优先级**: P0  
**设计复杂度**: ⭐⭐⭐⭐  
**使用场景**: 用户启动 Writer 但未打开任何工作区时显示

#### 设计内容

| 元素                 | 设计要求                             | 备注                         |
| -------------------- | ------------------------------------ | ---------------------------- |
| Logo/字标            | 居中，距顶部 80px                    | 使用项目现有 Logo            |
| 标语                 | "写作，心流"或"打开文件夹，开始写作" | 字号小于 Logo，颜色锌灰 -400 |
| 主按钮（打开文件夹） | 主按钮样式，最小宽度 140px           | 高优先级操作                 |
| 次按钮（打开工作区） | 次按钮样式，最小宽度 140px           | 可弱化视觉权重               |
| 分隔线               | 1px solid zinc-200，上下留白 24px    | -                            |
| 最近项目列表         | 最多 10 项，垂直排列，间距 8px       | 分组：工作区/文件夹/文件     |
| 拖拽提示             | 斜体，zinc-400 色                    | "或拖拽文件夹到此处"         |

#### 交互状态

| 状态       | 设计说明                                 |
| ---------- | ---------------------------------------- |
| 默认       | 上述布局                                 |
| 拖拽悬停   | 边框 1px solid zinc-300 + 背景 zinc-50   |
| 拖拽无效   | 红色边框 + 禁止图标 + 提示"仅支持文件夹" |
| 空历史记录 | 隐藏最近项目区域，仅显示主按钮           |

#### 响应式设计

| 断点         | 设计调整                              |
| ------------ | ------------------------------------- |
| 宽度 < 600px | 按钮垂直排列                          |
| 宽度 < 400px | 隐藏标语，仅保留 Logo 和按钮          |
| 高度 < 500px | 减少顶部留白，最近项目列表限制为 5 项 |

#### 交付物

- [ ] Figma 设计稿（默认状态）
- [ ] 拖拽悬停状态设计稿
- [ ] 响应式布局说明（<600px、<400px）
- [ ] 深色模式设计稿（可选，V6.1）

### 2.2 MultiRootFileTree（多根文件树容器）

**优先级**: P0  
**设计复杂度**: ⭐⭐⭐  
**使用场景**: 侧边栏文件树，支持多个根文件夹同时显示

#### 设计内容

| 元素           | 设计要求                                       | 备注                |
| -------------- | ---------------------------------------------- | ------------------- |
| 根文件夹头部   | 左对齐文件夹图标 + 显示名，右对齐展开/折叠按钮 | 悬停时背景 zinc-100 |
| 根文件夹分隔线 | 1px solid zinc-200                             | 分隔不同根文件夹    |
| 缩进层级       | 根文件夹 0px，一级子项 16px，每层+16px         | -                   |
| 展开/折叠按钮  | 12x12，悬停显示                                | 使用三角图标或 +/-  |

#### 交互状态

| 状态         | 设计说明                 |
| ------------ | ------------------------ |
| 根文件夹悬停 | 背景 zinc-100            |
| 展开状态     | 三角图标向下 ▼           |
| 折叠状态     | 三角图标向右 ▶           |
| 选中文件     | 背景 blue-50（浅色模式） |

| 工作区名称 | 次要色 zinc-500               | 与文件编码信息区分层级 |

| 分隔符     | 竖线 `│` 与状态栏其他信息分隔 | -                      |

| 状态           | 显示内容              | 颜色            |

| 多文件夹已保存 | 工作区名称 (工作区)   | zinc-500        |
| 无标题工作区   | 未命名工作区 (未保存) | zinc-400 + 斜体 |

### 2.5 RecentItemsMenu（最近项目菜单）

**优先级**: P1  
**设计复杂度**: ⭐⭐  
**使用场景**: `文件 > 打开最近` 子菜单

#### 设计内容

| 元素     | 设计要求                       | 备注                    |
| -------- | ------------------------------ | ----------------------- |
| 分组标题 | "工作区"、"文件夹"、"文件"     | 次要色 zinc-400，小字号 |
| 工作区项 | 💼 图标 + 名称 + (工作区) 后缀 | 最多 10 条              |
| 文件夹项 | 📁 图标 + 路径                 | 最多 5 条               |
| 文件项   | 📄 图标 + 路径                 | 最多 20 条              |
| 悬停状态 | 背景 zinc-100                  | -                       |
| 右键菜单 | "从最近移除"选项               | -                       |

#### 图标规范

| 类型   | 图标                  | 颜色     |
| ------ | --------------------- | -------- |
| 工作区 | 💼 或自定义公文包图标 | zinc-600 |
| 文件夹 | 📁 或自定义文件夹图标 | zinc-600 |
| 文件   | 📄 或自定义文档图标   | zinc-600 |

**注意**: 如使用 emoji，需确保跨平台一致性；否则使用项目图标系统。

### 2.6 FileConflictDialog（文件冲突对话框）

**优先级**: P1  
**设计复杂度**: ⭐⭐⭐  
**使用场景**: 外部文件被修改时提示用户选择操作

#### 设计内容

| 元素         | 设计要求                       | 备注               |
| ------------ | ------------------------------ | ------------------ |
| 对话框标题   | "文件已被外部修改"             | -                  |
| 文件信息     | 文件名 + 路径 + 修改时间对比   | 使用表格或列表布局 |
| 操作按钮     | "比较变更"、"重新加载"、"忽略" | 主按钮为"重新加载" |
| 不再询问选项 | 复选框"不再询问此类文件"       | 底部对齐           |

#### 布局示例

```
┌─────────────────────────────────────────┐
│                                         │
│  📄 document.md 已被外部修改             │
│                                         │
│  您的版本：2026-03-08 10:30 AM          │
│  外部版本：2026-03-08 11:45 AM          │
│                                         │
│     [比较变更]  [重新加载]  [忽略]       │
│                                         │
│  □ 不再询问此类文件                     │
│                                         │
└─────────────────────────────────────────┘
```

#### 交付物

- [ ] Figma 设计稿（对话框）
- [ ] 响应式布局说明（小屏幕适配）

## 4. 交付要求

### 4.1 设计稿格式

- **工具**: Figma（首选）、Sketch、Adobe XD
- **文件命名**: `V6-Workspace-{组件名}.fig`
- **画板命名**: `{组件名}-{状态}`（如 `EmptyState-Default`、`EmptyState-DragHover`）

### 4.2 设计说明

每个组件需附带：

1. **使用场景说明**: 何时显示该组件
2. **交互状态**: 默认、悬停、激活、禁用的视觉差异
3. **响应式规则**: 不同屏幕尺寸下的布局调整
4. **深色模式**（可选）: V6.1 前完成

### 4.3 交付时间

| 组件                | 优先级 | 建议完成时间 |
| ------------------- | ------ | ------------ |
| EmptyStateWorkspace | P0     | 1 天         |
| MultiRootFileTree   | P0     | 1 天         |
| WorkspaceRootHeader | P0     | 0.5 天       |
| WorkspaceIndicator  | P1     | 0.5 天       |
| RecentItemsMenu     | P1     | 0.5 天       |
| FileConflictDialog  | P1     | 1 天         |

**总计**: 约 4-5 天

## 5. 验收标准

设计完成后，需通过以下验收：

| 验收项     | 标准                                  |
| ---------- | ------------------------------------- |
| 视觉一致性 | 与现有 Writer UI 风格一致             |
| 平台惯例   | 符合 macOS/Windows 设计规范           |
| 可访问性   | 颜色对比度 ≥ 4.5:1（WCAG AA）         |
| 响应式     | 支持最小窗口尺寸（600x400）           |
| 深色模式   | 所有组件都有深色模式变体（可选 V6.1） |
