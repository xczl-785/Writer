## 2. Critical 问题汇总（必须修复）

### 🔴 CR-01: 拖拽交互与产品"沉静"定位冲突

**发现问题**: 交互规范第 2.3.1 节定义的外部文件拖拽反馈使用「蓝色边框 2px + 半透明蓝色背景」的高对比度视觉反馈。

**冲突点**:

- 蓝色高对比度边框过于醒目，打断用户注意力
- 半透明背景覆盖编辑区内容，破坏"专注写作"核心体验
- 与第 1.2 节"微妙的视觉反馈，避免炫目动画"原则直接矛盾

**必须修正**:

```
当前: 2px solid blue-500 + 半透明蓝色背景
建议: 1px solid zinc-300 + 轻微背景色变化 (zinc-50)
      或仅使用光标变化 + subtle tooltip 提示
```

---

### 🔴 CR-02: 多文件夹功能可发现性差（Critical UX 缺陷）

**发现问题**: 单文件夹模式下缺少添加文件夹的 UI 入口。

文档中定义的三条路径：

1. 菜单: `文件 > 添加文件夹到工作区` — 需探索菜单
2. 快捷键: `Cmd+K Cmd+A` — 进阶用户才使用
3. 右键文件树空白处 — **单文件夹模式下不显示此菜单项**

**严重性**: 用户必须知道功能存在才能使用，但没有任何自然的方式在 UI 上发现这个功能。

**必须修正**（二选一或同时实施）:

**方案 A（推荐）: 文件树头部常驻入口**

```
┌─────────────────────────────────┐
│ EXPLORER              [+] [≡]   │  ← 新增 [+] 按钮
├─────────────────────────────────┤
│ 📁 项目文档            [▼]      │
└─────────────────────────────────┘
[+] 按钮功能: 添加文件夹到工作区（始终可见）
```

**方案 B: 右键菜单始终显示**

```
右键文件树空白处（单/多文件夹模式都显示）:
┌────────────────────────────┐
│ 添加文件夹到工作区         │  ← 始终显示
│ ────────────────────────── │
│ 新建文件                   │
│ 新建文件夹                 │
└────────────────────────────┘
```

## 3. High/Medium 问题汇总

### 🟡 HIGH-01: 快捷键体系冲突

**问题**:

- `Cmd+W` 在 macOS 标准语义是"关闭当前窗口/标签页"，与"关闭工作区"概念不同
- `Cmd+R` 在 macOS 是刷新页面快捷键，系统级占用
- `Cmd+K Cmd+A` 双键序列过于复杂

**必须修正**:

- `Cmd+W`: 关闭当前文件/标签页（符合平台惯例）
- `Cmd+Shift+W`: 关闭工作区（与 VS Code 一致）
- `Cmd+Shift+A`: 添加文件夹到工作区
- `Cmd+Shift+O`: 打开最近文件

---

### 🟡 HIGH-02: 空状态规范不完整

**问题**:

1. **缺失视觉层级定义**: Logo 字号、按钮与最近项目区域权重关系未定义
2. **缺失响应式规范**: 窗口宽度<600px、高度不足时的布局调整
3. **图标使用不一致**: 使用 emoji 但未定义替代方案

**必须补充**:

- 窗口标题字号/按钮字号/标语字号比例
- 响应式布局断点（<600px、<400px）
- 统一图标系统（Lucide 或系统图标）

---

### 🟡 HIGH-03: 可访问性规范流于表面

**问题**:

1. **键盘导航不完整**: 文件树内文件项的键盘导航未定义
2. **ARIA 实现细节缺失**: 缺少 aria-level、aria-posinset 等关键属性
3. **焦点管理未定义**: 对话框关闭后焦点复位策略

**必须补充**:

- 完整的键盘导航表（包括 Esc 键行为）
- ARIA 完整属性清单
- 焦点管理流程图

---

### 🟡 MEDIUM-01: 通知系统缺乏优先级分层

**问题**: 仅定义了状态栏非模态通知和模态对话框两种形式，未定义何时使用哪种形式。

**建议分层**:

```
P0 - 状态栏被动提示: 操作成功、信息更新
P1 - 非模态 toast: 需要注意但不打断（保存提示）
P2 - 模态对话框: 需要用户决策（文件冲突、数据丢失风险）
```

---

### 🟡 MEDIUM-02: 边缘情况视觉处理不完整

| 边缘情况       | 当前状态         | 缺失内容                                 |
| -------------- | ---------------- | ---------------------------------------- |
| 文件夹权限不足 | 仅定义错误提示   | 错误提示的视觉样式（颜色、图标、位置）   |
| 工作区文件损坏 | 仅定义状态栏通知 | "创建新工作区"按钮的样式和位置           |
| 路径不存在     | 定义了灰显       | 具体色值、警告图标样式、tooltip 触发区域 |

**建议**: 补充边缘情况视觉规范，使用统一错误状态组件。

---

### 🟡 MEDIUM-03: 无标题工作区保存触发时机过晚

**问题**: 用户只能在关闭时被动收到保存提示，缺少主动保存的入口。

**建议**: 多文件夹模式下，在状态栏增加常驻的"保存工作区"入口：

```
状态栏: UTF-8 │ LF │ 未命名工作区 │ [保存]  │ 第 10 行，第 5 列
                                           ↑ 点击打开保存对话框
```

## 7. 审阅结论

### 7.1 整体评估

**两位设计师一致结论**: 🟡 **有条件通过**

文档框架完整，核心设计理念（无打断、渐进式披露）与产品人格基本对齐。但在可发现性、视觉克制、规范完整性上需要加强。

### 7.2 关键决策建议

| 决策         | 建议           | 理由                       |
| ------------ | -------------- | -------------------------- |
| 拖拽视觉反馈 | 降低侵略性     | 符合"沉静"产品定位         |
| 多文件夹入口 | 添加 [+] 按钮  | 解决 Critical 可发现性问题 |
| 快捷键体系   | Cmd+Shift 前缀 | 避免与系统冲突，符合惯例   |
| 空状态规范   | 补充响应式设计 | 确保开发准确还原           |
| 可访问性     | 完整键盘导航   | 满足 WCAG 2.1 AA 标准      |

### 7.3 进入 UI 开发的前提条件

**必须完成**（3 项）:

1. [ ] 修正拖拽视觉反馈（1px zinc-300 边框）
2. [ ] 添加多文件夹功能 UI 入口（文件树头部 [+] 按钮）
3. [ ] 修正快捷键体系（Cmd+Shift 前缀）

**建议完成**（3 项）:

1. [ ] 补充空状态视觉层级和响应式规范
2. [ ] 补充完整键盘导航和 ARIA 属性
3. [ ] 定义通知优先级分层体系
