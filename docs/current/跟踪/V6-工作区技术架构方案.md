## 2. 架构决策

### 2.1 核心架构决策

| 决策       | 方案                        | 理由                                 |
| ---------- | --------------------------- | ------------------------------------ |
| 状态模型   | 单一 `folders[]` 数组       | 避免 `currentPath` 和 `folders` 冗余 |
| 工作区类型 | 计算属性，不存储            | 从 `folders.length` 推导             |
| 文件树结构 | 多根节点 `RootFolderNode[]` | 支持多个独立根文件夹                 |
| 事务协调   | `workspaceActions` 统一协调 | 避免多 store 竞态条件                |
| 后端命令   | 批量操作优先                | 减少 IPC 往返次数                    |

### 2.2 架构变更摘要

| 模块                       | 变更类型 | 变更描述                             |
| -------------------------- | -------- | ------------------------------------ |
| `workspaceSlice`           | 重构     | 移除 `currentPath`，使用 `folders[]` |
| `filetreeSlice`            | 扩展     | 添加 `rootFolders[]` 支持多根        |
| `workspaceActions`         | 新建     | 多 store 事务协调器                  |
| `FsService`                | 扩展     | 添加批量命令和工作区命令             |
| `WorkspaceMetadataService` | 新建     | 工作区元数据管理                     |

  index: number; // 排序索引

// 计算属性（通过 selector）
export const getWorkspaceType = (state: WorkspaceState): WorkspaceType => {
  if (state.folders.length === 0) return 'empty';
  if (state.folders.length === 1) return 'single';
  return 'multi';
};

export const isUntitledWorkspace = (state: WorkspaceState): boolean => {
  return state.folders.length > 1 && state.workspaceFile === null;
};

## 4. 后端命令设计

### 4.1 新增 Tauri 命令

```rust
// src-tauri/src/fs.rs

/// 批量加载多个文件夹的树
#[tauri::command]
pub fn list_tree_batch(paths: Vec<String>) -> Result<Vec<FolderPathResult>, String> {
    paths.into_iter()
        .map(|path| {
            list_tree(&path).map(|nodes| FolderPathResult {
                path,
                nodes,
                error: None,
            }).unwrap_or_else(|e| FolderPathResult {
                path,
                nodes: vec![],
                error: Some(e),
            })
        })
        .collect()
}

/// 解析工作区文件
#[tauri::command]
pub fn parse_workspace_file(path: &str) -> Result<WorkspaceConfig, String> {
    let content = fs::read_to_string(path).map_err(|e| e.to_string())?;
    let config: WorkspaceConfig = serde_json::from_str(&content)
        .map_err(|e| format!("Failed to parse workspace file: {}", e))?;
    Ok(config)
}

/// 保存工作区文件
#[tauri::command]
pub fn save_workspace_file(path: &str, config: WorkspaceConfig) -> Result<(), String> {
    let content = serde_json::to_string_pretty(&config)
        .map_err(|e| e.to_string())?;

    // 使用原子写入
    let temp_path = format!("{}.tmp", path);
    fs::write(&temp_path, &content).map_err(|e| e.to_string())?;
    fs::rename(&temp_path, path).map_err(|e| e.to_string())?;

    Ok(())
}

/// 解析相对路径为绝对路径
#[tauri::command]
pub fn resolve_relative_path(base_path: &str, relative_path: &str) -> Result<String, String> {
    let base = Path::new(base_path);
    let parent = base.parent().ok_or("Invalid base path")?;
    let resolved = parent.join(relative_path);
    let resolved = resolved.canonicalize()
        .map_err(|e| format!("Failed to resolve path: {}", e))?;
    Ok(resolved.to_string_lossy().replace('\\', "/"))
}
```

## 9. 迁移策略

### 9.1 状态迁移

```typescript
// V5 → V6 迁移

const migrateV5ToV6 = (persistedState: any, currentState: any) => {
  const version = persistedState?.version || 0;

  if (version < 1) {
    // V5 使用 currentPath，V6 使用 folders[]
    const workspacePath = persistedState.currentPath;

    return {
      ...currentState,
      version: 1,
      folders: workspacePath
        ? [
            {
              path: workspacePath,
              index: 0,
            },
          ]
        : [],
      workspaceFile: null,
      isDirty: false,
      // V5 字段移除
      currentPath: undefined,
    };
  }

  return persistedState;
};
```

## 10. 风险与缓解

### 10.1 技术风险

| 风险               | 影响 | 可能性 | 缓解措施              |
| ------------------ | ---- | ------ | --------------------- |
| 状态不一致         | 高   | 中     | 事务协调器 + 严格测试 |
| 多根文件树渲染性能 | 中   | 中     | 虚拟滚动 + 懒加载     |
| 拖拽跨平台兼容性   | 中   | 中     | 降级到对话框          |
| 后端命令性能       | 中   | 低     | 批量操作 + 缓存       |

### 10.2 缓解措施详情

**状态不一致**:

- 使用 `workspaceActions` 统一协调
- 所有多 store 更新在一个 async 函数中完成
- 添加状态验证断言

**渲染性能**:

- 实现虚拟滚动（`react-window`）
- 懒加载子树（按需加载）
- 限制单次渲染节点数（<1000）
