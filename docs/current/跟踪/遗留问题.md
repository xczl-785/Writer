# 遗留问题

> 用于记录后续待推进的需求项（非 Bug）。

## 当前遗留

1. 文档加载性能待优化

- 现状：当前打开文档时，对比其他同类软件，加载速度稍慢
- 缺口：性能优化未专项推进
- 建议后续：专项调研瓶颈点并优化

2. 结构性感知待优化

- 现状：参考 ./design-block-indicator-system.md
- 缺口：设计存在4个阻塞项未解决
- 建议后续：先补齐阻塞项再进入开发

3. Slash Menu中，部分功能缺失，只有入口

- 现状：菜单项存在但功能未实现
- 缺口：产生用户困惑
- 建议后续：补齐功能或隐藏入口

4. 拖拽场景边界待明确

- 现状：当前拖拽功能已实现文件/文件夹区分显示不同提示，但多选拖入场景行为待确认。
- 缺口：
  - 混合拖入（文件+文件夹）：当前优先处理文件，忽略文件夹，需确认期望行为（阻止/分别处理/其他）。
  - 多文件拖入：当前只处理第一个文件，需确认是否支持批量打开（多标签页）。
- 建议后续：明确产品期望后，补充相应的交互设计和实现。

5. 重命名根文件夹的行为梳理
- 现状：当前根文件夹并不支持真实重命名，历史上的“重命名显示名”仅修改工作区中的显示别名，不会修改磁盘目录名。
- 缺口：产品语义与用户直觉不一致，且真实重命名会联动根路径、已打开文件、选中态、文件树根节点、监听器与工作区持久化状态，边界尚未梳理清楚。
- 建议后续：先明确目标是“仅改显示名”还是“真实重命名目录”，再分别补齐交互、状态迁移、异常回滚与测试矩阵。

6. 新建流程的 UI / 逻辑分层收敛
- 现状：新建文件 / 文件夹的目标解析已部分下沉到 `domains/workspace/services/createEntryTarget.ts`，但创建前展开目录、ghost 挂载、入口编排等逻辑仍停留在 `ui/sidebar/Sidebar.tsx`。
- 缺口：当前仍存在 UI 展示与交互编排耦合，排查问题时需要直接进入 Sidebar 组件加日志，说明分层尚未彻底收敛，不利于后续维护与测试。
- 建议后续：将新建流程的交互编排进一步抽离为独立 controller / hook / application service，使 Sidebar 仅负责事件绑定与状态渲染，减少组件内业务判断。
