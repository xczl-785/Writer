# Sprint 2.5 Tasks（交互体验纠偏）

## 1. Sprint 目标

在 Sprint 2 已有功能基础上，完成文件树交互体验纠偏：
- 对齐 VS Code 风格的操作行为（新建/重命名/删除）
- 对齐平台快捷键习惯（Win/macOS）
- 收口已知非阻断体验缺陷，形成可回归规范

## 2. 范围

### In Scope（本期承诺）

1. 文件树交互一致性
- 新建、重命名、删除流程标准化（输入态/确认态/取消态）
- 选中态与焦点连续性优化

2. 快捷键体系化
- 从“组件内硬编码按键”升级到“命令 + 平台映射 + 条件触发”
- 覆盖重命名、删除、新建等核心操作

3. 显示策略一致性
- 仅显示 markdown 文件
- 文件后缀隐藏显示一致（树节点 + 重命名输入）

4. 错误与反馈体验
- 非法命名、重名、权限错误具备明确反馈
- 删除确认策略可扩展到“不再提示”配置

### Out of Scope（本期不承诺）

1. 文件拖拽移动
2. 多选与批量操作
3. 文件树全文搜索
4. 多工作区

## 5. 风险与应对

1. 风险：快捷键与编辑器现有快捷键冲突
- 应对：通过 `when` 条件限制为 Explorer 焦点生效，编辑区快捷键优先级独立。

2. 风险：平台键位习惯不一致引发学习成本
- 应对：按平台分层映射，保留一套 fallback 键位并文档化。

3. 风险：交互调整引入状态回归（active/openFiles）
- 应对：所有命令统一走 dispatcher + store 原子更新，并补充回归用例。
