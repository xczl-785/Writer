# QA 验收清单 Sprint 2.5（交互体验纠偏）

## 1. 目的

验证 Sprint 2.5 的交互规范落地质量，确保文件树在“操作行为、键盘模型、错误反馈”上达到一致性。

## 2. 范围

- 新建/重命名/删除交互一致性
- Win/macOS 快捷键一致性（平台差异映射）
- 焦点与选中流
- 负向路径与错误提示
- markdown 可见性与后缀展示策略

## 3. 回归前置

- 准备 Windows + macOS 环境各一套。
- 使用同一测试目录结构（含 markdown 与非 markdown 文件）。
- 确保目录有读写权限，并准备权限受限目录用于负向测试。
