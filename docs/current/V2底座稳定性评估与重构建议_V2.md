# V2 底座稳定性评估与重构建议

## 1. 背景与范围

在进入 V2 开发阶段前，对 V1 遗留的底座代码进行全面质量评估。本次评估旨在识别影响系统稳定性、可维护性及扩展性的核心问题，并给出明确的重构建议，确保 V2 开发在稳固的底座上进行。

评估范围包括：

- `src/` 核心逻辑与 UI 组件
- `src-tauri/` 后端 Rust 代码
- 工程配置（Lint/Build/Test）
- 冗余资源与目录结构

## 2. 评估方法与证据

通过以下工具与方法获取客观状态：

- **静态分析**: `npm run lint` (ESLint)
- **类型检查**: `npx tsc --noEmit`
- **构建验证**: `npm run build` (Vite + TSC)
- **自动化测试**: `npm run test` (Vitest)
- **人工审查**: 针对已知基线问题进行代码走读

## 3. 必须修复 (Must Fix)

此类问题直接影响开发流程或系统稳定性，必须在进入 V2 功能开发前解决。

此类问题影响长期可维护性，建议在 V2 初期或并行处理。

| 问题描述                 | 影响                       | 证据路径                                 | 建议动作                             | 优先级 |

## 5. 正向结论 (Keep)

- **测试覆盖**: 核心服务（Markdown, Workspace）测试通过率 100% (66/66)。
- **类型系统**: 核心业务逻辑类型定义较完备，但当前构建仍受测试文件类型声明问题阻塞（见附录 Build 输出）。
- **模块化**: 基础服务层（Services）职责划分清晰。

## 6. 建议执行顺序

1. **进入 V2 前 (Gate A)**:
   - 修复 Lint 错误，确保 `npm run lint` 零报错。
   - 修复 Build 错误，确保 `npm run build` 成功。
   - 移除 Rust 冗余代码。
   - 定义 StatusBar 缺失的 CSS 变量。
2. **进入 V2 后 (Sprint 1)**:
   - 重构 Sidebar 大文件。
   - 清理空目录与冗余状态。
