# V2 启动看板

## 1. 启动目标

- 目标：在进入 V2 开发前，完成范围、门禁、文档、验证口径冻结。
- 原则：先定义、再开发；先可验证、再承诺。

## 2. V1 当前状态（进入 V2 前提）

- V1 需求追溯矩阵状态：`PENDING_WINDOWS_TEST`（已更新）。
- Windows 平台验证：当前无环境，统一待补测。
- 结论：允许并行开展 V2 规划文档，不进入 V2 功能开发。

## 4. 执行顺序与 Gate

1. Gate A：V2-01~V2-04（范围+标准+路线+PRD）
2. Gate B：V2-05~V2-07（技术/设计/工程）
3. Gate C：V2-08~V2-10（追溯+任务+QA）

进入开发前必须满足：

- [ ] V2 文档集完成评审
- [ ] V2 Must 范围冻结
- [ ] V2 QA 门禁可执行

## 5. 风险

1. V1 Windows 验证缺失可能影响 V2 节奏评估。
2. V2 目标过多会导致范围膨胀。
3. 表格可视化编辑若缺乏交互细则，开发返工风险高。
