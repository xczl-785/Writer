## 4. 执行顺序与 Gate

1. Gate A：V2-01~V2-02（需求去重 + 底座稳定性评估）
2. Gate B：V2-03~V2-06（范围+标准+路线+PRD）
3. Gate C：V2-07~V2-10（技术/设计/工程+追溯）
4. Gate D：V2-11~V2-12（任务+QA）

进入开发前必须满足：

- [x] V2 文档集完成评审（除稳定性评估项）
- [x] V2 Must 范围冻结
- [x] V2 QA 门禁可执行

## 5. 风险

1. V1 Windows 验证缺失可能影响 V2 节奏评估。
2. V2 目标过多会导致范围膨胀。
3. 表格可视化编辑若缺乏交互细则，开发返工风险高。
