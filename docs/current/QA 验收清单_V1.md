## 5. 核心回归用例

1. 用例 R1：最小写作闭环

- 步骤：打开目录 -> 打开文件 -> 连续输入 30 秒 -> 等待自动保存 -> 关闭应用 -> 重开检查。
- 预期：内容完整，无丢字、无回退。

2. 用例 R2：文件切换保护

- 步骤：在文件 A 修改未手动保存 -> 立即切到文件 B -> 再切回 A。
- 预期：A 的修改已落盘或明确失败提示，不可静默丢失。

3. 用例 R3：异常可观测

- 步骤：制造写入失败场景（只读目录或权限受限）并尝试保存。
- 预期：UI 显示明确错误，状态从 saving 进入 error，不出现“假成功”。

4. 用例 R4：冷/热启动可写时延

- 步骤：按 `docs/frozen/PRODUCT_CONSTRAINTS.md` 规则各测 5 次，记录中位数与 P95。
- 预期：
  - 冷启动 TTC（可输入）P95 <= 1800ms（目标 <= 1200ms）
  - 热启动 TTC（可输入）P95 <= 900ms（目标 <= 600ms）

5. 用例 R5：输入流畅性

- 步骤：在 30-100KB 文档连续输入 60 秒，统计丢字与明显卡顿。
- 预期：
  - 无丢字
  - 按键到渲染响应 P95 <= 33ms（目标 <= 16ms）

6. 用例 R6：原子写入不损坏原文件

- 步骤：对目标文件计算 hash，模拟写入中断（在 tmp 写入阶段失败），再检查原文件 hash。
- 预期：原文件 hash 不变；失败后有可见错误提示；不出现半写入文件覆盖原文件。

## 7. 通过门槛

1. QA-S1-01 ~ QA-S1-12 全部通过。
2. 核心回归用例 R1 ~ R6 全部通过。
3. NFR-01 ~ NFR-04 全部达标。
4. 无 P0 级缺陷（数据丢失、无反馈失败、无法启动）。
