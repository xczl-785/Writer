# REQ-SLASH-MENU-遗留专项需求（当前版）

## 1. 文档目标

本文件用于定义 Slash Menu 当前阶段的遗留需求基线，作为实现与验收唯一依据。  
阅读本文件即可完成需求理解、开发范围确认、测试回归与关闭判定，不依赖历史归档文档。

---

## 2. 背景与问题定义

Slash Menu 已具备基础能力（触发、搜索、键盘导航、命令执行），但在小视口与边界位置仍有两类关键遗留：

1. 向上翻转定位时，菜单会遮挡输入行（光标行）。
2. 菜单出现内部滚动时，键盘导航未稳定驱动可见区跟随，导致高亮项不可见。

这些问题会破坏“不中断写作心流”的交互目标，需作为同一专项收敛处理。

---

## 3. 需求范围

## 3.1 In Scope

1. 上翻定位与输入行安全间隔治理。
2. 键盘导航与菜单内部滚动联动治理。
3. 覆盖中文 IME、窗口尺寸变化、长列表场景的回归验证。

## 3.2 Out of Scope

1. Typewriter 模式算法调整。
2. Focus / Zen 模式退出优先级调整。
3. Slash 命令集合扩容（如新增命令类别）。

---

## 4. 规范基线（必须保持）

1. Slash 仅用于块级/插入型命令，不承接行内样式命令。
2. 触发边界保持严格模式：编辑器聚焦、空段落、折叠光标、非代码块。
3. 查询词不写入文档正文。
4. 菜单支持键盘与鼠标等价操作，`Esc` 可关闭。
5. 中文输入场景下不得出现明显抖动或状态错乱。

---

## 5. 遗留项与合并组

## 5.1 遗留项

1. `L-E-01 / TC-E02 / R-09`：菜单向上翻转时遮挡输入行。
2. `L-E-04 / TC-E09 / R-09`：向上翻转场景复现输入行遮挡（与 E02 同源）。
3. `L-E-02 / TC-E03 / R-09`：小窗内滚时，键盘选择项不跟随滚动。
4. `L-E-03 / TC-E05 / R-09`：菜单未完整显示时，`↑/↓` 不驱动可见区跟随。

## 5.2 合并处理组

1. `G-01`：`L-E-01 + L-E-04`（上翻定位与输入行安全间隔）
2. `G-02`：`L-E-02 + L-E-03`（键盘导航与滚动联动）

---

## 6. 行为验收标准

## 6.1 G-01 上翻定位

1. 当下方空间不足触发上翻时，菜单不得遮挡当前输入行。
2. 光标行与菜单边界之间需保持可感知安全间隔（实现可定义具体像素，但必须稳定一致）。
3. 在窗口缩放、容器滚动、长文档场景下结论一致。

## 6.2 G-02 键盘导航联动

1. 当高亮项移出菜单可见区时，菜单内部滚动必须自动跟随。
2. `ArrowUp/ArrowDown/Tab` 导航行为一致，循环导航时不出现“高亮丢失可见性”。
3. 鼠标悬停不应破坏后续键盘滚动联动。

## 6.3 通用稳定性

1. 中文 IME 输入期间，Slash 状态机不抖动、不跳态。
2. 菜单关闭后应清理临时状态，再次触发行为一致。
3. 不引入其它模块回归（Typewriter、Focus/Zen、编辑区三间距）。
