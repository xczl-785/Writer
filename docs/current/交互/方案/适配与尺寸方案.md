# Writer 窗口尺寸与适配设计方案

本文档定义了 Writer V5 在不同设备、屏幕分辨率及窗口尺寸下的视觉与交互适配标准。

## 设计核心：居中聚焦 + 极简禅模式

我们选择以 **“居中聚焦 (Centric Flow)”** 为核心布局逻辑，结合 **“极简禅模式 (Zen Mode)”** 的响应式降级，以确保“写作优先”的品牌体验。

---

## 1. 窗口尺寸定义 (Window Dimensions)

| 档位 | 建议尺寸 (px) | 侧边栏 (Sidebar) | 布局特征 |
| :--- | :--- | :--- | :--- |
| **最小尺寸 (Min)** | `640 x 480` | **初始收起** | 优先保证写作区。用户可手动展开，但会压缩正选列宽。 |
| **默认尺寸 (Default)** | `1180 x 800` | **标准显示 (240px)** | 黄金比例。720p (1280x720) 遵循此档位逻辑。 |
| **高分屏适配 (Hi-DPI)** | `2K / 4K / 5K` | **标准显示** | 增加左右负空间。 |

> [!NOTE]
> **字号一致性**：无论窗口尺寸如何变化，编辑器正文字号始终保持物理 **`16px`** 这一核心阅读基准。响应式调节仅针对侧边栏宽度、左右边衬（Margin）及顶底留白。

## 2. 编辑区视觉留白 (Vertical Rhythm)

*   **顶部留白 (Top Padding)**：面包屑 Header 与正文第一行之间预留视觉缓冲。**Default/Airy 档位为 64px**，**Min (Zen) 档位缩减为 32px**。
*   **底部留白 (Typewriter Padding)**：
    *   为保证光标不撞底，编辑区底部常驻 **`40vh`** 的空白滚动区域（用户可滚动文字至视口中央之后继续向上）。
*   **打字机模式 (Typewriter Mode)**：
    *   **定义**：一种可选的沉浸式写入模式。开启后，当前正在书写的行（光标行）始终锁定在视口垂直方向的 **`45%`** 处。随着翻行，文档内容会自动匀速向上滚动。
    *   **出发点**：保护颈椎，让视线始终保持在屏幕水平中心，避免因为文档变长而不得不低头盯着屏幕底部书写。

## 3. Slash Menu 边界处理 (Boundary Logic)

针对含有 14 个选项的高长菜单（预估高度 480px）：

*   **触发定位**：
    *   若光标位置距离视口底部空间 > 500px，菜单**向下**弹出。
    *   若光标位置距离视口底部空间 < 500px，菜单**向上**反转。
*   **极端情况保护**：若窗口高度总计不足 500px（如最小尺寸），Slash Menu 自动开启**内部滚动**，并限制最大高度为视口高度的 **`85%`**。

## 4. 高分屏适配优化 (High-DPI Refinement)

*   **线框精致度**：在高 DPI (> 2x) 下，所有分割线从 `1px` 切换为应用物理像素的 `0.5pt`，实现“若有若无”的效果。
*   **负空间布局**：最大文本列宽锁定在 **`850px`**，避免在大尺寸外显上产生过长的扫描行。

## 验证计划

### 自动化验证 (交互逻辑)
- 编写测试用例验证 Slash Menu 在窗口底部的反转坐标计算是否正确。
- 验证当窗口宽度变化至 `640px` 以下时，`collapsed-sidebar` 状态是否正确触发。

### 手动验证 (视觉审核)
- 在不同分辨率（Retina vs Standard）下对比分割线的视觉厚度。
- 拉伸窗口，观察编辑器左右留白的响应式平衡感。
- 确认 Slash Menu 在 14 个选项满载时，在窗口边缘的显示完整度。
