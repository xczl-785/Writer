# REQ-EDITOR-BOTTOM-GAP 底部间距需求

## 1. 目标

规范编辑区文末留白与状态栏的距离关系，避免光标或文末内容紧贴状态栏。

## 2. 需求来源

1. `docs/current/交互/方案/UI设计详细交付文档.md` §4.1
2. `docs/current/交互/方案/适配与尺寸方案.md` §2
3. `docs/current/交互/方案/尺寸清单.md` §2
4. `docs/current/交互/V5交互测试用例与需求映射.md` R-02, TC-A08, TC-D10
5. `docs/current/交互/V5交互测试遗留清单.md` L-A-01, G-04

## 3. 范围

1. 正文区域底部滚动留白。
2. 文末输入、滚动到最底、打字机开启/关闭场景。
3. 与状态栏视觉距离约束。

## 4. 规则

1. 文末常驻底部留白基线为 `40vh`（需求原文定义）。
2. 在任意档位滚动到文末时，文末输入行不得与状态栏直接贴边。
3. 打字机开启时，底部留白必须满足锚点滚动所需余量，不能因余量不足导致光标贴底。
4. Focus Zen 仅影响状态栏显隐，不改变底部留白策略本身。

## 5. 验收标准

1. `TC-A08`：任意档位滚动到文末，仍有约 `40vh` 可感知留白。
2. `TC-D10`：长文文末持续输入，光标不贴底且底部留白持续有效。
3. 对应遗留项 `L-A-01` 必须关闭后方可判定通过。

## 6. 非目标

1. 不定义状态栏内部样式（颜色、动画、文本）。
2. 不定义 Slash Menu 在底部区域的翻转策略。
