# REQ-EDITOR-INLINE-GAP 左右间距需求

## 1. 目标

规范正文左右内边距与最大正文列宽，确保在不同窗口档位下保持可读性与视觉聚焦。

## 2. 需求来源

1. `docs/current/交互/方案/UI设计详细交付文档.md` §2.1, §2.2
2. `docs/current/交互/方案/尺寸清单.md` §2
3. `docs/current/交互/V5交互测试用例与需求映射.md` R-01, R-02, TC-A04, TC-A05, TC-A06

## 3. 范围

1. 正文容器左右内边距。
2. 正文最大列宽与居中策略。
3. Min / Default / Airy 三档位。

## 4. 规则

1. `Min (W <= 640)`：左右内边距为 `16px`。
2. `Default/Airy (W > 640)`：左右内边距为 `32px`。
3. 正文最大宽度固定 `850px`，超出空间以 `margin: 0 auto` 居中分配。
4. 侧边栏 Overlay 展开时，不得挤压正文宽度（仅覆盖）。

## 5. 验收标准

1. `TC-A04`：Min 档左右约 `16px`。
2. `TC-A05`：Default/Airy 档左右约 `32px`。
3. `TC-A06`：高宽场景正文列宽不超过 `850px` 且保持居中。
4. `TC-B03`：Min 档 Overlay 展开不改变正文计算宽度。

## 6. 非目标

1. 不定义侧边栏自身宽度（例如 240px）。
2. 不定义字体与行高参数。
