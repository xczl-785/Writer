# 打字机模式 (Typewriter) 交互规格说明 v4.0

> **状态**：待评审 (Ready for Dev)  
> **更新日期**：2026-03-07  
> **设计目标**：构建基于状态机的打字机体验，消除强制居中导致的视觉跳跃，实现平滑的视线锁定。

---

## 1. 核心机制

摒弃传统的“绝对居中”逻辑，采用 **“软阈值触发 + 动态锚点锁定”** 机制：
- **触发水位线 (Threshold)**：视口高度的 45% 仅作为触发锁定的阈值，而非固定坐标。
- **动态锚点 (Dynamic Anchor)**：光标以任意方向自然触碰或越过 45% 水位线时，系统记录当前真实的物理坐标比例（如 45.5%），作为后续视图滚动的唯一的绝对基准。此举可消除因行高差异导致的像素级抖动。

---

## 2. 状态机与生命周期

打字机模式开启后，系统在以下两种状态间流转：

### 2.1 自由态 (Free Mode)
- **定义**：视口静止，光标自由移动，等同于常规编辑器。
- **进入条件**：
  1. 初始状态（光标在 45% 水位线以上）。
  2. 用户使用 **鼠标点击** 定位到文档任意位置（包含 45% 以下区域）。
- **交互表现**：
  - 输入、换行、方向键（`ArrowUp/Down/Left/Right`）均表现为光标的自然物理位移。
  - 在视口底部按常规逻辑触发原生边缘滚动（Edge Scrolling），不强制拉回 45% 水位线。

### 2.2 锁定态 (Locked Mode)
- **定义**：光标垂直坐标锁定，视口反向滚动补偿。
- **进入条件（双向捕获）**：
  - 在“自由态”下，通过输入、换行或方向键，光标**首次触碰或越过 45% 水位线**（无论自上而下，或自下而上）。
- **交互表现**：
  - 光标的屏幕垂直绝对坐标保持恒定。
  - 所有输入、换行及 **方向键导航**，均转化为背景文档的上下平滑滚动（卷轴效果）。

---

## 3. 行为判定矩阵

开发需严格依据下表执行状态流转与视图补偿：

| 操作指令 | 当前状态 | 目标视图行为 (System Response) | 后续状态流转 |
| :--- | :--- | :--- | :--- |
| **鼠标点击** | 任意状态 | 视口绝对静止 | 强制进入 **自由态** |
| **输入 / 换行 / 方向键** | 自由态 | 光标自然位移。若位移落点未触及 45% 水位线，视口静止。 | 若触及/越过 45%，进入 **锁定态** (并记录动态锚点) |
| **输入 / 换行 / 方向键** | 锁定态 | 光标垂直坐标静止，视口执行反向滚动补偿。 | 维持 **锁定态** |

---

## 4. 算法与边界处理

为兼容软换行、多态行高（如代码块/标题）及 IME 场景，严禁使用 `LineHeight` 累加策略。

### 4.1 动态坐标补偿算法
1. **坐标采集**：切入或维持“锁定态”响应时，通过 DOM 获取光标实体相对视口顶部的物理 Y 坐标 (`caretRect.top`)。
2. **偏差计算**：`Delta = caretRect.top - DynamicAnchor_Y`（`DynamicAnchor_Y` 为切入锁定态时记录的锚点物理坐标）。
3. **视图滚动**：`TargetScrollTop = CurrentScrollTop + Delta`。
4. **动画参数**：建议过渡时间 ≤ `150ms`，采用平滑缓动曲线。

### 4.2 特殊边界容错
- **IME 输入法**：组合输入期间（如拼音选词），补偿算法须在 `compositionend` 事件内增加一次闭环校准，防止候选浮层干扰高度测量。
- **短内容保护**：若文档总高度不足以向下滚动至 45% 水位线，主动屏蔽锁定态逻辑，降级为常规编辑模式。
- **配置精简**：废除并移除“始终将光标保持在屏幕中间”等冗余子级开关，维持底层交互协议的单一性。

---

## 5. 验收基准 (Acceptance Criteria)

- [ ] **TC-1. 阈值触发与双向捕获**：从页首输入至 45%，或从页尾使用方向键上移至 45%，均能无缝切入“锁定态”。
- [ ] **TC-2. 方向键卷轴表现**：处于“锁定态”时，长按上下方向键导航，光标垂直位置锁定，文档如卷轴般顺滑上下滚动。
- [ ] **TC-3. 鼠标寻址防跳跃**：处于“锁定态”时，使用鼠标点击文档任意远离 45% 的行落点，视口保持绝对静止，强制退回“自由态”。
- [ ] **TC-4. 设置项收敛**：确认系统设置内仅保留“打字机模式”总开关，无其他表现形式的干预项。
