# 产品需求文档 V2（草案）

## 1. 目标

在保持 V1 写作体验稳定的前提下，补齐结构化编辑与信息组织能力。

## 2. V2 核心场景

1. 在文档中可视化编辑表格并正确保存。
2. 在工作区内快速定位目标文件。
3. 通过顶部快捷菜单快速完成常用样式与块级操作。

## 3. 功能需求（V2 Must）

1. FR2-01：表格可视化编辑
2. FR2-02：快速文件搜索
3. FR2-03：查找/替换
4. FR2-04：顶部快捷菜单（表格、代码块、加粗等）

## 3.1 需求规模策略（4+2）

为避免 V2 需求量偏少或范围膨胀，采用“4 个功能主线 + 2 个质量保障项”策略。

### 功能主线（4）

1. 表格可视化编辑
2. 快速文件搜索
3. 查找/替换
4. 顶部快捷菜单

### 质量保障项（2）

1. V1 核心链路回归不破坏（编辑、文件树、自动保存、图片）
2. 稳定性门禁通过（测试/构建/关键异常可见）

## 3.2 顶部快捷菜单（MVP）

### 覆盖范围

1. 文本样式：加粗、斜体、行内代码
2. 块级操作：H1/H2/H3、无序列表、有序列表、引用、代码块
3. 插入能力：表格（默认 3x3）

### 交互要求

1. 按钮行为与快捷键行为一致（同一命令语义）
2. 光标所在位置可反映按钮激活态
3. 不可执行时提供禁用态

### 非目标（本阶段不做）

1. 工具栏自定义配置
2. 复杂弹层式表格面板
3. 主题级大改造

## 3.3 V1 继承能力（V2 仅回归，不重复开发）

1. 应用内新建文件夹

## 4. 验收原则

1. 每个 FR 必须具备可执行 QA 用例。
2. 每个 FR 必须可关联证据路径。
3. 不得破坏 V1 Must 12 的回归通过状态。

## 5. FR 细化（用于 Sprint 拆分）

### FR2-01 表格可视化编辑

- 用户场景：写作者以可视化方式插入并编辑表格，不依赖手写 Markdown 表格语法。
- 交互规则：
  1. 通过顶部菜单插入默认 3x3 表格。
  2. 支持基础单元格导航与编辑。
- 验收标准：
  1. 插入后立即可编辑。
  2. 保存并重开后语义不丢失。

### FR2-02 快速文件搜索

- 用户场景：在工作区内快速定位目标文件。
- 交互规则：
  1. 提供搜索入口并支持键盘连续输入。
  2. 结果列表可直接打开文件。
- 验收标准：
  1. 搜索结果与文件名/内容匹配。
  2. 切换文件不丢失当前未保存编辑状态。

### FR2-03 查找/替换

- 用户场景：在当前文档中高频查词并批量替换。
- 交互规则：
  1. 提供查找与替换输入框。
  2. 支持下一处/上一处定位与单次替换。
- 验收标准：
  1. 定位结果准确。
  2. 替换后撤销/重做链路正常。

### FR2-04 顶部快捷菜单

- 用户场景：不使用快捷键时，用户可通过按钮完成常用格式操作。
- 交互规则：
  1. 菜单按钮覆盖加粗、斜体、行内代码、标题、列表、引用、代码块、表格插入。
  2. 菜单按钮状态与光标语义一致（激活态/禁用态）。
- 验收标准：
  1. 点击按钮与快捷键行为一致。
  2. 不影响现有输入流畅性和自动保存链路。
