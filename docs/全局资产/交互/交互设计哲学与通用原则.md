# 交互设计哲学与通用原则

**文档类型**: 全局资产（长期生效）
**创建日期**: 2026-03-03
**适用范围**: Writer 所有后续版本

---

## 1. 设计核心：极致克制 (Maximum Restraint)

Writer 的设计灵魂在于 **“只在需要时出现”**。

- **隐形界面**：最好的界面是让用户忘记界面的存在。
- **干扰最小化**：除核心写作区外，所有 UI 元素应默认隐藏或极度弱化。
- **无打断流**：界面的反馈（如设置、通知）不应中断用户的“造物心流”。

## 2. 交互三层模型 (The Three-Tier Model)

所有功能交互应遵循由浅入深的原则：

1. **第一层：肌肉记忆 (Muscle Memory)**
   - 依赖标准 Markdown 语法和原生快捷键，无视觉占位。
2. **第二层：按需唤出 (Progressive Disclosure)**
   - 只有在选中文本或空行触发时，才会弹出针对性的“幽灵菜单”或“气泡菜单”。
3. **第三层：底部锚点 (Static Anchors)**
   - 全局状态和核心导航作为稳定的视觉锚点，保持低对比度显示。

## 3. 跨版本资产标准

- **中性美学 (Neutral Aesthetics)**：不追逐流行色，以灰阶 (Zinc/Slate) 为主，配合极简的重点行动色。
- **OS 原生感**：菜单结构、快捷键逻辑应尽可能对齐目标操作系统的原生习惯，降低用户的认知负担。
- **呼吸感 (Breathing Space)**：保持慷慨的留白，不仅是美感需求，更是为了让内容真正占据视觉中心。
