# Writer V5 组件设计字典 (Design Tokens)

## 1. 核心调色板 (Core Palette)

| 名称 | Token | 值 (Hex/HSL) | 应用场景 |
|---|---|---|---|
| **Background** | `--bg-primary` | `#FFFFFF` | 主编辑区、全局背景 |
| **Sidebar BG** | `--bg-sidebar` | `#F9FAFB` | Explorer 侧边栏背景 |
| **Border** | `--border-main` | `#E4E4E7` | 极细分割线。Retina 屏建议使用 0.5px。 |
| **Primary** | `--color-action` | `#3B82F6` | 光标、重点选中态、确定按钮 |
| **Text Main** | `--text-primary` | `#18181B` | 正文、各级标题 |
| **Text Weak** | `--text-tertiary`| `#9CA3AF` | 幽灵提示词、面包屑辅助路径 |
| **Popover BG** | `--bg-popover` | `#18181B` | Bubble Menu/Slash 菜单 (黑底方案) |

## 2. 基础组件物理形态规约

### 2.1 形状与圆角 (Radius)
*   **交互弹出层 (Dialog/Popover)**: `12px` (极致圆润)
*   **输入框/按钮**: `6px` (适中克制)
*   **侧边栏选中指示线 (Indicator Bar)**: `w-[3px] bg-blue-500 rounded-r-full`。此线具有 **最高持久等级**，标识当前已打开并正在编辑的文件。无论用户点击了哪个其他项，此蓝线条 **始终保持** 在对应的编辑文件左侧。
*   **侧边栏背景焦点 (Background Focus)**: 标识用户当前最后一次点击产生的“管理焦点”。
    *   **活跃文件**: 正在编辑且选中的文件，呈现 `bg-blue-50/50`。
    *   **普通选中项**: 点击了非活跃文件或文件夹，呈现 `bg-zinc-200/50` 背景，原活跃文件的背景色应随之消失。



### 2.2 深度与阴影 (Shadows)
*   **Floating Card (大纲窗口)**: `0 12px 40px rgba(0, 0, 0, 0.15)`
*   **Bubble Menu**: `0 4px 6px -1px rgba(0, 0, 0, 0.2)` (更紧凑的黑色投影)
*   **Slash Menu (唤起菜单)**: `shadow-2xl` (黑色阴影加深)。高度超出 320px 时需包含内部滚动。

### 2.3 状态指示灯 (Status Dot)
*   **Unsaved**: `bg-amber-400` / `opacity-100` (常亮)
*   **Saving**: `bg-blue-500` / `animate-pulse` (2s 周期)
*   **Saved**: `bg-green-500` / `opacity-30` (暗淡暗示)
*   **Error**: `bg-red-500` / `animate-bounce` (高频弹跳)

### 2.4 专注模式切换器 (Sidebar Toggle - Collapsed Context)
*   **位置**:面包屑左侧，`mr-4` 间距。
*   **默认色值**: `text-zinc-300` (极淡，降低视觉干扰)。
*   **交互态**: `hover:text-zinc-800` / `hover:bg-zinc-100`。

## 3. 面包屑导航与块级边界 (Layout Details)

### 3.1 面包屑容器 (Breadcrumbs)
*   **高度**: `h-12` (顶栏 48px)
*   **分隔符**: `text-zinc-300 mx-2` (使用 `/` 或 `›`)
*   **悬停状态**: `hover:text-zinc-600 cursor-pointer`

### 3.2 块级边界装订线 (Gutter Lines)
*   **代码块 (Code Block)**: `光标进入时显示 w-1 bg-blue-400/50 rounded-l-sm`
*   **引用块 (Blockquote)**: `常驻 border-l-2 border-zinc-200`

## 4. 特殊交互组件 (Special Interactions)

### 4.1 Ghost Slash (幽灵斜杠提示)
*   **视觉特征**：`text-zinc-300`，字号 `0.875rem`，与光标保持 `2px` 的间距。
*   **交互动效**：用户一旦在该行输入任何非斜杠字符，提示文本 **立即 0ms 消失**。

### 4.2 Slash Menu (唤起菜单样式)
*   **背景**: `bg-zinc-900` (深色方案，与编辑主背景拉开显著差距)。
*   **圆角与内边距**: `12px` 圆角，`p-1` 容器内边距。
*   **列表项 (Item)**:
    *   **容器**: 左右布局，`py-1.5 px-2` 内边距，`rounded-md`。
    *   **字体**: 名称 `text-sm text-zinc-100`，快捷键 `text-[10px] text-zinc-500`。
    *   **悬停 (Hover)**: `bg-zinc-800` (亮灰)。
    *   **分组标题**: `text-[10px] text-zinc-500 font-bold tracking-widest uppercase py-1.5 px-2`。
    *   **边界保护**: 菜单检测视口底距。若底距 < 500px 则自动向上反向弹出。
    *   **窗口高度限制**: 窗口高度总计不足 500px 时，限制菜单最大高度为视窗 80% 并开启内部滚动。

### 4.3 Cursor Style (光标样式)
*   **宽度**: `2.5px` (稍微加粗，增加视觉存在感)。
*   **颜色**: `bg-blue-500` (#3B82F6)。
*   **动效**: `animate-pulse` (呼吸动效)，周期建议 **1.5s**。
*   **高度**: 匹配当前行文字高度 (1.2em)。

### 4.4 Text Selection (文本选区)
*   **背景色**: `bg-blue-200/50` (亮色) / `bg-blue-900/40` (暗色)。
*   **文字颜色**: 不改变原文字颜色（保持高对比度），或轻微加深。
*   **边角**: 选区端点建议保持直角或极小圆角 (2px)。

### 4.5 Git Sync Status (业务同步状态)
仅当项目关联 Git 仓库时载入此组件。
*   **Synced**: `text-green-500/50` / `bg-green-50/50`。本地版本与远程分支一致。
*   **Diverged (冲突/超前)**: `text-blue-500` / `bg-blue-50`。提示有待推送或待拉取的内容。
*   **No Git**: 彻底隐藏该组件，不占用底栏视觉。

### 4.6 Command Fragment (斜杠菜单搜索词视觉)
当用户在 Slash Menu 开启状态下输入搜索词时：
*   **文本样式**: `text-blue-500`。
*   **背景色块**: `bg-blue-50` (极淡蓝色背景)，圆角 `2px`。
*   **光标位置**: 始终位于 Fragment 右侧，保持输入态。
