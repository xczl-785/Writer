**更新日期**: 2026-03-18

## 一、架构概览

Writer 使用 **发布-订阅** 模式的命令总线来处理原生菜单命令。

```
┌─────────────────────────────────────────────────────┐
│           Tauri 原生菜单 (Rust 后端)                 │
│           'writer://menu-command' 事件              │
└───────────────────────┬─────────────────────────────┘
                        │ listen<MenuCommandPayload>
                        ▼
┌─────────────────────────────────────────────────────┐
│           useNativeMenuBridge (React Hook)         │
│  • 监听 'writer://menu-command' 事件               │
│  • 调用 menuCommandBus.dispatch(id)                │
└───────────────────────┬─────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────┐
│              menuCommandBus (发布-订阅)             │
│  • register(id, handler) → 返回注销函数            │
│  • dispatch(id) → 查找并执行处理器                  │
└───────────────────────┬─────────────────────────────┘
                        │
          ┌─────────────┼─────────────┐
          ▼             ▼             ▼
     fileCommands  editCommands  formatCommands
```

对于“新建文件 / 新建文件夹”入口，当前实现已额外收敛一层 **创建入口路由**：

```
menu.file.new / menu.file.new_folder
    ↓
createEntryCommands.resolveCreateEntryMenuTarget()
    ↓
dispatchCreateEntry()  // 需要时先显示 Sidebar，再延迟派发
    ↓
writer:sidebar-command
    ↓
createEntryCommands.resolveCreateEntryExplorerCommand()
    ↓
dispatchExplorerCommand()
```

## 三、menuCommandBus 设计

### 3.1 API

```typescript
interface MenuCommandBus {
  // 注册命令处理器，返回注销函数
  register(id: string, handler: () => void | Promise<void>): () => void;

  // 派发命令
  dispatch(id: string): void;
}

export const menuCommandBus: MenuCommandBus;
```

### 3.2 实现

```typescript
const handlers = new Map<string, () => void | Promise<void>>();

export const menuCommandBus = {
  register(id: string, handler: () => void | Promise<void>) {
    handlers.set(id, handler);
    return () => handlers.delete(id);
  },

  dispatch(id: string) {
    const handler = handlers.get(id);
    if (handler) {
      handler();
    }
  },
};
```

## 四、命令注册流程

### 4.1 在 App.tsx 中注册

```typescript
useEffect(() => {
  const unregister = [
    registerFileCommands(setIsSidebarVisible, isSidebarVisible),
    registerEditCommands(),
    registerFormatCommands(),
    registerParagraphCommands(),
    registerViewCommands(toggleSidebar),
  ];

  return () => {
    unregister.forEach((fn) => fn());
  };
}, [isSidebarVisible, toggleSidebar]);
```

### 4.2 单个命令注册示例

```typescript
// commands/fileCommands.ts
export function registerFileCommands(
  setIsSidebarVisible: (value: boolean) => void,
  isSidebarVisible: boolean,
): CleanupFn {
  const cleanups: CleanupFn[] = [];

  cleanups.push(
    menuCommandBus.register('menu.file.save', async () => {
      const workspace = useWorkspaceStore.getState();
      const status = useStatusStore.getState();

      if (!workspace.activeFile) {
        status.setStatus('error', t('status.menu.noWorkspace'));
        return;
      }

      // 执行保存逻辑...
    }),
  );

  return () => cleanups.forEach((fn) => fn());
}
```

### 4.3 创建入口命令收敛

`menu.file.new` 与 `menu.file.new_folder` 不再各自硬编码 Sidebar 事件细节，而是统一通过
`domains/workspace/services/createEntryCommands.ts` 处理以下语义：

- 菜单命令 ID → Sidebar 创建目标（`new-file` / `new-folder`）
- Sidebar 未显示时先展开，再在下一帧派发创建命令
- 工作区是否允许创建（基于 `currentPath`）
- Sidebar 创建目标 → Explorer 命令（`explorer.newFile` / `explorer.newFolder`）

## 六、编辑器命令转发

部分命令需要转发到编辑器组件执行：

```typescript
const emitEditorCommand = (id: string) => {
  window.dispatchEvent(
    new CustomEvent('writer:editor-command', { detail: { id } }),
  );
};

// 注册转发
menuCommandBus.register('menu.edit.undo', () => emitEditorCommand('edit.undo'));
```

编辑器组件监听并处理：

```typescript
useEffect(() => {
  const onMenuCommand = createMenuCommandHandler(
    editor,
    findReplace,
    setStatus,
    setIsOutlineOpen,
  );
  window.addEventListener(
    'writer:editor-command',
    onMenuCommand as EventListener,
  );
  return () =>
    window.removeEventListener(
      'writer:editor-command',
      onMenuCommand as EventListener,
    );
}, [editor, findReplace, setStatus]);
```

## 七、Sidebar / Explorer 创建命令

创建相关的 Sidebar 命令与 Explorer 命令当前约定如下：

| 层级 | ID | 说明 |
| --- | --- | --- |
| Sidebar 事件 | `new-file` | 触发侧边栏进入“新建文件”输入态 |
| Sidebar 事件 | `new-folder` | 触发侧边栏进入“新建文件夹”输入态 |
| Explorer 命令 | `explorer.newFile` | 统一创建文件命令 |
| Explorer 命令 | `explorer.newFolder` | 统一创建文件夹命令 |

说明：

- 顶部菜单、侧边栏按钮、空白区域右键菜单最终都应汇聚到 Explorer 命令层。
- 创建可用性必须统一基于 `canCreateFromWorkspace(currentPath)`，不得再直接用 `rootFolders.length > 0` 代替。

## 八、扩展指南

### 7.1 新增命令

1. 在 `src/app/commands/` 对应文件中添加注册：

```typescript
cleanups.push(
  menuCommandBus.register('menu.file.new_command', () => {
    // 实现逻辑
  }),
);
```

2. 在 `src-tauri/src/menu.rs` 中添加对应菜单项

### 7.2 新增命令分类

1. 创建新文件 `src/app/commands/xxxCommands.ts`
2. 实现 `registerXxxCommands` 函数
3. 在 `src/app/commands/index.ts` 中导出
4. 在 `App.tsx` 中注册

### 8.3 扩展创建入口

若未来增加新的“创建类”菜单入口，建议按以下顺序扩展：

1. 在 `src/ui/chrome/menuSchema.ts` 的 `FILE_MENU_CREATE_ITEMS` 中补充前端菜单定义
2. 在 `src/shared/i18n/messages.ts` 补充对应文案
3. 在 `src-tauri/src/menu.rs` 补充原生菜单项
4. 在 `domains/workspace/services/createEntryCommands.ts` 中补充：
   - 菜单 ID → Sidebar 目标映射
   - Sidebar 目标 → Explorer 命令映射
5. 补齐以下测试：
   - `src/ui/chrome/FileMenuCreateEntriesBehavior.test.ts`
   - `src/ui/chrome/NativeFileMenuCreateEntriesBehavior.test.ts`
   - `src/domains/workspace/services/createEntryCommands.test.ts`
