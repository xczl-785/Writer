# 滚动协调器设计

**更新日期**: 2026-03-07  
**状态**: 已实现  
**版本**: V5

---

## 一、背景

Writer 编辑器中存在多个独立的滚动触发源，原有实现分散在各个模块中，缺乏统一管理：

### 1.1 滚动来源

| 来源 | 触发场景 | 原实现位置 |
|------|---------|-----------|
| 打字机模式 | 光标位置变化、输入事件 | `useTypewriterAnchor.ts` |
| 大纲导航 | 点击大纲项跳转 | `useOutlineExtractor.ts` |
| Slash 菜单 | 键盘导航 | `SlashMenuView.tsx` |
| 查找替换 | 跳转到匹配项 | `useFindReplace.ts` |

### 1.2 存在问题

- 滚动逻辑分散，难以统一管理
- 多个滚动源可能产生冲突
- 缺乏滚动优先级机制
- 难以追踪和调试滚动行为

---

## 二、解决方案

引入**滚动协调器（Scroll Coordinator）**模块，提供统一的滚动事件管理入口。

### 2.1 设计目标

1. **集中管理**：统一滚动请求入口
2. **向后兼容**：不改变现有功能行为
3. **可扩展性**：支持未来添加优先级和冲突检测
4. **可测试性**：纯逻辑，易于单元测试

### 2.2 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                     UI 层（hooks/components）                 │
│  useTypewriterAnchor / useOutlineExtractor / useFindReplace │
└───────────────────────┬─────────────────────────────────────┘
                        │ requestScroll()
                        ▼
┌─────────────────────────────────────────────────────────────┐
│              domain/scroll/ScrollCoordinator                │
│  - 统一滚动请求入口                                          │
│  - 协调多个滚动源（未来可扩展优先级）                          │
│  - 提供类型安全的 API                                        │
└───────────────────────┬─────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────┐
│                  scrollUtils（工具函数）                     │
│  - findScrollContainer                                       │
│  - setScrollTop                                              │
│  - shouldSkipScrollAdjustment                                │
│  - resolveEditorContentTopOffset                             │
└─────────────────────────────────────────────────────────────┘
```

## 四、使用示例

### 4.1 打字机模式

```typescript
// useTypewriterAnchor.ts

import {
  findScrollContainer,
  setScrollTop,
  shouldSkipScrollAdjustment,
} from '../domain';

const scrollContainer = findScrollContainer(editorDom);
if (scrollContainer) {
  const targetScrollTop = computeTypewriterTargetScrollTop({...});
  
  if (!shouldSkipScrollAdjustment(targetScrollTop, scrollContainer.scrollTop)) {
    setScrollTop(scrollContainer, targetScrollTop);
  }
}
```

### 4.2 未来扩展示例（优先级管理）

```typescript
// 未来可扩展：添加滚动优先级

const coordinator = createScrollCoordinator();

// 打字机模式请求滚动（优先级：高）
coordinator.requestScroll({
  source: 'typewriter',
  targetScrollTop: 300,
  priority: 'high'
});

// 大纲导航请求滚动（优先级：中）
coordinator.requestScroll({
  source: 'outline-navigation',
  targetPosition: 150,
  priority: 'medium'
});
```

## 七、未来扩展方向

### 7.1 优先级管理（可选）

当多个滚动源同时触发时，可引入优先级机制：

```typescript
type ScrollPriority = 'high' | 'medium' | 'low';

type ScrollRequest = {
  // ...
  priority?: ScrollPriority;
};
```

### 7.2 冲突检测（可选）

检测并处理滚动冲突：

```typescript
coordinator.requestScroll({
  source: 'typewriter',
  targetScrollTop: 300,
  onConflict: 'override' | 'queue' | 'ignore'
});
```

### 7.3 滚动历史记录（可选）

记录滚动历史，支持撤销：

```typescript
coordinator.getHistory(): ScrollRequest[];
coordinator.undo(): void;
```
