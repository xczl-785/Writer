# 打包指南

**更新日期**: 2026-03-01
**定位**: Writer 项目跨平台打包参考文档

---

## 概述

Writer 是基于 Tauri 2.x 构建的桌面应用，支持 macOS、Windows、Linux 三大平台。本文档描述完整的打包流程、配置说明及常见问题。

---

## 快速开始

项目提供跨平台一键打包脚本，位于 `build/` 目录：

```bash
# macOS / Linux
./build/build.sh

# Windows (PowerShell)
./build/build.ps1

# Windows (Git Bash / WSL)
./build/build.sh
```

### macOS 架构选择

macOS 支持多种架构打包：

```bash
# 当前机器架构（默认）
./build/build.sh

# Apple Silicon (M1/M2/M3)
./build/build.sh --arch arm64

# Intel Mac
./build/build.sh --arch x86_64

# 通用二进制（同时支持两种架构）
./build/build.sh --arch universal
```

| 架构 | 说明 | 目标 |
|------|------|------|
| `auto` | 当前机器架构（默认） | 自动检测 |
| `arm64` | Apple Silicon | `aarch64-apple-darwin` |
| `x86_64` | Intel Mac | `x86_64-apple-darwin` |
| `universal` | 通用二进制 | `universal-apple-darwin` |

### 脚本选项

```bash
# macOS / Linux
./build/build.sh [选项]

# Windows PowerShell
./build/build.ps1 [选项]
```

| 选项 | 说明 |
|------|------|
| `-h, --help` | 显示帮助信息 |
| `-c, --clean` | 清理构建产物后重新构建 |
| `-d, --debug` | 调试模式构建 |
| `--no-install` | 跳过 npm 依赖安装 |
| `--arch <arch>` | macOS 架构选择 (arm64/x86_64/universal/auto) |

### 手动打包

```bash
# 基础打包命令
npm run tauri build

# 或使用 npx
npx tauri build
```

## 打包流程

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   前端构建      │ ──▶ │   Rust 编译     │ ──▶ │   打包安装程序  │
│  npm run build  │     │  cargo build    │     │   bundle/       │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

1. **前端构建**
   - 执行 `tsc -b && vite build`
   - 输出到 `dist/` 目录

2. **Rust 编译**
   - Cargo 编译 `src-tauri/` 为 release 版本
   - 产物位于 `src-tauri/target/release/`

3. **打包安装程序**
   - 根据平台生成对应安装包
   - 产物位于 `src-tauri/target/release/bundle/`

## 配置说明

### tauri.conf.json 关键配置

```json
{
  "productName": "writer",
  "version": "0.1.0",
  "identifier": "com.writer.app",
  "build": {
    "frontendDist": "../dist",
    "devUrl": "http://127.0.0.1:43173",
    "beforeDevCommand": "npm run dev",
    "beforeBuildCommand": "npm run build"
  },
  "bundle": {
    "active": true,
    "targets": "all",
    "icon": [
      "icons/32x32.png",
      "icons/128x128.png",
      "icons/128x128@2x.png",
      "icons/icon.icns",
      "icons/icon.ico"
    ]
  }
}
```

### 配置项说明

| 配置项           | 说明             | 默认值 |
| ---------------- | ---------------- | ------ |
| `bundle.active`  | 是否启用打包     | `true` |
| `bundle.targets` | 打包目标格式     | `all`  |
| `bundle.icon`    | 应用图标路径数组 | -      |

## 平台输出格式

### macOS

| 格式   | 说明           | 输出路径        |
| ------ | -------------- | --------------- |
| `.app` | macOS 应用包   | `bundle/macos/` |
| `.dmg` | 磁盘镜像安装包 | `bundle/dmg/`   |

### Windows

| 格式    | 说明              | 输出路径       |
| ------- | ----------------- | -------------- |
| `.msi`  | Windows Installer | `bundle/msi/`  |
| `.nsis` | NSIS 安装程序     | `bundle/nsis/` |

### Linux

| 格式        | 说明             | 输出路径           |
| ----------- | ---------------- | ------------------ |
| `.deb`      | Debian/Ubuntu 包 | `bundle/deb/`      |
| `.rpm`      | RedHat/Fedora 包 | `bundle/rpm/`      |
| `.AppImage` | 通用 Linux 包    | `bundle/appimage/` |

## 命令行选项

### 基础选项

```bash
# 仅生成 .app（macOS）
npx tauri build -b app

# 仅生成 .dmg（macOS）
npx tauri build -b dmg

# 生成多个格式
npx tauri build -b app,dmg

# 调试模式打包（更快，体积更大）
npx tauri build --debug

# 详细日志输出
npx tauri build -v
```

```bash
# macOS Universal (同时支持 Intel 和 Apple Silicon)
npx tauri build -t universal-apple-darwin

# 仅 Apple Silicon
npx tauri build -t aarch64-apple-darwin

# 仅 Intel Mac
npx tauri build -t x86_64-apple-darwin

# Windows 64位
npx tauri build -t x86_64-pc-windows-msvc
```

> **提示**: 使用 `./build/build.sh --arch <arch>` 可自动安装所需的 Rust 目标并执行构建。

### 高级选项

```bash
# 跳过打包步骤（仅编译）
npx tauri build --no-bundle

# 使用自定义配置文件
npx tauri build -c tauri.release.conf.json

# 指定 Cargo features
npx tauri build -f feature-name

# CI 模式（跳过交互提示）
npx tauri build --ci
```

## 环境要求

### 通用要求

| 工具    | 最低版本 | 推荐版本 |
| ------- | -------- | -------- |
| Node.js | 18.x     | 20.x+    |
| npm     | 9.x      | 10.x+    |
| Rust    | 1.77.2   | stable   |
| Cargo   | 1.77+    | latest   |

### macOS 额外要求

- Xcode Command Line Tools: `xcode-select --install`
- Apple Developer 账号（用于签名和公证，可选）

### Windows 额外要求

- Microsoft Visual Studio C++ Build Tools
- WebView2 Runtime（Windows 10/11 已内置）

### Linux 额外要求

```bash
# Debian/Ubuntu
sudo apt install libwebkit2gtk-4.1-dev \
    build-essential \
    curl \
    wget \
    file \
    libssl-dev \
    libgtk-3-dev \
    libayatana-appindicator3-dev \
    librsvg2-dev

# Fedora
sudo dnf install webkit2gtk4.1-devel \
    openssl-devel \
    curl \
    wget \
    file \
    libappindicator-gtk3-devel \
    librsvg2-devel

# Arch
sudo pacman -S webkit2gtk-4.1 \
    base-devel \
    curl \
    wget \
    file \
    openssl \
    appmenu-gtk-module \
    gtk3 \
    libappindicator-gtk3 \
    librsvg \
    libvips
```

## 代码签名

### macOS 签名与公证

1. 配置 Apple Developer 证书
2. 在 `tauri.conf.json` 中配置签名信息：

```json
{
  "bundle": {
    "macOS": {
      "signingIdentity": "Developer ID Application: Your Name (TEAM_ID)",
      "entitlements": null,
      "exceptionDomain": "",
      "hardenedRuntime": true
    }
  }
}
```

3. 执行打包并公证：

```bash
# 打包并等待公证完成
npx tauri build

# 跳过 stapling（首次公证可能需要数小时）
npx tauri build --skip-stapling
```

### Windows 签名

1. 获取代码签名证书
2. 配置环境变量或 `tauri.conf.json`：

```json
{
  "bundle": {
    "windows": {
      "certificateThumbprint": "YOUR_CERT_THUMBPRINT",
      "digestAlgorithm": "sha256",
      "timestampUrl": "http://timestamp.digicert.com"
    }
  }
}
```

## 常见问题

### 构建失败：前端资源未找到

**症状**: `Error: failed to read frontendDist`

**解决方案**: 确保先执行前端构建：

```bash
npm run build
npx tauri build
```

### macOS: 找不到 webkit2gtk

**症状**: macOS 不需要 webkit2gtk，此错误通常出现在错误平台上。

**解决方案**: 确保在 macOS 上构建 macOS 版本。

### Windows: 缺少 WebView2

**症状**: 应用启动失败或显示空白窗口

**解决方案**:

- Windows 10/11 已内置 WebView2
- 旧版 Windows 需手动安装 [WebView2 Runtime](https://developer.microsoft.com/en-us/microsoft-edge/webview2/)

### Linux: 编译错误

**症状**: Rust 编译失败，缺少系统库

**解决方案**: 安装系统依赖（见上方 Linux 环境要求）

### 构建产物过大

**解决方案**:

1. 确保 release 模式构建（默认）
2. 检查是否误包含了开发依赖
3. 考虑启用 Rust 的 LTO 优化
