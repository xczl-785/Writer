# 打字机模式设计

**更新日期**: 2026-03-08  
**状态**: 已实现（V2.1）

---

## 一、定位

打字机模式采用**事件驱动状态机**实现，核心目标是提供稳定的 `Free / Locked` 双态体验，避免传统“固定居中拉扯”造成的视觉跳跃。

---

## 二、核心模型

### 2.1 状态模型

```ts
type TypewriterState = {
  mode: 'free' | 'locked';
  dynamicAnchorY: number | null;
  lastTrigger: 'input' | 'ime' | 'arrow' | 'mouse' | 'external' | null;
};
```

- `free`: 光标自然移动，不做强制补偿。
- `locked`: 光标垂直位置锁定，视图反向补偿。
- `dynamicAnchorY`: 进入 `locked` 时采样并固定的锚点坐标。

### 2.2 事件模型

统一事件封装：

```ts
type TypewriterEventEnvelope = {
  type: TypewriterEventType;
  timestamp: number;
  source: TriggerSource;
  payload?: unknown;
};
```

事件分三类：
- 输入链：`before-input` / `enter-keydown` / `arrow-keydown` / `composition-start` / `composition-end`
- 定位链：`mouse-caret-placed` / `find-jump` / `outline-jump` / `programmatic-selection`
- 环境链：`viewport-measured` / `selection-changed` / `force-free`

---

## 三、转移规则

### 3.1 `Free -> Locked`

- 输入链事件使光标**触碰或越过**阈值线（默认 `45%`）时进入 `locked`。
- 锚点采样为当前 `nextCaretTop`。
- 支持双向捕获（上行/下行均可触发）。

### 3.2 `Locked -> Locked`

- 仅输入链事件触发补偿。
- 补偿目标：
  - `targetScrollTop = currentScrollTop + (caretTop - dynamicAnchorY)`
- 动画策略：`requestAnimationFrame` 手动补偿，默认 `140ms`。

### 3.3 `Locked -> Free`

以下事件直接回退 `free`，且不做回拉：
- `mouse-caret-placed`
- `find-jump`
- `outline-jump`
- `programmatic-selection`
- `force-free`

此外，外部触发出现“向上补偿极限路径”时，立即降级 `free`。

---

## 四、边界策略

- **短内容保护**：内容高度不足时，整体降级 `free`。
- **激活边界捕获**：短内容刚转为可激活且光标已在阈值线下方时，立即进入 `locked`，避免“底部一直不锁定”。
- **IME 处理**：组合输入期间不补偿，`composition-end` 触发闭环更新。
- **非落点点击**：不改变状态。
