## 六、文件系统抽象设计

### 6.1 原子写入

使用临时文件 + rename 模式，确保崩溃安全：

```rust
pub async fn write_file_atomic(path: String, content: String) -> Result<(), String> {
    let temp_path = format!("{}.tmp", path);
    fs::write(&temp_path, &content).map_err(|e| e.to_string())?;
    fs::rename(&temp_path, &path).map_err(|e| e.to_string())?;
    Ok(())
}
```

### 6.2 平台适配

`reveal_in_file_manager` 支持跨平台：

```rust
#[cfg(target_os = "macos")]
Command::new("open").args(["-R", &path]).spawn();

#[cfg(target_os = "windows")]
Command::new("explorer").args(["/select,", &path]).spawn();

#[cfg(target_os = "linux")]
Command::new("xdg-open").arg(parent_dir).spawn();
```

## 七、原生菜单设计

### 7.1 菜单结构

- **File**: 新建、打开文件夹、关闭文件夹、保存、另存为、导出
- **Edit**: 撤销、重做、剪切、复制、粘贴、全选、查找、替换
- **Paragraph**: H1-H6、引用、代码块、列表、表格、分割线
- **Format**: 加粗、斜体、行内代码、删除线、链接、图片
- **View**: 显示大纲、切换侧栏、专注模式、源码模式

### 7.2 国际化

根据 `LANG` 环境变量自动切换中英文：

```rust
fn get_locale() -> String {
    std::env::var("LANG")
        .unwrap_or_else(|_| "en".to_string())
        .split('.')
        .next()
        .unwrap_or("en")
        .to_string()
}
```

## 八、前后端通信协议

### 8.1 调用模式

```typescript
// 前端调用
import { invoke } from '@tauri-apps/api/core';
const content = await invoke('read_file', { path: '/path/to/file' });
```

### 8.2 返回格式

- 成功：直接返回 Rust 类型的序列化 JSON
- 失败：`Result<T, String>` → 错误时返回 String 消息

### 8.3 事件通道

| 事件名                  | 方向        | 用途         |
| ----------------------- | ----------- | ------------ |
| `writer://menu-command` | 后端 → 前端 | 菜单命令事件 |
| `writer://file-change`  | 后端 → 前端 | 文件监听事件 |

## 九、架构评估

### 9.1 优点

- 模块划分清晰（fs/workspace/config/menu 分离）
- 命令设计原子化，单一职责
- 错误处理统一（返回 `Result<T, String>`）
- 跨平台文件操作适配
- 工作区锁机制防止多实例冲突

### 9.2 已移除功能

- Git 同步状态查询（`get_git_sync_status`）已移除
