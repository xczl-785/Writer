## 六、文件系统抽象设计

### 6.1 原子写入

使用临时文件 + rename 模式，确保崩溃安全：

```rust
pub async fn write_file_atomic(path: String, content: String) -> Result<(), String> {
    let temp_path = format!("{}.tmp", path);
    fs::write(&temp_path, &content).map_err(|e| e.to_string())?;
    fs::rename(&temp_path, &path).map_err(|e| e.to_string())?;
    Ok(())
}
```

### 6.2 平台适配

`reveal_in_file_manager` 支持跨平台：

```rust
#[cfg(target_os = "macos")]
Command::new("open").args(["-R", &path]).spawn();

#[cfg(target_os = "windows")]

#[cfg(target_os = "linux")]
Command::new("xdg-open").arg(parent_dir).spawn();
```

## 七、原生菜单设计

### 7.1 菜单结构

- **File**: 新建、打开文件夹、关闭文件夹、保存、另存为、导出
- **Edit**: 撤销、重做、剪切、复制、粘贴、全选、查找、替换
- **Paragraph**: H1-H6、引用、代码块、列表、表格、分割线
- **Format**: 加粗、斜体、行内代码、删除线、链接、图片
- **View**: 显示大纲、切换侧栏、专注模式、源码模式

### 7.2 国际化

根据 `LANG` 环境变量自动切换中英文：

```rust
fn get_locale() -> String {
    std::env::var("LANG")
        .unwrap_or_else(|_| "en".to_string())
        .split('.')
        .next()
        .unwrap_or("en")
        .to_string()
}
```

## 九、架构评估

### 9.1 优点

- 模块划分清晰（fs/menu 分离）
- 命令设计原子化，单一职责
- 错误处理统一（返回 `Result<T, String>`）
- 跨平台文件操作适配

### 9.2 潜在改进

- 缺少命令权限控制
- fs.rs 混入了 Git 业务逻辑（可考虑分离）
- 无命令版本控制和 API 契约文档
