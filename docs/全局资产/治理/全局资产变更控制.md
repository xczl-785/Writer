# 全局资产变更控制规范

## 1. 目的

规范 `docs/全局资产/` 的修改流程，确保长期资产可追溯、可审计、可复盘。

## 2. 适用范围

- 目录：`docs/全局资产/`
- 文档类型：产品定位、决策台账、工程规范、治理规范等长期有效文档

## 3. 变更触发条件

仅允许以下场景修改全局资产文档：

1. 发现与已通过复盘结论冲突的错误信息（事实性修复）。
2. 关键约束发生已批准变更（范围、门禁、质量口径）。
3. 审计要求的可追溯性修正（如错误路径、失效引用、缺失决策编号）。

## 4. 禁止项

1. 未经评审修改长期定位、治理规则或工程规范。
2. 将阶段性执行策略直接写入全局资产而不标注时效。
3. 删除既有决策记录而不保留替代说明。

## 5. 变更流程

1. 提交变更申请：说明“变更原因/影响范围/风险”。
2. 评审确认：至少 1 名规划负责人 + 1 名架构/技术负责人确认。
3. 同步更新：

- `docs/全局资产/治理/决策日志.md`（新增或更新决策项）
- 对应 `docs/archive/**` 的评审记录（记录本次变更结论）

4. 合并后回填：在相关 `docs/current/*` 执行文档同步引用新版本约束。

## 6. 记录模板（最小必填）

- 变更 ID：`FDC-YYYYMMDD-序号`
- 变更文档：
- 变更类型：`事实修复` / `约束变更` / `可追溯性修复`
- 变更原因：
- 影响范围：
- 回滚策略：
- 评审人：
- 生效日期：

## 7. 版本与命名规则

1. 默认不改文件名，仅在文档内维护“最后更新日期”。
2. 重大变更可新增后缀版本（如 `_V2`），并在旧文档首部声明“已被替代”。
