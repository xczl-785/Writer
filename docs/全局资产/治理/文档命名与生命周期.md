# 文档命名与生命周期规范

## 1. 目的

统一文档命名风格与状态流转，降低多版本并行造成的歧义。

## 2. 目录职责

1. `docs/全局资产/`：长期生效的产品/治理/工程资产。
2. `docs/current/`：当前执行与协作文档。
3. `docs/archive/`：历史复盘、过程资产、证据归档。

## 3. 命名规范

1. 规则文档（长期有效）：

- `UPPER_SNAKE_CASE.md` 或既有固定名（如 `产品需求文档_V1.md`）。

2. 执行态文档（current）：

- 建议：`YYYY-MM-DD_<主题>.md`

3. 评审/审阅文档：

- 建议：`YYYY-MM-DD_REVIEW_<主题>_<角色>.md`

4. 禁止使用难以区分的模糊后缀（如仅 `V2`、`final`）。

## 4. 生命周期状态

状态定义：

1. `DRAFT`：草案，允许高频修改。
2. `IN_REVIEW`：评审中，原则上只改评审意见项。
3. `APPROVED`：评审通过，等待冻结或归档。
4. `FROZEN`：冻结基线，仅走变更控制流程。
5. `ARCHIVED`：历史记录，不再作为当前执行输入。

## 5. 状态流转规则

1. `current` 文档评审通过后，若为长期约束，迁入 `docs/全局资产/`。
2. 评审纪要、执行台账、历史报告迁入 `archive`。
3. 任一文档流转时应在文内标注：

- 当前状态
- 最后更新时间
- 上游来源（若有）

## 6. 兼容规则

现存历史文档可保留原名；新建文档必须遵循本规范。
