# UI/UX 基线（Step C）

## 1. 目标
将 V1 的体验要求固化为可实现、可验收的界面与交互标准。

## 2. 设计原则（强制）
1. 无打断：常用写作流程不弹窗打断。
2. 低认知负担：界面元素最小化，信息层级稳定。
3. 快速反馈：关键动作 1 次操作内完成并有即时反馈。
4. 本地优先：文件状态与保存状态明确可见。

## 3. 信息架构
- 左侧：文件树区（工作区、目录、文件）
- 中央：编辑区（单文档所见即所得）
- 底部：状态区（保存状态、路径简写）

## 4. 布局规范
1. 左侧栏默认宽度 260px，可拖动，范围 220-360px。
2. 编辑区内容最大宽度：默认态 860px；当窗口宽度较小（<= 1366）时按 800px 收敛；超宽屏保持居中。
3. 主内容区域留白：左右 24px。
4. 顶部/底部留白分为两档：
- 默认态：顶部 20px，底部 80px（贴近日常编辑效率）。
- 专注态（仅视觉态，不新增功能）：顶部 120px，底部 300px（长文沉浸阅读/写作）。
5. 默认单窗口单编辑区，不做分屏。

## 5. 视觉规范（V1）
1. 视觉风格：浅色为默认，低对比背景 + 高可读正文。
2. 字体：中文优先系统无衬线，等宽用于代码块。
3. 正文字号 16px，行高 1.65，段间距稳定。
4. 状态颜色：
- 保存成功：中性灰
- 保存中：蓝色提示
- 保存失败：红色提示（常驻直到恢复）

## 6. 关键交互流程

### 6.1 打开工作区
- 启动后显示“打开文件夹”入口。
- 打开成功后渲染文件树，并进入“默认文件或空白态”。

### 6.2 新建文件
- 在文件树触发“新建文件”。
- 新文件立即进入编辑态并聚焦光标。
- 若命名冲突，给出内联错误并阻止提交。

### 6.3 重命名文件
- 支持右键或快捷操作进入重命名。
- 回车确认，Esc 取消。
- 重命名成功后当前编辑上下文不丢失。

### 6.4 删除文件
- 删除前二次确认。
- 删除当前文件后自动切换到相邻文件或空状态页。

### 6.5 图片粘贴
- 粘贴后立即可见占位/预览。
- 落盘成功后替换为最终路径引用。
- 失败时在状态区和编辑区局部同时提示。

### 6.6 自动保存
- 输入后 800ms 防抖保存。
- 失焦保存、关闭前强制保存。
- 保存失败时状态区常驻错误，成功后自动清除。

## 7. 快捷键基线（V1）
1. Cmd/Ctrl + B：加粗
2. Cmd/Ctrl + I：斜体
3. Cmd/Ctrl + 1..6：标题层级
4. 列表快捷键：Cmd/Ctrl + Shift + L
5. 引用快捷键：Cmd/Ctrl + Shift + B
6. Cmd/Ctrl + Z：撤销
7. Cmd/Ctrl + Shift + Z：重做

## 8. 边界与禁区
1. 不做复杂主题系统（仅保留基础视觉参数）。
2. 不做动画导向交互（只保留必要反馈）。
3. 不引入多窗格、多面板工作流。
4. 不在 Step C 引入 Should/Could 功能范围。

## 9. 验收标准（Step C DoD）
1. 信息架构、布局、视觉、关键流程均已明确。
2. 每个关键流程可映射到 V1 Must 功能。
3. 禁区清晰，不与 Not in V1 冲突。
4. 关键尺寸（宽度/留白）在不同窗口或视觉态下有明确切换规则。
