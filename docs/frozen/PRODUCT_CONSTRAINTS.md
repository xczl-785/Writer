# 产品约束确认（Step A）

## 1. 文档目的
冻结 V1 开发前的产品约束，包含：
- 性能口径（可测）
- 平台边界
- 非目标（Not in V1）
- 待确认项（仅保留必要决策）

## 2. 已确认项

### 2.1 平台边界
- V1 必做：macOS、Windows
- Linux：可选（不阻塞 V1 首发）

### 2.2 非目标（Not in V1）
- 插件生态、双向链接、知识图谱
- 云同步、多人协作、账号系统
- 分屏预览模式、Vim/Emacs 键位
- 流程图/图表支持（Mermaid 等）
- 表格公式计算、智能引号/破折号、文件标签/属性系统

### 2.3 需求范围基线
- V1 只做 Must 12（见 `docs/frozen/SCOPE_FREEZE_V1.md`）
- 表格可视化编辑不在 V1，列为 Phase 2 Top Priority

## 3. 性能口径（规划版）

说明：用户（开发者）未做过性能基线分析，本节采用“先可测、再优化”的工程化口径；后续可基于真实数据调整。

### 3.1 启动性能
1. 冷启动到“可输入”时间（TTC, Time To Caret）
- 目标值：<= 1200ms
- 可接受上限：<= 1800ms

2. 热启动到“可输入”时间
- 目标值：<= 600ms
- 可接受上限：<= 900ms

### 3.2 交互性能
1. 常规输入响应（按键到屏幕更新）
- 目标值：P95 <= 16ms
- 上限值：P95 <= 33ms

2. 文件切换时间（中等文档，约 30-100KB）
- 目标值：<= 120ms
- 可接受上限：<= 200ms

3. 文件树操作（展开/折叠/打开）
- 目标值：<= 100ms
- 可接受上限：<= 150ms

### 3.3 稳定性与数据安全
1. 自动保存可靠性
- 崩溃/异常退出后，已保存内容不丢失

2. 长时间编辑稳定性
- 连续编辑 60 分钟无明显卡顿恶化、无输入丢失

### 3.4 资源约束（V1 建议）
1. 空闲内存占用（仅打开应用，1 个中等文档）
- 建议目标：<= 300MB

2. 包体大小（Release 安装包）
- 建议目标：尽量控制在 <= 80MB

## 4. 测量规则（避免口径漂移）
1. 每项指标至少测 5 次，取中位数与 P95。
2. 冷启动测试前重启应用并清理热缓存影响。
3. 同一台机器、同一数据集进行横向比较。
4. 每次迭代记录结果到测试日志，避免“体感替代数据”。

## 5. 三项关键确认（已拍板）

### 5.1 工作区模式（已确认）
- 确认结论：V1 仅支持单工作区（单根目录）。
- 描述说明：一次只能加载 1 个根目录；该目录下所有子目录可见可编辑；不支持同时加载多个互不相关根目录（多根工作区）。
- 规划：
  1. 启动时选择并加载一个根目录。
  2. 在设置或菜单中提供“切换工作区”。
  3. 多根工作区能力不进入 V1，列入后续版本评估。

### 5.2 图片命名规则（已确认）
- 确认结论：采用 `YYYYMMDD-HHMMSS-随机后缀.png`。
- 描述说明：图片保存到当前文档同级 `assets/`，文档使用相对路径引用。
- 隐藏风险与应对：
  1. 同秒多图重名风险：通过随机后缀规避。
  2. 时间排序受本机时钟影响：命名仅用于可读与去重，不作为业务逻辑依赖。
  3. assets 目录膨胀：V1 先不做自动清理，后续版本评估“未引用图片清理”。
- 规划：
  1. 默认输出格式为 PNG。
  2. 文件名碰撞时自动重试生成新后缀。
  3. 失败时给出可见错误提示，不静默失败。

### 5.3 自动保存策略（已确认）
- 确认结论：输入后 800ms 防抖保存 + 窗口失焦保存 + 关闭前强制保存。
- 描述说明：以“无感保存、尽量不丢”为目标；保存失败必须可见提示。
- 规划：
  1. 输入防抖计时期间持续输入则重置计时。
  2. 失焦与关闭前保存优先级高于防抖保存。
  3. 保存失败记录错误状态，并允许用户继续编辑。
