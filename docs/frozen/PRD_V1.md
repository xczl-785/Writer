## 2. 产品目标
构建一个轻量、启动快、无打断的本地 Markdown 写作工具：
1. 输入即渲染（无需切换预览）
2. 应用内完成基础文件管理
3. 保证写作过程稳定、安全（自动保存、撤销重做）

## 3. 目标用户与核心场景
- 用户：重视写作手感与速度的个人作者（技术/非技术皆可）
- 核心场景：
  1. 打开文件夹作为工作区
  2. 新建文档并连续写作
  3. 插入代码块与图片
  4. 重命名/删除文件并继续写作
  5. 关闭后重开，内容仍完整存在

## 4. V1 功能范围（冻结）
仅实现以下 Must 12：
1. 即时渲染编辑
2. 段落级格式快捷键
3. 行内样式快捷键
4. 代码块编辑
5. 图片粘贴与本地保存
6. 自动保存

8. 侧边栏文件树

12. 文件夹作为工作区

## 5. 功能需求明细（每项含场景/交互/验收）

### FR-01 即时渲染编辑
- 用户场景：用户输入 Markdown 时，界面直接显示格式化结果。
- 交互规则：
  1. 默认单编辑视图，不提供分屏预览。
  2. 常见语法（标题、列表、加粗、斜体、代码）输入后即时生效。
- 验收标准：
  1. 输入 `# 标题` 后立即显示为一级标题样式。
  2. 输入 `**bold**` 后立即显示加粗效果。
  3. 过程中不出现“切换预览模式”步骤。

### FR-02 段落级格式快捷键
- 用户场景：用户通过快捷键快速设置标题/列表/引用。
- 交互规则：
  1. 提供 Cmd/Ctrl+1..6 标题快捷键。
  2. 提供列表与引用的快捷键入口（平台一致映射）。
- 验收标准：
  1. 选中段落后触发快捷键，样式立即变化。
  2. 同一快捷键再次触发可回退到普通段落（或明确规则）。

### FR-03 行内样式快捷键
- 用户场景：用户快速对选中文本加粗、斜体、行内代码。
- 交互规则：
  1. Cmd/Ctrl+B 加粗，Cmd/Ctrl+I 斜体。
  2. 行内代码提供可达快捷键或统一工具入口。
- 验收标准：
  1. 选中文本后快捷键生效，光标与选区行为稳定。
  2. 再次触发可取消样式（或按规则包裹/解包裹）。

### FR-04 代码块编辑
- 用户场景：技术写作中插入并编辑代码块。
- 交互规则：
  1. 输入三个反引号可创建代码块。
  2. 代码块支持基础语法高亮。
- 验收标准：
  1. 输入 ``` 后创建独立代码块区域。
  2. 至少支持常见语言高亮（如 js、ts、python、json、bash）。

### FR-05 图片粘贴与本地保存
- 用户场景：用户截图后直接粘贴到文档。
- 交互规则：
  1. 粘贴图片时自动落盘到当前文档同级 `assets/`。
  2. 文档中写入相对路径引用。
- 验收标准：
  1. 截图粘贴后，编辑区立即显示图片。
  2. 文件系统可看到新图片文件，文档中为相对路径。

### FR-06 自动保存
- 用户场景：用户连续写作，不手动保存也不丢数据。
- 交互规则：
  1. 文本变更后自动保存到当前文件。
  2. 保存失败需有可见提示。
- 验收标准：
  1. 输入内容后关闭再打开，修改仍存在。
  2. 模拟保存失败时，界面可见错误提示。

### FR-07 撤销/重做
- 用户场景：用户修正误操作并恢复内容。
- 交互规则：
  1. Cmd/Ctrl+Z 撤销。
  2. Cmd/Ctrl+Shift+Z（或平台等价）重做。
- 验收标准：
  1. 连续输入至少 20 步可撤销。
  2. 撤销后可按顺序重做恢复。

### FR-08 侧边栏文件树
- 用户场景：用户浏览工作区文件结构并打开文件。
- 交互规则：
  1. 左侧显示目录树，支持展开/折叠。
  2. 点击文件在编辑区打开。
- 验收标准：
  1. 工作区目录层级正确显示。
  2. 文件切换时编辑区内容与目标文件一致。

### FR-09 应用内新建文件
- 用户场景：用户不离开应用直接新建文档。
- 交互规则：
  1. 在文件树提供“新建文件”。
  2. 新建后自动打开并进入可编辑状态。
- 验收标准：
  1. 新文件立即出现在文件树。
  2. 可直接输入并自动保存到磁盘。

### FR-10 文件重命名
- 用户场景：用户整理文件名。
- 交互规则：
  1. 文件树支持重命名（右键或快捷方式）。
  2. 重命名后当前打开状态保持一致。
- 验收标准：
  1. 重命名后文件树立即刷新为新名称。
  2. 若该文件正在编辑，编辑区不中断。

### FR-11 文件删除
- 用户场景：用户删除不需要的文档。
- 交互规则：
  1. 删除动作需二次确认。
  2. 删除后若当前文件被删，编辑区切到安全空状态或相邻文件。
- 验收标准：
  1. 删除后文件树不再显示该文件。
  2. 磁盘上对应文件被删除。

### FR-12 文件夹作为工作区
- 用户场景：用户选择本地目录开始写作。
- 交互规则：
  1. 启动后可选择并打开一个文件夹。
  2. 记住上次工作区并支持再次打开。
- 验收标准：
  1. 选择目录后文件树正确加载。
  2. 重启应用可恢复上次工作区（若目录仍可访问）。

## 6. 非功能要求（V1 最小集）
1. 启动速度：冷启动应明显快于典型 IDE 级工具（以“秒开可写”为目标）。
2. 交互流畅：常规输入不出现明显卡顿或丢字。
3. 稳定性：异常场景不应导致已保存内容损坏。

## 7. Not in V1（明确排除）
- 插件生态、双向链接、知识图谱
- 云同步、多人协作、账号系统
- 分屏预览模式、Vim/Emacs 键位
- 流程图/图表支持（Mermaid 等）
- 表格公式计算、智能引号/破折号、文件标签/属性系统

## 8. 依赖与风险
1. 编辑器内核能力不足会影响所见即所得一致性。
2. 文件系统权限差异会影响跨平台行为。
3. 图片自动保存路径策略需在实现前锁定细节（命名规则、冲突处理）。
