## 2. Critical 问题解决情况验证

### ✅ CR-01: FileWatcherService 技术选型 - 架构已解决，实现有缺陷

**原问题**: 使用 `chokidar`（Node.js 库），Tauri 前端无法使用。

**V1.2 修改验证**:

| 检查项         | 状态 | 说明                                                             |
| -------------- | ---- | ---------------------------------------------------------------- |
| 架构决策表更新 | ✅   | 明确声明 "Rust notify crate + Tauri 事件推送"                    |
| Rust 后端实现  | ✅   | 完整实现 `start_watching`、`stop_watching`、`update_watch_paths` |
| 前端订阅模式   | ✅   | 使用 `listen('writer://file-change')` 订阅后端事件               |

**🔴 新发现的实现缺陷**:

1. **watcher 生命周期管理错误**（后端工程师发现）:

   ```rust
   // 当前实现 - watcher 被泄漏
   pub fn start_watching(app: tauri::AppHandle, paths: Vec<String>) -> Result<(), String> {
       let mut watcher = watcher(tx, Duration::from_millis(300))?;
       // watcher 被 move 到线程中，函数返回后无法从外部停止！
       std::thread::spawn(move || { ... });
       Ok(())  // watcher 已被 move，外部无法访问！
   }
   ```

2. **前端防抖逻辑失效**（前端工程师发现）:
   ```typescript
   // 当前实现 - 防抖不会生效
   private handleBackendEvent(payload: any): void {
       const debounceTimers = new Map<string, NodeJS.Timeout>(); // 每次都创建新 Map
       // ...
   }
   ```

**必须修正**:

- 后端：使用全局状态（`Arc<Mutex<Option<WatcherState>>>`）管理 watcher 生命周期
- 前端：将 `debounceTimers` 提升为类成员变量

| removeFolderFromWorkspace 快照 - 回滚 | ✅   | 同样实现快照 - 回滚模式              |
| 错误处理                              | ✅   | catch 块中执行回滚并返回错误         |

**结论**: 快照 - 回滚机制实现完整，满足原子性保障要求。

   - `list_tree_batch` 未调用验证
   - `save_workspace_file` 未调用验证
   - `resolve_relative_path` 未调用验证

2. **新文件路径验证失败**:

**必须修正**:

- 在所有文件操作命令中注入 `WorkspaceAllowlist` 状态
- 修改 `validate_path` 支持 `must_exist` 参数，区分已存在/新文件场景

### ✅ M-02: list_tree_batch 并行化 - 已实现，缺少限制

**V1.2 修改验证**:

| 检查项   | 状态 | 说明                          |
| -------- | ---- | ----------------------------- |
| 并行实现 | ✅   | 使用 `thread::spawn` 并行加载 |
| 错误隔离 | ✅   | 单个路径失败不影响其他路径    |

**🟡 遗留问题**:

- 无并发限制（用户打开 50+ 文件夹会创建 50+ 线程）
- 无输入验证（未验证路径是否在工作区内）
- 无超时机制（挂载的网络驱动器可能无限阻塞）

**建议修正**:

- 添加并发限制（建议最大 8-20 线程）
- 添加路径长度限制（如 max 20 个文件夹）
