# Sprint 2.5 交互规范（文件树体验对齐）

## 1. 阶段定位

- 阶段名称：Sprint 2.5（体验纠偏阶段）
- 阶段目标：在不扩展新功能边界的前提下，对齐文件树交互体验与平台习惯，重点参考 VS Code 的 Explorer 交互模型。
- 与 Sprint 2 关系：Sprint 2.5 为新增阶段，关注“行为一致性与体验质量”，不再重复定义 Sprint 2 的功能闭环。

## 2. 设计原则

1. 原位操作优先（Inline First）
- 新建、重命名在树内完成，不跳离上下文。

2. 危险操作显式确认（Safe by Default）
- 删除必须确认；支持后续接入“不要再提示”配置。

3. 键盘与鼠标双通路等价
- 任一核心操作都有鼠标入口与快捷键入口。

4. 平台化快捷键映射
- 同一命令，不同平台按键可不同；命令层统一，键位层按平台映射。

5. 可预测的焦点流
- 操作完成后，焦点、选中、展开状态保持连续，避免“找不到刚操作的节点”。

## 3. 文件树显示规范

1. 可见性
- 仅显示 Markdown 文件：`.md` / `.markdown` / `.mdx`。
- 非 Markdown 文件不显示。
- 过滤后无子项的目录不显示。

2. 命名展示
- 文件节点显示“主名”（隐藏后缀）。
- 目录节点显示原始目录名。

3. 新建命名规则
- 新建文件输入默认不带后缀（如 `untitled`）。
- 提交时自动补全 `.md`（若用户未输入 markdown 后缀）。

4. 重命名规则
- 文件重命名输入默认不带后缀。
- 提交时自动保留原后缀。
- 目录重命名按原名提交，不涉及后缀逻辑。

## 4. 交互行为规范（对齐 VS Code 风格）

### 4.1 新建文件/文件夹

- 触发入口：顶部工具按钮（New File / New Folder）。
- 上下文定位：
  - 选中目录：在该目录下创建。
  - 选中文件：在该文件同级目录创建。
  - 无选中：在工作区根目录创建。
- 交互流程：
  1. 进入树内输入态。
  2. `Enter` 提交，`Esc` 取消。
  3. 提交成功后刷新树。
  4. 新建文件自动打开；新建目录不自动打开编辑器。

### 4.2 重命名文件/文件夹

- 触发入口：节点操作按钮（后续可补 F2/Enter）。
- 交互流程：
  1. 节点切换为可编辑输入态。
  2. `Enter` 提交，`Esc` 取消。
  3. 提交前进行合法性校验（空名、非法字符、重名）。
  4. 若目标节点有脏数据，先执行 flush 保护。
  5. 成功后刷新树并更新 active/openFiles/editor 状态。

### 4.3 删除文件/文件夹

- 触发入口：节点删除按钮（后续可补 Delete/Backspace 键）。
- 交互流程：
  1. 弹出确认层。
  2. 显示“Do not ask again（coming soon）”禁用入口。
  3. 仅在点击 Confirm 后执行删除。
  4. 删除前先 flush 脏文件。
  5. 成功后刷新树并清理 editor/workspace 关联状态。

### 4.4 焦点与选中态

- 单击节点：设置 selected。
- 单击目录：展开/收起（不触发文件打开）。
- 单击文件：打开并成为 active。
- 删除/重命名成功后：
  - 优先保持同层焦点连续（邻接项）；
  - 若无邻接项，回退父目录；
  - 无父目录时清空选中。

## 5. 平台快捷键策略（Win / macOS）

> 目标：实现“同命令 + 分平台键位映射 + 上下文条件”的架构，而不是把快捷键写死在组件逻辑里。

### 5.1 命令层（统一）

定义稳定命令 ID：
- `explorer.newFile`
- `explorer.newFolder`
- `explorer.rename`
- `explorer.delete`
- `explorer.open`
- `explorer.collapseOrClose`

### 5.2 键位层（按平台）

建议映射（实现口径）：
- 重命名：
  - Win：`F2`
  - macOS：`Enter`
- 删除：
  - Win：`Delete`
  - macOS：`Cmd+Backspace`（或 `Delete`，按最终产品定义）
- 新建文件：
  - Win：`Ctrl+N`（仅 Explorer 焦点下）
  - macOS：`Cmd+N`（仅 Explorer 焦点下）

### 5.3 上下文条件（when）

仅在满足条件时触发：
- `explorerFocus == true`
- `renameInputActive == false`（避免输入态冲突）
- `modalOpen == false`

### 5.4 实现建议

1. 新建 `keybinding` 配置层（按平台导出）
2. 新建 `command dispatcher`（统一调用 UI/store/fs 操作）
3. 组件中只订阅 command，不直接硬编码平台按键

## 6. 视觉与反馈规范

1. 图标显隐
- 普通态弱显，hover 态增强。

2. 输入态强调
- 输入框边框高亮，明确当前编辑对象。

3. 状态反馈
- 失败必须弹可见错误。
- 成功动作不打断输入流（少弹窗，除删除）。

4. 文案语气
- 错误文案直接说明原因（如：名称非法、目标已存在、权限不足）。

## 7. 非目标（Sprint 2.5 不做）

1. 拖拽移动文件
2. 多选批量操作
3. 搜索与过滤器 UI
4. 多工作区与 Tabs 体系

## 8. 验收建议（供 Sprint 2.5 使用）

1. 新建、重命名、删除均支持 Enter/Esc 流。
2. 删除取消后不产生任何磁盘变更。
3. Win/macOS 快捷键符合平台习惯并通过回归。
4. 文件树仅展示 markdown 文件，后缀隐藏显示一致。
