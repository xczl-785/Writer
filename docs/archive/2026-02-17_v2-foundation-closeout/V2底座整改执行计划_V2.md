# V2 底座整改执行计划\_V2

## 1. 背景与目标

根据 `docs/archive/2026-02-17_v2-foundation-closeout/V2底座稳定性评估与重构建议_V2.md` 的评估结果，当前底座存在 Lint 报错、构建阻塞及样式变量缺失等问题。为确保 V2 功能开发在稳固的底座上进行，特制定本整改计划。

**核心目标**：

- 实现 `npm run lint` 零错误。
- 实现 `npm run build` 成功构建。
- 清理冗余代码与目录，优化底座结构。
- 保持 `npm run test` 100% 通过。

## 2. 核心原则

1. **禁止功能变更**：仅进行代码质量治理与结构优化，严禁修改业务逻辑或引入新功能。
2. **子代理分工**：主代理负责全局配置与门禁检查，子代理负责大批量、重复性的文件修复。
3. **线性逐项推进**：严格遵循阶段顺序，前一阶段未通过 Gate 校验，禁止进入下一阶段。

- **Test**: ✓ 66 passed (100%)

## 4. 分阶段执行计划

### Phase 1: 规范与类型治理 (Lint/Prettier/Any)

- **目标**：消除所有 Lint 错误，统一代码风格，减少 `any` 滥用。
- **影响范围**：`src/` 目录下所有被 ESLint 覆盖的文件。
- **执行步骤**：
  1.  主代理：下发子代理任务，执行 `npm run lint -- --fix` 处理可自动修复项。
  2.  子代理：逐个修复 `AutosaveService.test.ts` 和 `useImagePaste.test.ts` 中的 `any` 类型。
  3.  子代理：修复 `pasteHandler.ts` 等文件的 Prettier 格式问题。
- **验证命令**：`npm run lint`
- **完成定义**：输出显示 `0 errors`（允许保留必要的 warnings，但需人工确认）。
- **回滚点**：若修复导致测试失败，回滚对应文件的修改。

### Phase 2: 构建阻塞修复 (Build/TS Config)

- **目标**：解决测试文件中的类型引用错误，确保生产构建通畅。
- **影响范围**：`src/ui/editor/EditorStyles.test.ts` 及相关 TS 配置。
- **执行步骤**：
  1.  主代理：分析 `node:fs` 等模块引用失败原因（通常为 `@types/node` 缺失或 `tsconfig` 配置问题）。
  2.  主代理：修正 `tsconfig.json` 或在受影响文件中添加正确的类型声明。
  3.  主代理：执行构建测试。
- **验证命令**：`npm run build`
- **完成定义**：`vite build` 成功完成并产出 `dist` 目录。
- **回滚点**：`git checkout tsconfig.json src/ui/editor/EditorStyles.test.ts`

### Phase 3: 样式变量与冗余项治理

- **目标**：修复 UI 渲染隐患，清理 Rust 模板冗余代码及空目录。
- **影响范围**：`src/ui/statusbar/StatusBar.css`, `src-tauri/src/lib.rs`, 物理目录结构。
- **执行步骤**：
  1.  子代理：在全局 CSS 中定义 `StatusBar.css` 缺失的变量（如 `--bg-secondary`）。
  2.  子代理：移除 `src-tauri/src/lib.rs` 中的 `greet` 函数及相关调用。
  3.  子代理：删除 `src/filetree`, `src/editor`, `tests` 等路径下的空占位目录。
- **验证命令**：`npm run build` && `npm run test`
- **完成定义**：样式变量生效（可通过预览确认），冗余代码移除，目录结构清爽。
- **回滚点**：按文件路径回滚。

| **子代理 (Sub-Agent)**   | 批量文件修复、冗余代码清理、简单样式补全        | 修复 `any` 类型、执行 `prettier` 修复、移除 `greet` 函数 |

## 6. 风险与止损策略

- **风险 1：修复 Lint/Any 导致逻辑变更**
  - 止损：每步修复后必须运行 `npm run test`，确保核心逻辑（Markdown/Workspace）不受影响。
- **风险 2：TS 配置修改导致更多编译错误**
  - 止损：仅针对报错文件进行最小化配置调整，不进行全局性的编译器选项大改。
- **风险 3：删除空目录误删配置文件**
  - 止损：执行删除前使用 `ls -A` 确认目录确实为空。
