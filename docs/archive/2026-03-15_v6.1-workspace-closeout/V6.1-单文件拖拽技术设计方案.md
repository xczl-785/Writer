## 1. 概述

### 1.1 设计目标

将单文件拖拽能力**优雅地**集成到现有 V6 工作区架构中，避免后续重构。

### 1.2 设计原则

| 原则     | 含义                                   |
| -------- | -------------------------------------- |
| 分层清晰 | 服务层与 UI 层分离，拖拽逻辑不侵入组件 |
| 复用现有 | 复用路径获取机制，不重复实现           |
| 可扩展   | 多文件支持只需扩展服务层，UI 层零改动  |
| 类型安全 | 完整的 TypeScript 类型定义             |
| 可测试   | 服务层逻辑独立，便于单元测试           |
| 事务安全 | 多 Store 更新采用快照回滚机制          |

```typescript
  // ========== 状态读取 ==========
  selectedPath: string | null;
  rootFolders: RootFolderNode[];
  hasWorkspace: boolean;

  // ========== 状态写入（回调形式）==========

  onRefreshFileTree: (rootPath: string) => Promise<void>;
  onShowStatus: (type: 'success' | 'error', message: string) => void;

  // ========== 对话框 ==========
  showConflictDialog: (fileName: string) => Promise<'overwrite' | 'cancel'>;

  // ========== 禁用态反馈（新增）==========
  /** 拖拽进入/离开反馈 */
  onSetDragOver: (over: boolean) => void;
  /** 禁用态反馈（拖拽到非接受区域） */
  onSetDropBlocked: (blocked: boolean, reason?: string) => void;

  // ========== 并发控制（新增）==========
  /** 检查是否有未完成的保存操作 */
  isSaving: () => boolean;
}

/**
 * 拖拽结果
 */
export interface DropResult {
  success: boolean;
  error?: DropError;
  openedFile?: string;
}

export type DropError =
  | { type: 'UNSUPPORTED_TYPE'; path: string }
  | { type: 'SAVE_CURRENT_FAILED'; reason: string }
  | { type: 'COPY_FAILED'; source: string; target: string; reason: string }
  | { type: 'OPEN_FAILED'; path: string; reason: string }
  | { type: 'USER_CANCELLED' }
  | { type: 'OPERATION_IN_PROGRESS' } // 新增：操作锁冲突
  | { type: 'SAVE_IN_PROGRESS' }; // 新增：保存中

/**
 * 拖入编辑区 - 直接打开

export async function handleDropToEditor(
  sourcePath: string,
  deps: DropHandlerDeps,
): Promise<DropResult>;

/**
 * 拖入文件树侧边栏 - 复制到工作区
 */
export async function handleDropToSidebar(
  sourcePath: string,
  deps: DropHandlerDeps,
```

### 3.5 异步边界：变更处理决策矩阵

| 当前文件状态  | 用户选择 | 保存结果 | 后续行为                                 |
| ------------- | -------- | -------- | ---------------------------------------- |
| 有未保存内容  | 保存     | 成功     | 打开新文件                               |
| 有未保存内容  | 保存     | 失败     | **保持当前文件，提示错误，不打开新文件** |
| 有未保存内容  | 放弃     | -        | 打开新文件                               |
| 已保存/无变更 | -        | -        | 打开新文件                               |

**实现逻辑**:

```typescript
// App.tsx 拖拽事件处理器中

async function handleFileDrop(sourcePath: string) {
  const { activeFile } = useWorkspaceStore.getState();
  const { fileStates } = useEditorStore.getState();

  // 检查当前文件是否有未保存变更
  if (activeFile && fileStates[activeFile]?.isDirty) {
    const action = await showUnsavedChangesDialog();

    if (action === 'save') {
      const success = await FsSafety.flushAffectedFiles(activeFile);
      if (!success) {
        // 保存失败 → 终止拖拽流程
        useStatusStore.getState().setStatus('error', '保存失败，请重试');
        return;
      }
    }
    // action === 'discard' → 直接继续
  }

  // 继续执行拖拽逻辑
  await handleDropToEditor(sourcePath, deps);
}
```

## 4. 关键设计决策

### 4.1 路径获取：提取为独立工具函数

**问题**: 如何确定文件复制到哪个目录？

**方案**: 将 `MultiRootFileTree.tsx` 中的 Ghost Node 路径判断逻辑提取为纯函数

```typescript
// 新文件：src/domains/file/utils/ghostPathUtils.ts

import type { RootFolderNode } from '../state/fileStore';
import { getParentPath } from '../../../utils/pathUtils';

export interface GetDropTargetOptions {
  selectedPath: string | null;
  rootFolders: RootFolderNode[];
}

/**
 * 获取拖拽目标目录
 * 复用 Ghost Node 逻辑，但解耦为纯函数
 */
export function getDropTargetDirectory({
  selectedPath,
  rootFolders,
}: GetDropTargetOptions): string {
  // 无工作区或无根目录
  if (rootFolders.length === 0) {
    return '';
  }

  // 有选中项
  if (selectedPath) {
    // 判断选中项是文件还是目录
    const isDirectory = selectedPath.endsWith('/');

    if (isDirectory) {
      // 选中文件夹 → 目标 = 选中路径
      return selectedPath;
    } else {
      // 选中文件 → 目标 = 父目录
      return getParentPath(selectedPath);
    }
  }

  // 无选中 → 目标 = 第一个根目录
  return rootFolders[0].workspacePath;
}
```

**优势**:

- 解耦 UI 组件和工具逻辑
- 便于单元测试（无需 Mount 组件）
- 与新建文件/文件夹行为一致

### 4.2 服务层封装：依赖注入解耦

**问题**: 如何避免服务层直接依赖多个 Store？

**方案**: 通过依赖注入将 Store 访问限制在 UI 层

```typescript
// 错误示例（紧耦合）
export function handleDropToSidebar() {
  const selectedPath = fileTreeStore.getState().selectedPath; // ❌
  // ...
}

// 正确示例（松耦合）
export function handleDropToSidebar(deps: DropHandlerDeps) {
  const { selectedPath, rootFolders } = deps; // ✅
  // ...
}

// App.tsx 中调用
const deps: DropHandlerDeps = {
  selectedPath: useFileTreeStore.getState().selectedPath,
  rootFolders: useFileTreeStore.getState().rootFolders,
  hasWorkspace: folders.length > 0,
  onOpenFile: workspaceActions.openFile,
  onRefreshFileTree: async (rootPath) => {
    const nodes = await FsService.listTree(rootPath);
    useFileTreeStore.getState().setNodes(rootPath, nodes);
  },
  onShowStatus: useStatusStore.getState().setStatus,
  showConflictDialog: FileConflictDialog.show,
};

await handleDropToSidebar(sourcePath, deps);
```

**优势**:

- 服务层不感知 Store 存在
- Store 接口变更不影响服务层
- 测试时可注入 Mock 依赖

### 4.4 文件类型：策略模式

**问题**: 如何支持未来扩展文件类型？

**方案**: 策略模式定义文件类型检查

```typescript
// src/domains/file/utils/fileTypeUtils.ts

export interface FileTypePolicy {
  name: string;
  isSupported: (path: string) => boolean;
  getExtensions: () => string[];
  getUnsupportedMessage: (path: string) => string;
}

/**
 * Markdown 文件策略（V6.1）
 */
export const MARKDOWN_POLICY: FileTypePolicy = {
  name: 'Markdown',
  isSupported: (path) => {
    const ext = getPathExtension(path).toLowerCase();
    return ['.md', '.markdown', '.mdx'].includes(ext);
  },
  getExtensions: () => ['.md', '.markdown', '.mdx'],
  getUnsupportedMessage: (path) => {
    const fileName = path.split('/').pop();
    return `"${fileName}" 不是支持的 Markdown 文件类型`;
  },
};

/**
 * 检查文件是否支持
 */
export function isSupportedFile(
  path: string,
  policy: FileTypePolicy = MARKDOWN_POLICY,
): boolean {
  return policy.isSupported(path);
}
```

### 4.5 事务保障：快照回滚

**问题**: 多 Store 更新失败时如何恢复？

**方案**: 参考 `workspaceActions.addFolder` 的快照模式

```typescript
// src/domains/file/services/SingleFileDropHandler.ts

import { useFileTreeStore } from '../state/fileStore';
import { useWorkspaceStore } from '../../workspace/state/workspaceStore';
import { useEditorStore } from '../../editor/state/editorStore';

interface DropSnapshot {
  // ========== 文件树完整状态 ==========
  fileTree: {
    rootFolders: RootFolderNode[];
    expandedPaths: Set<string>;
    selectedPath: string | null;
    loadingPaths: Set<string>;
    errorPaths: Map<string, string>;
    deletedPaths: Set<string>;
  };

  // ========== 工作区状态 ==========
  workspace: {
    folders: WorkspaceFolder[];
    openFiles: string[];
    activeFile: string | null;
    isDirty: boolean;
  };

  // ========== 编辑器状态 ==========
  editor: {
    fileStates: Record<string, FileEditorState>;
    activeFile: string | null;
  };
}

function createSnapshot(): DropSnapshot {
  const fileTreeStoreState = useFileTreeStore.getState();
  const workspaceStoreState = useWorkspaceStore.getState();
  const editorStoreState = useEditorStore.getState();

  return {
    fileTree: {
      rootFolders: fileTreeStoreState.rootFolders,
      expandedPaths: new Set(fileTreeStoreState.expandedPaths),
      selectedPath: fileTreeStoreState.selectedPath,
      loadingPaths: new Set(fileTreeStoreState.loadingPaths),
      errorPaths: new Map(fileTreeStoreState.errorPaths),
      deletedPaths: new Set(fileTreeStoreState.deletedPaths),
    },
    workspace: {
      folders: workspaceStoreState.folders,
      openFiles: workspaceStoreState.openFiles,
      activeFile: workspaceStoreState.activeFile,
      isDirty: workspaceStoreState.isDirty,
    },
    editor: {
      fileStates: { ...editorStoreState.fileStates }, // 浅拷贝
      activeFile: editorStoreState.activeFile,
    },
  };
}

function restoreSnapshot(snapshot: DropSnapshot): void {
  // 文件树恢复（使用原子恢复，避免多次触发渲染）
  useFileTreeStore.getState().setRootFolders(snapshot.fileTree.rootFolders);
  useFileTreeStore.getState().expandedPaths = snapshot.fileTree.expandedPaths;
  useFileTreeStore.getState().selectedPath = snapshot.fileTree.selectedPath;
  useFileTreeStore.getState().loadingPaths = snapshot.fileTree.loadingPaths;
  useFileTreeStore.getState().errorPaths = snapshot.fileTree.errorPaths;
  useFileTreeStore.getState().deletedPaths = snapshot.fileTree.deletedPaths;

  // 工作区恢复
  useWorkspaceStore.getState().restoreState(snapshot.workspace);

  // 编辑器恢复
  useEditorStore.getState().fileStates = snapshot.editor.fileStates;
  useEditorStore.getState().activeFile = snapshot.editor.activeFile;
}

async function executeCopyAndOpen(
  sourcePath: string,
  targetDir: string,
  deps: DropHandlerDeps,
): Promise<DropResult> {
  // ========== 并发控制：操作锁 ==========
  if (isDropOperationInProgress) {
    return {
      success: false,
      error: { type: 'OPERATION_IN_PROGRESS' },
    };
  }

  // 检查是否有保存操作进行中
  if (deps.isSaving()) {
    return {
      success: false,
      error: { type: 'SAVE_IN_PROGRESS' },
    };
  }

  // 设置操作锁
  isDropOperationInProgress = true;
  deps.onSetDragOver(false); // 清除拖拽态

  const snapshot = createSnapshot();

  try {
    // 1. 复制文件
    const copyResult = await FsService.copyFileWithResult(sourcePath, targetPath);

    // 2. 刷新文件树
    await deps.onRefreshFileTree(rootPath);

    // 3. 打开文件（使用实际写入路径）
    await deps.onOpenFile(copyResult.actualPath);

    return { success: true, openedFile: copyResult.actualPath };
  } catch (error) {
    // 回滚快照
    restoreSnapshot(snapshot);

    // 禁用态反馈（2s 后自动解除）
    deps.onSetDropBlocked(true, getErrorMessage(error));
    setTimeout(() => deps.onSetDropBlocked(false), 2000);

    return {
      success: false,
      error: {
        type: 'COPY_FAILED',
        source: sourcePath,
        target: targetPath,
        reason: error instanceof Error ? error.message : String(error),
      },
    };
  }
}
}

// ========== 全局操作锁 ==========
let isDropOperationInProgress = false;
    restoreSnapshot(snapshot);

    return {
      success: false,
      error: {
        type: 'COPY_FAILED',
        source: sourcePath,
        target: targetPath,
        reason: error instanceof Error ? error.message : String(error),
      },
    };
  }
}
```

## 6. 安全考量

### 6.1 后端路径安全校验

```rust
// src-tauri/src/commands/fs.rs

#[tauri::command]
pub async fn copy_file_with_result(
  source: String,
  target: String,
  app_handle: AppHandle,
) -> Result<CopyResult, String> {
  // 1. 路径规范化（防止路径遍历）
  let source = normalize_path(&source)
    .ok_or("Invalid source path")?;
  let target = normalize_path(&target)
    .ok_or("Invalid target path")?;

  // 2. 校验目标路径是否在工作区内
  let workspace = get_current_workspace(&app_handle)?;
  if !is_path_within_workspace(&target, &workspace) {
    return Err("目标路径不在当前工作区内".to_string());
  }

  // 3. 执行复制（保持元数据）
  std::fs::copy(&source, &target)
    .map_err(|e| format!("复制失败：{}", e))?;

  // 4. 返回实际写入路径（处理自动重命名场景）
  let actual_path = target;
  if !std::path::Path::new(&target).exists() {
    // 若目标已存在，尝试自动重命名
    actual_path = generate_unique_path(&target);
    std::fs::copy(&source, &actual_path)
      .map_err(|e| format!("复制失败（重命名）：{}", e))?;
  }

  Ok(CopyResult {
    actual_path,
    bytes_written: std::fs::metadata(&actual_path)
      .map(|m| m.len())
      .unwrap_or(0),
  })
}

/// 生成唯一路径（若文件已存在）
fn generate_unique_path(original: &str) -> String {
  let path = std::path::Path::new(original);
  let parent = path.parent().unwrap_or(std::path::Path::new(""));
  let stem = path.file_stem().unwrap_or_default().to_string_lossy();
  let ext = path.extension().unwrap_or_default().to_string_lossy();

  let mut counter = 1;
  loop {
    let new_name = if ext.is_empty() {
      format!("{} ({})", stem, counter)
    } else {
      format!("{} ({}).{}", stem, counter, ext)
    };
    let new_path = parent.join(&new_name);
    if !new_path.exists() {
      return new_path.to_string_lossy().to_string();
    }
    counter += 1;
  }
}

#[derive(Serialize)]
pub struct CopyResult {
  pub actual_path: String,
  pub bytes_written: u64,
}
```
