# V6.1 单文件打开与拖拽需求说明

**文档类型**: 增量需求说明  
**版本**: V1.1  
**创建日期**: 2026-03-14  
**最后更新**: 2026-03-14  
**状态**: 待排期

## 1. 目标

将"单个 Markdown 文件"的打开与拖拽行为从 V6 工作区能力中拆分出来，作为 V6.1 独立需求推进，避免继续与工作区主链路耦合。

本说明独立描述单文件能力，不依赖 V6 工作区文档作为前置阅读。

## 2. 范围

### 2.1 包含

- `Cmd/Ctrl+O` 打开单文件对话框
- 从系统文件管理器拖拽 `.md/.markdown/.mdx` 文件到应用
  - 拖入编辑区：直接打开文件
  - 拖入文件树侧边栏：复制文件到工作区
- 单文件拖拽（多文件 `TODO`，见 4.3）
- 最近文件记录与打开
- 非 Markdown 文件拖入时的拦截（`TODO: 补充拦截提示`）

### 2.2 不包含

- 工作区内跨目录批量导入
- 工作区外文件的自动归档
- 文件拖拽后的批量重命名/清洗
- V6 既有工作区错误提示系统的统一改造

## 3. 用户场景

### 3.1 单文件快速打开

作为轻量记录者，我希望通过 `Cmd/Ctrl+O` 直接打开任意 Markdown 文件，以便快速查看和编辑，而不必先组织工作区。

### 3.2 单文件拖拽打开

作为轻量记录者，我希望把本地 Markdown 文件直接拖入 Writer 编辑区打开，以便减少对话框操作。

### 3.3 单文件拖拽归档

作为知识管理者，我希望把 Markdown 文件拖入文件树侧边栏，以便快速将文件复制到工作区中并立即编辑。

## 4. 交互定义

### 4.1 打开单文件

| 项目 | 定义                                         |
| ---- | -------------------------------------------- |
| 入口 | `Cmd/Ctrl+O`                                 |
| 入口 | `文件 > 打开文件...`                         |
| 行为 | 弹出文件选择对话框，仅允许选择 Markdown 文件 |
| 结果 | 打开文件并进入编辑状态                       |

---

### 4.2 拖拽单文件

#### 4.2.1 拖入编辑区

| 条件                 | 行为                                    |
| -------------------- | --------------------------------------- |
| 无论当前是否有工作区 | 直接打开该文件                          |
| Markdown 文件        | 直接打开并进入编辑状态                  |
| 非 Markdown 文件     | **拦截**，阻止打开 `TODO: 补充拦截提示` |
| 无工作区时           | 同上，不提示"添加到工作区"              |

#### 4.2.2 拖入文件树侧边栏

| 条件             | 行为                                                              |
| ---------------- | ----------------------------------------------------------------- |
| Markdown 文件    | **复用新建文件/文件夹的路径获取逻辑**，将文件复制到目标路径       |
| 复制后           | **自动打开该文件**进入编辑状态                                    |
| 同名文件冲突     | 弹出确认对话框：`[覆盖]` `[取消]`                                 |
| 非 Markdown 文件 | **拦截**，阻止放置 `TODO: 补充拦截提示`，不显示"添加到工作区"提示 |
| 无工作区时       | 退化为编辑区行为：直接打开文件，不提示"添加到工作区"              |

**路径获取逻辑说明**：

- 直接复用新建文件/文件夹时的路径判断逻辑
- 无需额外判断是否选中目录

## 8. 后续并行收口项

### 8.1 错误提示与通知统一

- 当前 V6 中，拖拽错误、保存失败、权限提示等反馈仍主要复用底部状态栏
- V6.1 需单独梳理"状态栏提示 / 非模态 toast / 错误面板"三类反馈边界
- 目标统一以下场景：
  - 拖拽到不接受的区域
  - 工作区文件保存失败
  - 文件夹权限不足
  - 工作区文件损坏
  - 非 Markdown 文件拖拽（`TODO: T01, T02`）

### 8.2 拖拽禁用态反馈

- 当前 macOS 原生文件拖拽时，系统光标由 Finder / WebView 宿主层控制，前端 `cursor: not-allowed` 不能稳定覆盖
- V6.1 需把"禁用态"定义为应用内视觉反馈，不依赖系统光标是否变化
- 推荐形式：
  - 主编辑区出现轻量禁用蒙层
  - 文案提示"请拖到左侧文件树以添加到工作区"
  - 状态栏或 toast 仅作为次级反馈
