# 技术基线（Step B，合并评审后）

## 1. 输入约束

来源：`docs/frozen/产品约束.md`

- 平台：macOS + Windows（Linux 可选）
- 目标：轻量、快启动、无打断写作
- 范围：V1 Must 12（见 `docs/frozen/V1范围冻结.md`）

## 2. 主方案（冻结）

### 2.1 桌面框架

- `Tauri 2 + Rust`
- 原因：包体和启动速度更符合目标。

### 2.2 前端栈

- `React + TypeScript + Vite`
- 原因：生态成熟、AI 产码稳定、迭代快。

### 2.3 编辑器内核

- `TipTap (ProseMirror)`
- 原因：更适合 Typora 风格交互控制（快捷键、代码块、图片、撤销重做）。

### 2.4 状态管理

- `Zustand`
- 原因：轻量、简单、足够支撑 V1。

### 2.5 文件系统策略

- 统一走文件服务层（UI 不直连文件系统）
- 单工作区（单根目录）
- 文件树实时加载

### 2.6 自动保存与图片

- 自动保存：800ms 防抖 + 失焦保存 + 关闭前强制保存
- 图片：文档同级 `assets/`，命名 `YYYYMMDD-HHMMSS-rand.png`

## 3. 备选方案与不选理由

### 3.1 Electron（备选）

- 优点：生态成熟。
- 当前不选：包体/内存压力较大，不利于轻量目标。

### 3.2 Lexical / Slate（备选）

- 当前不选：V1 阶段相对 TipTap 落地风险更高。

### 3.3 Cherry Markdown（不采用）

- 当前不选：产品形态偏成品编辑器，和目标交互不一致。

## 4. 评审合并（来自 `docs/archive/TECH_BASELINE_REVIEW.md`）

### 4.1 已采纳动作

1. **TipTap Roundtrip Spike（强制）**

- 必须验证 `.md -> TipTap -> .md` 往返语义保持（标题/列表/代码块/图片）。
- 阶段门禁：Step D 结束前必须完成；未完成不得进入 Step E。

2. **原子写入（强制）**

- 文件保存必须使用“写临时文件 -> rename 覆盖”模式，避免崩溃损坏原文件。

3. **Tauri 回退条件（收敛版）**

- 仅在出现 `P0 阻塞` 且满足以下条件时评估回退 Electron：
  1. 可稳定复现；
  2. 两轮修复失败；
  3. 经评审确认。
- 不采用“固定 48 小时无进展即回退”的硬阈值。

### 4.2 回退编辑器策略（修订）

- 不预设“失败后强制切 CodeMirror 6”。
- 若 Spike 失败，进入“备选内核评估”（候选可含 CodeMirror 6 / Lexical），由评审决策。

## 5. 强制工程约束

1. UI 层禁止直接读写文件。
2. 自动保存失败必须可见。
3. 文件写入必须原子化。
4. Step D -> Step E 受 Roundtrip Spike 门禁控制。

## 6. 风险与缓解

1. **Markdown 往返一致性风险**

- 缓解：先定义 V1 语法子集并完成 Spike。

2. **WebView 输入法/剪贴板差异风险**

- 缓解：跨平台用例前置测试；触发回退条件才评估换框架。

3. **多模型并行造成代码边界漂移**

- 缓解：Step D 冻结目录结构、模块边界、提交检查清单。
