# 竞品右键菜单调研报告

**调研日期**: 2026-02-26
**调研目标**: 为 Writer V5 右键菜单设计提供参考依据
**参考产品**: Typora, Obsidian, VSCode, Notion

## 一、竞品右键菜单项汇总

### 1.1 Typora

| 场景         | 菜单项                                                           | 设计特点                 |
| ------------ | ---------------------------------------------------------------- | ------------------------ |
| **未选中**   | • 粘贴<br>• 选择全部<br>• 拼写检查                               | 极简设计，仅基础编辑操作 |
| **选中文本** | • 剪切/复制/粘贴<br>• 粗体/斜体/下划线<br>• 超链接<br>• 内联公式 | 格式操作为主，无层级菜单 |
| **代码块内** | • 复制代码<br>• 代码块语言选择                                   | 专用操作独立分组         |
| **图片上**   | • 复制图片<br>• 图片标题<br>• 图片缩放                           | 图片专用操作             |

**设计特点**:

- 菜单项极少（通常 3-6 项）
- 无嵌套子菜单
- 与写作无关的操作一律不显示
- 符合"无打断"原则

## 三、Writer 右键菜单设计方案

### 3.1 设计原则

1. **最小打断** — 菜单项控制在 8 项以内，避免长菜单
2. **上下文精确** — 根据选中状态/位置动态显示
3. **键盘友好** — 所有菜单项有快捷键/键盘访问路径
4. **可扩展** — 框架支持未来添加菜单项

---

### 3.2 文件树右键菜单

#### 场景 A：右键点击文件

| 菜单项             | 快捷键    | 说明                        | 优先级 |

### 3.4 菜单分组策略

```
文件树菜单 (无子菜单，扁平设计)
├── 打开类
├── 编辑类 (重命名/删除)
└── 新建类 (新建文件/文件夹)

编辑器菜单 (使用分隔线分组，无嵌套)
├── 编辑操作 (剪切/复制/粘贴)
├── 格式操作 (粗体/斜体/删除线)
├── 插入操作 (链接/图片/代码块)
└── 块类型 (标题/列表)
```

## 四、技术实现建议

### 4.1 TipTap 能力

| 能力             | 支持情况    | 说明                                                     |
| ---------------- | ----------- | -------------------------------------------------------- |
| **右键菜单扩展** | ⚠️ 需自定义 | TipTap 无内置右键菜单，需使用 ProseMirror 插件或自行实现 |
| **选区检测**     | ✅ 支持     | `editor.state.selection` 可获取选中状态                  |
| **节点类型检测** | ✅ 支持     | `editor.isActive('heading')` 等 API                      |
| **上下文感知**   | ✅ 可实现   | 基于选区 + 节点类型判断上下文                            |

### 4.2 推荐实现方案

**方案 A: Tauri 原生菜单**

- 优点：性能最佳，系统级体验
- 缺点：Web/Tauri 通信开销，动态菜单复杂

**方案 B: Web 自定义菜单 (推荐)**

- 优点：完全可控，动态菜单容易实现
- 缺点：需要自行处理定位/关闭逻辑
- 技术栈：React + Portal + Popper.js

### 4.3 性能注意

- 菜单渲染延迟 < 50ms
- 菜单位置计算使用 FLIP 动画
- 大量菜单项时虚拟滚动

## 五、决策建议

### 5.1 立即实现 (P0)

1. **文件树右键菜单** — 新建/重命名/删除
2. **编辑器右键菜单 (选中)** — 剪切/复制/粘贴/格式
3. **编辑器右键菜单 (未选中)** — 粘贴/插入操作

### 5.2 推迟实现 (P1/P2)

1. **表格右键菜单** — V6 表格功能完善后
2. **代码块专用菜单** — 使用频率低
3. **评论功能** — 需先实现评论系统

### 5.3 不采用

1. **在系统中显示** — 跳出应用，违背无打断原则
2. **复制路径** — 开发者需求
3. **分割笔记** — 非核心写作场景
