## 二、右键菜单方案

### 选定方案: Web 自定义菜单

**实现方式**: React Portal + contextMenuRegistry

**理由**:

1. UI 规范要求跨平台一致性 (Neutral Modern 风格)
2. 支持动态菜单项 (根据上下文显示/隐藏)
3. `prototypes/01_context_menus.html` 已定义样式规范

**实现要点**:

```typescript
// src/ui/components/ContextMenu/contextMenuRegistry.ts
interface MenuItemConfig {
  id: string;
  label: string;
  shortcut?: string;
  icon?: React.ReactNode;
  danger?: boolean;
  disabled?: () => boolean;
  action: () => void;
}

interface ContextMenuRegistry {
  register(context: string, items: MenuItemConfig[]): void;
  unregister(context: string): void;
  getItems(context: string): MenuItemConfig[];
}
```

**样式规范** (来自 `prototypes/01_context_menus.html`):

| 元素     | Tailwind 类名                                                                                |
| -------- | -------------------------------------------------------------------------------------------- |
| 菜单容器 | `bg-white dark:bg-zinc-800 border border-zinc-200 dark:border-zinc-700 shadow-xl rounded-md` |
| 菜单项   | `px-3 py-1.5 text-sm text-zinc-700 hover:bg-zinc-100`                                        |
| 危险项   | `text-red-500 hover:bg-red-50`                                                               |
| 快捷键   | `text-xs text-zinc-400 font-mono`                                                            |
| 分隔线   | `h-[1px] bg-zinc-100 my-1 mx-2`                                                              |

**键盘导航**:

- `ArrowUp` / `ArrowDown`: 切换菜单项
- `Enter`: 执行选中项
- `Escape`: 关闭菜单

## 三、Bubble Menu 方案

### 选定方案: TipTap BubbleMenu 扩展

**实现方式**: `@tiptap/extension-bubble-menu`

**理由**:

1. 官方扩展，与 TipTap 内核集成度高
2. 基于 Tippy.js，定位和动画成熟
3. 满足 V5 需求 (加粗/斜体/链接等格式化)

**实现要点**:

```typescript
import { BubbleMenu } from '@tiptap/extension-bubble-menu';

const bubbleMenuExtension = BubbleMenu.configure({
  element: document.querySelector('#bubble-menu'),
  tippyOptions: {
    duration: 150,
    placement: 'top',
    animation: 'shift-away',
  },
});
```

**菜单项** (来自 `prototypes/01_context_menus.html`):

| 操作     | 图标      | 快捷键 |
| -------- | --------- | ------ |
| 加粗     | B         | ⌘B     |
| 斜体     | I         | ⌘I     |
| 删除线   | S         | ⇧⌘X    |
| 行内代码 | `</>`     | ⌘E     |
| 超链接   | Link      | ⌘K     |
| 高亮     | Highlight | -      |

**样式规范**:

| 元素     | Tailwind 类名                                                        |
| -------- | -------------------------------------------------------------------- |
| 菜单容器 | `bg-zinc-800 text-white shadow-lg rounded-md border border-zinc-700` |
| 按钮     | `p-1.5 rounded hover:bg-zinc-700 text-zinc-300 hover:text-white`     |
| 分隔线   | `w-[1px] h-4 bg-zinc-600 mx-1`                                       |

## 四、删除确认对话框方案

### 选定方案: Tauri 原生对话框

**实现方式**: `@tauri-apps/plugin-dialog`

**理由**:

1. 文件删除是破坏性操作，用户预期系统级体验
2. 阻断性强，防止误操作
3. 与 macOS Finder / Windows Explorer 体验一致

**实现要点**:

```typescript
import { ask } from '@tauri-apps/plugin-dialog';

async function confirmDelete(filename: string): Promise<boolean> {
  const confirmed = await ask(`确定要将 "${filename}" 移至废纸篓吗？`, {
    title: '移至废纸篓',
    kind: 'warning',
    okLabel: '移至废纸篓',
    cancelLabel: '取消',
  });
  return confirmed;
}
```

**降级策略**: Web 模式下使用自定义 Dialog 组件

## 五、错误提示方案

### 选定方案: 非阻断 Tooltip

**实现方式**: 状态栏指示器 + 悬停详情面板

**理由**:

1. UI 规范要求 "无打断写作主流程优先"
2. 错误面板可嵌入操作按钮 (如 "另存为")
3. 状态栏收敛所有状态反馈，符合 "唯一性" 要求

**实现要点**:

```typescript
// src/ui/components/StatusIndicator/statusMachine.ts
type SaveStatus = 'saved' | 'dirty' | 'saving' | 'error';

interface StatusState {
  status: SaveStatus;
  message?: string;
  errorDetails?: {
    reason: string;
    suggestion: string;
    actions: { label: string; action: () => void }[];
  };
}
```

**样式规范** (来自 `prototypes/02_status_indicator.html`):

| 状态   | 样式                                   |
| ------ | -------------------------------------- |
| 已保存 | `bg-emerald-500` 绿色圆点              |
| 未保存 | `bg-amber-400` 黄色实心                |
| 保存中 | `bg-blue-500 animate-pulse` 蓝色呼吸灯 |
| 错误   | `bg-red-500` 红色 + 悬停面板           |

**稳态隐匿**: 已保存状态超过 5 秒后，透明度降至 30%

## 六、大纲虚拟滚动方案

### 选定方案: 按需启用

**阈值策略**:

| 大纲项数量 | 渲染策略                             |
| ---------- | ------------------------------------ |
| < 100      | 全量渲染                             |
| 100-500    | 防抖更新 (150ms)                     |
| > 500      | 虚拟滚动 (`@tanstack/react-virtual`) |

**理由**:

1. 大多数文档大纲项 < 100，虚拟滚动收益有限
2. 减少小文档的额外开销
3. SPIKE-003 将验证阈值准确性
