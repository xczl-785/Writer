## 一、性能预算总览

### 1.1 核心原则

> "无打断写作主流程优先" — 所有交互响应必须在用户感知阈值内完成

| 感知等级 | 时间范围   | 用户感受         |
| -------- | ---------- | ---------------- |
| 即时     | < 100ms    | 用户感觉直接操控 |
| 流畅     | 100-300ms  | 用户感觉平滑过渡 |
| 可接受   | 300-1000ms | 用户注意到延迟   |
| 需反馈   | > 1000ms   | 需要进度指示     |

### 1.2 V5 性能预算表

| 操作类别     | 操作项             | 预算目标 | 测量方法                    |
| ------------ | ------------------ | -------- | --------------------------- |
| **交互响应** | 右键菜单显示       | < 50ms   | contextmenu 事件到 DOM 渲染 |
|              | Bubble Menu 显示   | < 100ms  | 选区确定到菜单渲染          |
|              | 大纲点击跳转       | < 100ms  | 点击到滚动完成              |
|              | 状态指示器切换     | < 300ms  | 状态变更到视觉更新          |
|              | Tooltip 显示       | < 400ms  | 悬停开始到显示              |
| **文档处理** | 大纲提取 (5000行)  | < 100ms  | editor.state.doc 遍历完成   |
|              | 大纲提取 (10000行) | < 500ms  | editor.state.doc 遍历完成   |
|              | 自动保存触发       | < 50ms   | 防抖计时器到写入开始        |
| **运行时**   | 内存增量           | < 10MB   | V5 新增组件内存占用         |
|              | 首屏渲染           | 不增加   | V5 功能懒加载               |

## 二、交互性能指标

### 2.1 右键菜单 (INT-002, INT-010, INT-011, INT-012)

| 阶段       | 预算       | 说明                      |
| ---------- | ---------- | ------------------------- |
| 事件捕获   | < 5ms      | preventDefault + 阻止冒泡 |
| 菜单项计算 | < 20ms     | 根据上下文过滤菜单项      |
| DOM 渲染   | < 25ms     | Portal 挂载 + 样式计算    |
| **总计**   | **< 50ms** |                           |

**测试方法**:

```typescript
const start = performance.now();
element.dispatchEvent(new MouseEvent('contextmenu'));
requestAnimationFrame(() => {
  const duration = performance.now() - start;
  console.log(`菜单显示耗时: ${duration}ms`);
});
```

### 2.2 Bubble Menu (INT-011)

| 阶段         | 预算        | 说明                     |
| ------------ | ----------- | ------------------------ |
| 选区检测     | < 20ms      | selectionchange 事件处理 |
| 定位计算     | < 30ms      | Tippy.js 定位算法        |
| 动画渲染     | < 50ms      | 150ms 动画，首帧 < 50ms  |
| **首帧总计** | **< 100ms** |                          |

### 2.3 大纲导航 (INT-015)

| 场景            | 预算       | 策略                                              |
| --------------- | ---------- | ------------------------------------------------- |
| 点击跳转        | < 100ms    | editor.commands.setTextSelection + scrollIntoView |
| 实时更新 (防抖) | 150ms 防抖 | 编辑后延迟更新                                    |
| 视口跟随        | < 150ms    | 滚动时防抖高亮当前项                              |

### 2.4 状态指示器 (INT-014)

| 状态切换       | 预算    | 视觉效果            |
| -------------- | ------- | ------------------- |
| saved → dirty  | < 100ms | 绿 → 黄圆点         |
| dirty → saving | < 100ms | 黄 → 蓝呼吸灯       |
| saving → saved | < 300ms | 蓝 → 绿 + 5s 后淡出 |
| saving → error | < 100ms | 蓝 → 红 + 面板      |

## 六、性能测试方案

### 6.1 自动化测试

```typescript
// 性能测试示例
describe('ContextMenu Performance', () => {
  it('should render within 50ms', async () => {
    const start = performance.now();

    fireEvent.contextMenu(fileElement);

    await waitFor(() => {
      const menu = screen.getByRole('menu');
      expect(menu).toBeInTheDocument();
    });

    const duration = performance.now() - start;
    expect(duration).toBeLessThan(50);
  });
});
```

### 6.3 性能监控

**开发阶段**: React DevTools Profiler + Chrome DevTools Performance

**生产阶段**: 建议集成 Web Vitals 监控

```typescript
// 可选：性能监控埋点
const measureInteraction = (name: string, fn: () => void) => {
  const start = performance.now();
  fn();
  const duration = performance.now() - start;

  if (duration > 100) {
    console.warn(`[Perf] ${name} took ${duration}ms`);
  }
};
```
