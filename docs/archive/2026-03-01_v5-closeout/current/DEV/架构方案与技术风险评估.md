### 4.2 核心接口设计

#### 4.2.1 右键菜单注册中心

```typescript
// src/ui/components/ContextMenu/contextMenuRegistry.ts

interface MenuItemConfig {
  id: string;
  label: string;
  shortcut?: string;
  icon?: React.ReactNode;
  danger?: boolean; // 危险项 (红色样式)
  disabled?: () => boolean; // 动态禁用
  action: () => void;
}

interface ContextMenuRegistry {
  register(context: string, items: MenuItemConfig[]): void;
  unregister(context: string): void;
  getItems(context: string): MenuItemConfig[];
}
```

#### 4.2.2 状态指示器状态机

```typescript
// src/ui/components/StatusIndicator/statusMachine.ts

type SaveStatus = 'saved' | 'dirty' | 'saving' | 'error';

interface StatusState {
  status: SaveStatus;
  message?: string;
  errorDetails?: {
    reason: string;
    suggestion: string;
    actions: { label: string; action: () => void }[];
  };
}

// 状态流转:
// saved --(edit)--> dirty --(save)--> saving --(success)--> saved
//   ^                                           |
//   +-----------------(error)-------------------+
//                     |
//                     v
//                   error
```

#### 4.2.3 大纲提取器

```typescript
// src/ui/components/Outline/useOutlineExtractor.ts

interface OutlineItem {
  id: string;
  level: 1 | 2 | 3 | 4 | 5 | 6;
  text: string;
  position: number; // 文档位置
}

interface OutlineExtractor {
  extract(editor: Editor): OutlineItem[];
  subscribe(callback: (items: OutlineItem[]) => void): () => void;
}
```

## 五、技术选型决策

### 5.1 右键菜单实现方案

| 方案                  | 优点                                 | 缺点                           | 决策        |
| --------------------- | ------------------------------------ | ------------------------------ | ----------- |
| **A: Web 自定义菜单** | 完全可控、与 UI 规范一致、跨平台统一 | 需自行处理键盘导航             | ⭐ **推荐** |
| B: Tauri 原生菜单     | 系统级体验                           | 跨平台样式不一致、动态菜单复杂 | 备选        |

**决策理由**:

- UI 规范要求跨平台一致性 (Neutral Modern 风格)
- `prototypes/01_context_menus.html` 已定义样式
- 动态菜单项 (根据上下文显示/隐藏) 更易实现

### 5.2 对话框实现方案

| 场景         | 方案             | 理由                             |
| ------------ | ---------------- | -------------------------------- |
| 文件删除确认 | Tauri 原生对话框 | 系统级安全操作，用户预期原生体验 |
| 错误详情面板 | Web 自定义       | 非阻断、可嵌入操作按钮           |

**决策**: 采用混合策略

### 5.3 大纲面板性能方案

| 方案     | 适用场景         | 决策     |
| -------- | ---------------- | -------- |
| 全量遍历 | < 1000 行文档    | 默认方案 |
| 增量更新 | 1000-5000 行文档 | 过渡方案 |
| 虚拟滚动 | > 5000 行文档    | 按需启用 |

**决策**: 先实现全量遍历，在 SPIKE-003 中验证是否需要虚拟滚动

## 六、风险评估与缓解措施

### 6.1 技术风险

| 风险项                    | 等级 | 缓解措施                                         |
| ------------------------- | ---- | ------------------------------------------------ |
| TipTap 扩展与现有代码冲突 | 高   | **强制门禁**: 所有扩展在 `spike/` 验证后方可合并 |
| 右键菜单与系统菜单冲突    | 中   | `preventDefault()` + Tauri 原生降级备选          |
| 长文档大纲性能塌陷        | 中   | SPIKE-003 验证 + 按需虚拟滚动                    |
| 全局快捷键碰撞            | 中   | 建立 `ShortcutRegistry` + 冲突检测               |

### 6.2 业务风险

| 风险项         | 等级 | 缓解措施                                 |
| -------------- | ---- | ---------------------------------------- |
| 功能范围蔓延   | 中   | 严格遵守 PM 冻结需求，新增需求走变更流程 |
| 跨平台交互差异 | 低   | UI 规范已定义 Neutral Modern 风格        |

## 七、性能预算

### 7.1 交互响应指标

| 操作             | 目标    | 测量方法                      |
| ---------------- | ------- | ----------------------------- |
| 右键菜单显示     | < 50ms  | `contextmenu` 事件到 DOM 渲染 |
| Bubble Menu 显示 | < 100ms | 选区确定到菜单渲染            |
| 大纲跳转定位     | < 100ms | 点击到滚动完成                |
| 状态指示器切换   | < 300ms | 状态变更到视觉更新            |

### 7.2 运行时性能指标

| 指标           | 目标    | 测量方法            |
| -------------- | ------- | ------------------- |
| 长文档大纲提取 | < 100ms | 5000 行文档全量遍历 |
| 内存占用增量   | < 10MB  | V5 新增组件内存占用 |
| 首屏渲染       | 不增加  | V5 功能懒加载       |
