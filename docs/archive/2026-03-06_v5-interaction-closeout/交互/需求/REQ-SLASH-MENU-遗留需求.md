# REQ-SLASH-MENU-遗留需求

## 1. 目标

收敛 Slash Menu 当前未闭环问题，形成可独立排期与验收的需求基线。

## 2. 需求来源

1. `docs/current/交互/V5交互测试用例与需求映射.md`（模块 E）
2. `docs/全局资产/技术/Slash菜单设计.md`
3. `docs/current/交互/方案/Slash Menu重做规格文档.md`
4. 遗留迁移记录：`docs/current/跟踪/遗留问题.md`

## 3. 遗留范围（当前待闭环）

1. `L-E-01 / TC-E02`：菜单向上翻转时遮挡输入行。
2. `L-E-04 / TC-E09`：向上翻转场景复现输入行遮挡。
3. `L-E-02 / TC-E03`：小窗内滚时，键盘选择项不跟随滚动。
4. `L-E-03 / TC-E05`：菜单未完整显示时，↑/↓ 不驱动可见区跟随。

## 4. 合并处理组

1. `G-01`：`L-E-01 + L-E-04`（上翻定位与输入行安全间隔）
2. `G-02`：`L-E-02 + L-E-03`（键盘导航与菜单滚动联动）

## 5. 验收用例

1. `TC-E02`、`TC-E09`（上翻不遮挡输入行）
2. `TC-E03`、`TC-E05`（键盘导航可见区跟随）
3. 回归：`TC-E01`、`TC-E04`、`TC-E06`、`TC-E07`、`TC-E08`

## 6. 非目标

1. 不修改 Typewriter 锚点算法。
2. 不修改 Focus Zen 退出优先级策略。
