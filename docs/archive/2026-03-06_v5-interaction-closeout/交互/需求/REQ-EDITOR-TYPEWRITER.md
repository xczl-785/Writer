# REQ-EDITOR-TYPEWRITER 打字机模式需求

## 1. 目标

定义打字机模式的锚点行为、激活条件、状态联动与稳定性，确保“光标趋近 45% 视点”可稳定复现。

## 2. 需求来源

1. `docs/current/交互/方案/UI设计详细交付文档.md` §0, §4.1, §4.2
2. `docs/current/交互/方案/适配与尺寸方案.md` §2
3. `docs/current/交互/V5交互测试用例与需求映射.md` R-04, R-05, TC-D01~TC-D10
4. `docs/archive/2026-03-06_v5-test-leftovers-closeout/V5交互测试遗留清单.md` L-D-01, G-03

## 3. 范围

1. 菜单与设置的同源开关行为。
2. 光标锚点滚动行为。
3. 与 Zen / Focus Zen 的进入与恢复联动。
4. 长文、短文、方向键移动、中文输入法场景。

## 4. 规则

1. 模式定义：开启后，光标行应趋近视口纵向 `45%` 锚点。
2. 全局双态：菜单与设置开关必须同源，任一路径切换后状态一致。
3. Zen 联动：
   1. 进入 Zen 时强制开启打字机。
   2. 退出 Zen 时恢复进入前用户偏好。
4. 短文保护：当内容高度不足时，不应触发强制滚动锁定。
5. 稳定性：连续输入、方向键移动、中文输入法候选过程中不得出现明显抖动或回跳。

## 5. 验收标准

1. `TC-D01`：长文开启后，光标明显趋近 `45%` 锚点（关闭遗留 `L-D-01`）。
2. `TC-D02`：关闭后停止锚点滚动。
3. `TC-D03`：菜单与设置开关同步。
4. `TC-D04~TC-D06`：Zen 强制开启与退出恢复链路正确。
5. `TC-D07`：短文不误触发强制滚动。
6. `TC-D08~TC-D09`：输入法与方向键场景仍保持稳定锚点行为。

## 6. 协作约束

1. 不得破坏 Esc 优先级（先处理活跃浮层）。
2. 不得修改 Slash Menu 定位坐标来源。
