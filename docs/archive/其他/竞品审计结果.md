## 二、V1 功能清单（Must 汇总，收敛版）

为匹配“高频主流程优先”，V1 Must 收敛为 12 项：

### 编辑核心能力

1. **即时渲染编辑** - 所见即所得核心体验
2. **段落级格式快捷键** - 标题/列表等高频结构化输入
3. **行内样式快捷键** - 加粗/斜体等高频标注
4. **代码块编辑** - 技术写作常用结构
5. **图片粘贴与本地保存** - 截图即贴即用
6. **自动保存** - 无感防丢失
7. **撤销/重做** - 基础编辑安全网

### 文件管理能力

8. **侧边栏文件树** - 目录可视化定位
9. **应用内新建文件** - 不离开应用即可创建文档
10. **文件重命名** - 文件管理基础动作
11. **文件删除** - 文件管理基础动作
12. **文件夹作为工作区** - 打开目录即开始写作

## 备注：其余原 Must 项下调为 Should，用于 Phase 2 体验完善。

## 三、V1 边界清单（明确 Out）

以下功能明确不在 V1 范围内：

| 功能                         | 排除理由                      |
| ---------------------------- | ----------------------------- |
| 插件生态                     | 高复杂度，与核心写作无关      |
| 双向链接、知识图谱           | 知识管理扩展，非写作刚需      |
| 云同步、多人协作             | 基础设施依赖，V1 聚焦单机体验 |
| 账号系统                     | 与核心功能无关                |
| 分屏预览（独立预览模式）     | 与即时渲染冲突，增加复杂度    |
| Vim/Emacs 键位               | 小众需求，增加输入层复杂度    |
| 流程图/图表支持（Mermaid等） | 可用图片替代，非核心需求      |
| 表格公式计算                 | 偏离写作工具定位              |
| 智能引号/破折号              | 中文场景使用率低              |
| 文件标签/属性系统            | 与无打断写作流程无关          |

## 四、待确认问题

1. **工作区模式**：是否支持同时打开多个工作区（文件夹），还是单工作区+多标签页即可？
2. **图片存储策略**：图片是强制存放到文档同级 assets 目录，还是允许用户配置统一图片目录？
3. **自动保存频率**：是实时保存（每次输入后）还是定时保存（如 5 秒）？是否需要显示保存状态？
4. **标签页上限**：是否限制同时打开的标签页数量（防止性能问题）？
5. **搜索范围**：快速文件搜索是否支持拼音/模糊匹配？是否需要全文搜索的 Must 级别？

## 五、优先级分层建议

### Phase 1（V1 Must，先做）

- 即时渲染编辑
- 段落级/行内格式快捷键
- 代码块编辑
- 图片粘贴与本地保存
- 自动保存 + 撤销/重做
- 侧边栏文件树 + 新建文件 + 重命名/删除
- 文件夹作为工作区

### Phase 2（Should，体验完整）

- 表格可视化编辑
- 大纲/目录导航
- 标签页切换、快速文件搜索、最近文件列表
- 查找/替换
- 新建文件夹、文件移动
- 导出 PDF、拼写检查、字数统计、主题/字体

### Phase 3（Could，体验优化）

- 专注/打字机/禅模式
- 脚注、目录自动生成、表情等扩展写作能力
- 其他 Could 级别功能
