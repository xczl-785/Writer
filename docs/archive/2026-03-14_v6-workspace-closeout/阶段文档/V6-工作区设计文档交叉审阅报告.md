## 2. Critical 问题汇总（必须修正）

### 🔴 CR-01: 模态对话框 vs 无打断流原则

**发现问题**: V6 交互规范 3.4.2"外部文件打开询问"使用模态对话框，与全局原则"无打断流"冲突。

**位置**: V6 交互规范 3.4.2

**冲突详情**:

- 全局原则: "无打断流：界面的反馈（如设置、通知）不应中断用户的'造物心流'"
- V6 规范: "形式: 模态对话框（需用户决策）"

**必须修正**:

```
方案 A（推荐）: 非模态侧边通知栏
- 默认行为: 直接打开文件，状态栏提示
- 首次询问: 仅首次遇到时询问，后续记住选择
- 严格模式: 在设置中开启"始终询问"

方案 B（折中）: 模态对话框 + 默认行为配置
- 保留模态对话框，但在设置中提供默认行为选项
- 用户可配置"默认打开"或"默认复制到工作区"
```

---

### 🔴 CR-02: 强调色对比度不足

**发现问题**: `blue-500 (#3b82f6)` 在白色背景上对比度约 3.9:1，低于 WCAG AA 标准 4.5:1。

**位置**: V6 UI 任务清单 3.1 颜色令牌

**必须修正**:

```css
/* 当前 */
--color-accent: #3b82f6 (blue-500); /* 3.9:1 对比度 */

/* 建议改为 */
--color-accent: #2563eb (blue-600); /* 5.1:1 对比度 - 符合 WCAG AA */
--color-accent-subtle: #dbeafe (blue-100); /* 选中背景 */
```

---

### 🔴 CR-03: 设计令牌命名体系分裂

**发现问题**: 交互规范使用语义化命名（`--workspace-*`），UI 任务清单使用通用命名（`--color-*`）。

**位置**:

- V6 交互规范 8.2
- V6 UI 任务清单 3

**必须修正**: 统一采用通用设计令牌体系：

| 当前（交互规范）            | 统一后（推荐）          |
| --------------------------- | ----------------------- |
| `--workspace-root-bg-hover` | `--color-surface-hover` |
| `--workspace-divider`       | `--color-border`        |
| `--workspace-highlight`     | `--color-accent`        |
| `--workspace-root-padding`  | `--spacing-sm`          |
| `--workspace-indent`        | `--spacing-md`          |

**缺失内容**:

## 8. 审阅结论

### 8.1 整体评估

**三位设计师一致结论**: 🟡 **有条件通过**

文档框架完整，核心设计理念与全局原则基本对齐。但存在 3 个 Critical 问题（模态对话框、对比度、令牌命名）和 5 个 High 问题必须在视觉设计开始前修正。

### 8.2 关键决策建议

| 决策           | 建议                     | 理由                      |
| -------------- | ------------------------ | ------------------------- |
| 模态对话框降级 | 改为非模态通知或配置选项 | 符合"无打断流"核心原则    |
| 强调色升级     | blue-500 → blue-600      | 满足 WCAG AA 可访问性标准 |
| 令牌体系统一   | 采用通用命名体系         | 避免技术债务，便于维护    |
| 拖拽反馈补充   | 新增 DragFeedbackStates  | 确保设计完整性            |
