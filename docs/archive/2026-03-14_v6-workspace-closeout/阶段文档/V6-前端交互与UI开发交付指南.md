## 2. 隐性交互状态说明 (IxD State Machine)

静态 HTML 无法完全表达微交互事件，开发落地时必须补充实现以下状态流转：

### 2.1 文本溢出与悬浮提示 (Overflow & Truncate)
*   **侧边栏宽度**：固定且可调整（基础宽度 `w-64` / 256px）。
*   **截断处理**：当文件名（包括深层级缩进后）超出可用宽度时，强制使用 `truncate` (即 `white-space: nowrap; overflow: hidden; text-overflow: ellipsis;`)。
*   **气泡补偿**：只要文本发生截断，鼠标悬停 (`Hover`) 该元素超过 `400ms`，必须唤出系统级的或者自定义的 Tooltip 完整显示文件名及后缀。

### 2.2 多根文件树边界交互
*   **Hover 显隐**：根目录末尾的移除按钮 `[x]` 在日常状态下**必须**为 `opacity-0`，仅当鼠标 `group-hover` 于这一栏头时才变为 `opacity-100`。
*   **禁止变色反馈**：注意！根目录自身是一个“容器盖板”，它的 `stroke` 线框图标永远是 `text-zinc-600`，不管是否被 Hover 或者其下的文件是否被选中，**根目录图标和文字永不高亮为蓝色**。

### 2.3 拖拽动作反馈 (Drag & Drop)
*   拖拽文件进入空状态时的提示面板：背景叠加 `bg-white/80` 磨砂毛玻璃效果（`backdrop-blur-sm`），并居中显露出“释放以打开该工作区”。

## 3. 图标资产集成规范 (Iconography)

为了摆脱旧系统笨重的 SVG 硬编码，V6 已全链路切换为 **Lucide Icons** 开源库。
*   **引入方式**：使用轻量级的包引用或者按需加载 svg（参考原型文件头部引入的 `https://unpkg.com/lucide@latest`）。
*   **关键图标对应映射**：
    *   普通文件：`file-text` / `lucide-file`
    *   普通文件夹：`folder`
    *   折叠/展开箭头：`chevron-right`, `chevron-down`
    *   打开动作：`folder-plus`, `file-plus`
*   **注**：图标大小统一设定为 `w-4 h-4` (16x16px)。

## 4. 业务边缘与异常处理 (Edge Cases)

| 触发场景 | 交互表现与视觉处理 |
| :--- | :--- |
| **打开工作区时配置文件损坏** | 界面不可崩溃。底部状态栏需闪烁黄灯 `bg-yellow-500`，并在界面中央降级为带警告图标的 Empty State 面板。 |
| **拖入不兼容的非文本文件（如视频）** | 判定触发拖拽 Reject。指针变为 `not-allowed` 样式，停止在工作区侧边栏提供任何蓝晕的高亮反馈。 |
| **工作区内的根目录之一在磁盘被物理删除** | 列表不清空，但该目录名变为 `text-zinc-400` 并附带中划线 `line-through`，并在右侧强制常驻显示 `[x]` 提醒用户手动一键移除无效项。 |
| **未命名工作区主动退出** | 不使用系统强中断模态框（Modal）。采用界面底部非模态通知条（Toast Notification）轻提示是否需要保存该多组结构。 |

## 5. UI/UX 发版验收清单 (QA Checklist)

当你的 PR 准备合并时，请对照执行以下走查：

1.  **[ ] 第一视觉对比**：整个工作区的背景是否是高级的 `zinc-50` (#FAFAFA)，且和主编辑区的纯白 (`bg-white`) 形成了清晰但柔和的空间分离？
2.  **[ ] 指示器位置**：任意选中一个带有深层级缩进的文件（如第3层级），其左侧的蓝色长条 `w-[3px]` 是否精准贴靠在了最左侧垂直边缘，而**没有**跟着文字一起向右缩进跑偏？
3.  **[ ] 根目录分割**：当存在两个以上的独立项目区时，后续项目区与上方最后一个文件之间是否严格保持了 `mt-4` (`16px`) 的留白排版以及一条用于防眩晕的极淡的分割线？
4.  **[ ] 字体质感**：侧边栏选中文件是否呈现字重较重的 `font-medium` (500)？常规未选中文件是否是安静的 `text-zinc-500`？

如果你在开发中发现任何影响性能或与底层框架冲突的样式，请优先按照本指南精神进行优雅降级，并在 Issue 中 @ 设计团队同步决策。
