# V2 技术基线

## 1. 目标

为 V2 功能提供可落地、可验证、可回滚的技术边界。

## 2. 关键技术议题

1. 表格节点的数据模型与序列化策略。
2. 编辑器扩展兼容性与回归风险控制。
3. 搜索性能与大工作区下的响应上限。

## 2.1 执行优先级（非稳定性）

1. P1：编辑命令统一层（快捷键与顶部菜单共用命令入口）。
2. P1：表格节点最小模型（插入/基础编辑/序列化回写）。
3. P2：工作区搜索索引最小实现（按文件名 + 内容片段）。
4. P2：查找/替换与编辑器状态桥接（不破坏自动保存链路）。

## 2.2 可回滚策略

1. 顶部菜单通过特性开关接入，异常时可降级为仅快捷键。
2. 表格能力按节点级开关控制，异常时回退到纯 Markdown 输入。
3. 搜索功能独立模块化，异常不影响编辑主链路。

## 3. 强制约束

1. 不破坏 V1 Markdown Roundtrip 稳定性。
2. 新能力必须有失败可见反馈。
3. 每个技术决策必须记录风险与回退方案。
