# Gemini UI/UX 方案 (UI/UX 交付提案)

## 1. 设计概述

### 1.1 设计理念： "Invisible Fluidity" (隐形流动)
我们不仅仅在设计一个编辑器，而是在构建一个**思维的各种容器**。核心理念是**“隐形”**——UI 应尽可能消隐，只在需要时通过流畅的**微交互**浮现。

*   **Premium (高级感)**：拒绝粗糙的工具感。使用完美的排版韵律和精心调配的色彩，传达“精密仪器”的质感。动效仅用于必要的逻辑衔接，极速且克制。
*   **Focus (专注)**：内容是绝对主角。侧边栏、状态栏在视觉上大幅弱化，通过透明度和低对比度融入背景。
*   **Tactile (触感)**：把操作（如新建、保存、拖拽）做得有物理反馈感（拟态或弹性动画），让写作过程像在雕刻。

### 1.2 视觉方向 (Moodboard)

#### 色彩体系 (Colors)
采用 **HSL 动态配色**，确保在不同光照下的和谐度。

**Light Mode: "Morning Mist" (晨雾)**
*   **背景**: `HSL(210, 20%, 98%)` (冷白，非纯白，护眼且高级)
*   **侧边栏**: `HSL(210, 15%, 95%)` (半透明毛玻璃效果)
*   **文字 (主)**: `HSL(210, 20%, 20%)` (深岩灰)
*   **文字 (次)**: `HSL(210, 10%, 50%)`
*   **强调色**: `HSL(205, 90%, 45%)` (克莱因蓝变体，用于光标和选中态)
*   **错误/警示**: `HSL(350, 80%, 60%)` (柔和珊瑚红)

**Dark Mode: "Deep Space" (深空)**
*   **背景**: `HSL(220, 25%, 12%)` (深邃蓝黑，非纯黑 OLED)
*   **侧边栏**: `HSL(220, 20%, 10%)`
*   **文字 (主)**: `HSL(220, 10%, 90%)` (星光银)
*   **文字 (次)**: `HSL(220, 10%, 55%)`
*   **强调色**: `HSL(205, 100%, 65%)` (荧光蓝)

#### 字体排印 (Typography)
*   **UI 字体**: `Inter`, `San Francisco` (macOS), `Segoe UI` (Windows)。字重克制，多用 Regular (400) 和 Medium (500)。
*   **正文 (Editor)**: 默认使用 `Merriweather` 或 `Source Serif 4`。衬线体能显著提升长文阅读的沉浸感。
*   **代码 (Code)**: `JetBrains Mono` 或 `Fira Code`。支持连字特性。
*   **韵律**: 基准行高 `1.75`，段间距 `1.5em`。标题与正文的比例遵循黄金分割 `1.618`。

## 2. 页面结构与布局 (Layout)

### 2.1 整体框架 (The Frame)
*   **原生窗口 (Native Window)**: V1 保留系统原生 Titlebar 以确保跨平台窗口拖拽与控制的稳定性。
*   **视觉融合**: 尽量让 Titlebar 背景色与 Sidebar 一致，在视觉上弱化边界。

### 2.2 区域划分
1.  **侧边栏 (Sidebar) - 宽度 260px (Resziable)**
    *   **顶部**: 只有红绿灯和当前“工作区名称”（文件夹名）。字体极小 (11px)，全大写，字母间距宽松 (Letter-spacing: 1px)。
    *   **中部**: 文件树。利用缩进和连接线（可选隐形）展示层级。
    *   **底部**: 极其隐蔽的“新建文件”触发区（或悬浮按钮）。

2.  **编辑核心 (Editor Core) - 弹性宽度 (Max 800px)**
    *   居中悬浮于主视图。
    *   **Padding**: 顶部留白 120px (给思考留空间)，底部留白 300px (允许滚动超过最后一行)。

3.  **隐形状态栏 (Status Layer) - 底部 Fixed**
    *   仅在有状态变化（保存中、错误）时浮现，平时完全透明。
    *   右下角极简显示：`Word Count` (如需) / `Saved` 状态。

## 3. 关键交互流程细化 (Micro-interactions)

### 3.1 打开工作区 (Onboarding)
*   **初始态**: 界面中央只有一个极简的 Logo 和一行字 "Open a folder to start writing"。
*   **动作**: 
    1.  点击按钮或拖入文件夹。
    2.  **动画**: 侧边栏从左侧平滑滑出 (Slide In)，编辑器背景淡入 (Fade In)。无突兀跳转。

### 3.2 文件树与新建 (File Tree & Creation)
*   **Hover 效果**: 鼠标滑过文件项时，背景显现极淡的圆角矩形 (`border-radius: 6px`)，左侧出现微小的“指示条”。
*   **新建文件**:
    1.  点击侧栏顶部 `+` 号或文件夹右键。
    2.  **动画**: 文件树列表中即时推开一个空位 (Height expand animation)，出现输入框。
    3.  **焦点**: 光标直接进入输入框，全选默认文件名 "Untitled"。

### 3.3 重命名 (Renaming)
*   **交互**: 双击文件名（慢速）或右键 -> Rename。
*   **视觉**: 文件名变为输入框，背景高亮，文字垂直居中对齐。
*   **反馈**: 回车确认时，若文件名冲突，输入框边框变红（无震动，仅颜色提示）。

### 3.4 删除文件 (Deletion)
*   **交互**: 右键 -> Delete 或 快捷键。
*   **确认**: 
    *   不使用系统弹窗。
    *   使用 **"Inline Confirm" (行内确认)** 或 **"Toast Undo"**。
    *   **方案**: 使用极简的 **"Inline Confirm" (行内确认)** 或模态弹窗。
    *   删除前必须二次确认，删除后文件在列表中消失（折叠动画）。

### 3.5 图片粘贴 (Image Paste)
*   **动作**: Cmd+V 粘贴图片。
*   **过程**:
    1.  编辑器内立即出现图片占位符。
    2.  图片上层覆盖一层模糊滤镜 (Blur) + 居中 Loading Spinner。
    3.  后台保存完成 (写 `assets/`)。
    4.  **完成**: 模糊褪去，图片锐化显示。
*   **反馈**: 若失败，图片变红框遮罩，点击可重试。

### 3.6 自动保存 (Auto-Save)
*   **可视化**: 
    *   输入时：右下角状态点变为黄色/蓝色呼吸态。
    *   保存成功：变为绿色/灰色实心点，随即淡出消失。
    *   **失败**: 状态栏变为红色底条，显示 "Save Failed"，并提供 "Retry" 按钮。

## 4. 组件规范 (Design System Specs)

### 4.1 侧边栏 (Sidebar Item)
*   **Height**: 28px
*   **Font**: System UI, 13px
*   **Icon**: 
    *   Folder: 闭合/展开 箭头 (Chevron)。
    *   File: 纯文本图标或无图标（仅靠缩进）。
*   **State**:
    *   `Active`: 背景色 `HSL(205, 90%, 96%)`，文字深蓝，左侧 2px 蓝色指示条。
    *   `Hover`: 背景色 `HSL(0, 0%, 96%)`。

### 4.2 编辑器 (Editor)
*   **Headings**:
    *   `# H1`: 32px, Bold, `margin-bottom: 24px`, `color: Primary`.
    *   `## H2`: 24px, SemiBold, `margin-top: 40px`, `margin-bottom: 16px`.
    *   `### H3`: 20px, Medium.
*   **List**: 缩进 24px，Bullet 点使用强调色。
*   **Blockquote**: 左侧 4px 强调色边框，背景淡灰，字体斜体。
*   **Inline Code**: 背景色 `HSL(0, 0%, 94%)`, padding `2px 4px`, radius `4px`, font `Mono`.
*   **Code Block**: 
    *   背景: 深色 (即使在 Light Mode 也是深灰背景，增加对比)。
    *   拥有 macOS 风格红绿灯/语言标签的 Header。

### 4.3 弹窗与覆盖层 (Overlays)
*   **Shadow**: 大的弥散阴影 `0 20px 50px -12px rgba(0,0,0,0.25)`。
*   **Glass**: `backdrop-filter: blur(12px)` + 半透明白/黑背景。
*   **Border**: 1px 极细边框 `rgba(255,255,255,0.1)` 提升精致感。

## 5. 可访问性 (A11y)
1.  **Focus Ring**: 键盘操作 Focus 时，必须有清晰的蓝色光晕环绕 (3px blue ring)。
2.  **Contrast**: 确保正文与背景对比度至少 4.5:1 (WCAG AA)。
3.  **Motion Preference**: 检测系统 `prefers-reduced-motion`，若开启则关闭所有位移动画，仅保留透明度变化。

## 6. 功能映射表 (Must 12 Check)

| Must ID | 功能点 | UI 承载位置/形态 | 视觉/交互重点 |
| :--- | :--- | :--- | :--- |
| **01** | 即时渲染 | Editor Area | 样式切换无闪烁，输入即渲染，无预览切换 |
| **02** | 段落格式 | Editor (Shortcut) | 甚至不需要 Toolbar，按下快捷键即时变样 |
| **03** | 行内样式 | Editor (Shortcut) | 选中文字后的背景色与系统一致，样式叠加清晰 |
| **04** | 代码块 | Editor Block | 独立的深色容器，语法高亮必须准确 |
| **05** | 图片粘贴 | Editor Block | Loading 态 -> 成功态的细腻转换 |
| **06** | 自动保存 | Status Bar (Bottom Right) | 极简呼吸点，不干扰视觉，除非失败 |
| **07** | 撤销/重做 | Editor (Shortcut) | 无 UI，但需要由光标位置的平滑跳动来体现 |
| **08** | 侧边栏目录 | Sidebar (Left) | 树形结构，层级缩进清晰，支持点击切换文件 |
| **09** | 新建文件 | Sidebar Action | 行内展开输入框，而非弹窗 |
| **10** | 重命名 | Sidebar Action | 行内编辑态，回车确认 |
| **11** | 删除 | Context Menu | 必须二次确认，物理删除，无 Undo |
| **12** | 文件夹工作区 | Welcome Screen / Sidebar | 首次启动的引导页设计要极其优雅 |

## 7. 不做项 (Not in scope)
1.  **设置面板**: V1 无需设置界面，所有配置走默认或配置文件。
2.  **多标签页 (Tabs)**: 严格单页应用，文件切换即替换内容。
3.  **搜索栏 (Search)**: V1 仅通过肉眼在文件树查找，不实现全文搜索/文件名搜索 UI。
4.  **底部面包屑**: 若增加复杂度则砍掉，直接看 Sidebar 高亮即可。
5.  **自定义主题**: 仅提供 Light/Dark 随系统自动切换，不提供手动切换开关 UI。

---

## 8. 总结
本方案旨在通过**极简主义 (Minimalism)** 和 **微交互 (Micro-interactions)** 来实现“超越功能”的体验。在不增加任何后端逻辑复杂度的前提下，通过前端 CSS/Animation 的打磨，构建出世界级的写作体验。
