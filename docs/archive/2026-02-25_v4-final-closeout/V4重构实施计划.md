# V4 重构实施计划

- 状态：已执行（待覆盖率门禁补齐后收口）
- 最后更新时间：2026-02-24
- 关联文档：`docs/current/V4需求主文档.md`

## 1. 计划原则

1. 以最小可验证增量推进，每个阶段都可独立回滚。
2. 先建立基础设施，再迁移调用点，最后做系统回归。
3. 不以 Sprint 文档为交付单位，以代码状态和门禁结果为交付单位。

## 2. 工作分解（Phase）

### Phase 0：基线锁定与遗漏分流

目标：在动代码前确认范围和风险。

- 核对当前基线：Editor 行数、`console.error` 分布、跨 store 直连调用。
- 将 `V3遗留与V4待办` 中事项按“阻塞/不阻塞”分流。
- 锁定 V4 FR（FR-V4-01~04），避免执行期改 scope。

退出标准：`V4执行与证据包.md` 的 Baseline 区块填写完成。

### Phase 1：结构抽离（FR-V4-01 + FR-V4-04）

目标：拆分 Editor 结构并配置化硬编码。

- 新增 `src/ui/editor/components/` 与 `src/ui/editor/hooks/`。
- 将工具栏、查找替换、表格插入相关 UI 与逻辑拆分。
- 新增 `src/config/editor.ts`，迁移常量。

退出标准：`Editor.tsx` 显著收敛，并通过测试/构建/lint。

### Phase 2：行为收敛（FR-V4-02 + FR-V4-03）

目标：统一 action 层和错误处理入口。

- 新增 `src/state/actions/*`，封装跨 store 协调。
- 新增 `src/services/error/ErrorService.ts`，统一错误上报。
- 替换关键路径中的直接跨 store 调用与分散错误处理。

退出标准：关键路径完成迁移，回归通过。

### Phase 3：回归与收口

目标：完成 Go/No-Go 之前的质量闭环。

- 全量测试、构建、lint 与覆盖率校验。
- 关键业务流手工回归。
- 更新技术债务条目与遗留项去向。

退出标准：满足 `V4需求主文档.md` 的全部门禁。

## 4. 未完成功能导致的遗漏补充策略

1. 对“功能缺失但阻塞重构验证”的事项，作为 `V4-GAP-*` 子任务进入执行包。
2. 对“功能缺失但不阻塞重构主线”的事项，保持冻结，不进入 V4 FR。
3. 每个 `V4-GAP-*` 必须记录来源、是否阻塞、处理去向。

## 5. 风险控制

| 风险           | 触发信号                  | 应对                        |
| -------------- | ------------------------- | --------------------------- |
| 行为回归       | 测试失败或手工回归异常    | 暂停新增迁移，先修复再推进  |
| 拆分失控       | Editor 行数下降但耦合未降 | 回到 hooks/组件职责边界审查 |
| 文档与代码漂移 | 追溯矩阵证据为空          | 当日补齐证据，不跨阶段欠账  |
