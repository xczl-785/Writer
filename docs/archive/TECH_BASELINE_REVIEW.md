## 4. 评审过程中额外评估的方案

### 4.1 Cherry Markdown (Tencent)

- **评估结果**：🔴 **不采用**
- **理由**：
  1. **形态不符**：它是带重 UI 的成品编辑器，而非纯内核。
  2. **交互不符**：主打双栏预览架构，与 Writer 追求的单窗口“输入即渲染”目标冲突。
  3. **定制成本**：剥离其 UI 逻辑的成本高于基于 TipTap 自建。
