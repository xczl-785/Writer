## 已知限制 (长期架构级)

- Tauri 环境原生调用补测：Web 模式无法覆盖，需 Tauri 桌面环境验证（V3 遗留留档项）
- 从外部复制内容，粘贴到Writer中时，复制的内容会套上代码的样式（这些内容只是标准内容）。—— **[BUG-06: 机制性妥协]**
  - **根因分析**：由于 TipTap / ProseMirror 严格遵守 CommonMark Markdown 规范：**任何带有 4 个以上前导空格起始的文本段落，都会按标准被截获转换为“缩进代码块（Indented Code Block）”**。而当从 VSCode 或部分带代码高亮和层叠样式的网页拷贝时，剪贴板会携带大量无形的前导空格以及 `text/html` DOM 结构指纹。遇到普通编辑器（例如 Typora）会特意识别这些指纹并针对性清洗降级，但标准 TipTap 解析器会机械匹配 4 个空格从而错误命中代码块逻辑。
  - **处理结论**：考虑到后续对于真正的纯代码块的复制需求，不宜直接去势掉代码块的匹配逻辑。这需要一套专门的剪贴板深度清洗或协议（Clipboard Text Parser Hook），此项暂不修改盲动，作为长期优化的架构级特征去统筹。
